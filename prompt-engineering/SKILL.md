# Промпт-инжиниринг

## Когда использовать

- Создание нового промпта / системного промпта для LLM
- Улучшение или рефакторинг существующего промпта
- Выбор техник под конкретную задачу (аналитика, генерация, классификация)
- Структурирование больших промптов (>1000 токенов)
- Проектирование self-check / валидации для промптов

## Ключевые правила

### 11 принципов (из исследований Anthropic, OpenAI, MIT, Google)

1. **Структура = инструкции для внимания модели** -- форматирование направляет фокус LLM
2. **XML-теги для модульности** -- `<role>`, `<context>`, `<task>`, `<rules>`, `<output>`
3. **Разделители** -- `---` между секциями, `===` между примерами (+16-24% точности)
4. **Таблицы для 5+ элементов** -- табличная подача +40% точности
5. **Приоритеты через иерархию** -- Критично / Важно / Желательно
6. **Псевдокод для условий** -- IF/ELSE, SWITCH/CASE (+36% точность, -87% токенов)
7. **JSON Schema с enum** -- ограничивай выбор модели
8. **Примеры обязательны** -- минимум 2-3 (Few-Shot), разделённые `===`
9. **Self-check блок** -- чеклист в конце промпта (-50% ошибок)
10. **Схема-Примеры-Задача** -- универсальная структура
11. **System-2 для 30+ элементов** -- разбивка на чанки с промежуточными результатами

### Структура промпта (каноническая)

```xml
<glossary> -- термины (если >3)
<role> -- кто ты, специализация, подход
<context> -- данные (таблицы/JSON/списки)
<task> -- что делать (2-3 шага)
<rules> -- критично / важно / желательно
<logic> -- IF/ELSE для ветвлений
<output> -- формат выхода (JSON Schema / таблица)
<examples> -- 2-3 примера через ===
<self_check> -- чеклист валидации
```

### 28 техник (4 группы)

| Группа | Техники | Номера |
|--------|---------|--------|
| Организация промпта | Markdown, разделители, заголовки, КАПС, ASCII-рамки, глоссарий, таблицы, токенное разделение | 1-8 |
| Подача данных | XML-теги, маркировка источников, System-2 Counting, визуальные маркеры, JSON, Few-shot | 9-14 |
| Описание правил | YAML, TOML, псевдокод, MetaGlyph, иерархия правил, Prompt Decorators | 15-20 |
| Контроль формата | Плейсхолдеры, JSON Schema, TypeScript, MD-таблицы, лестница контроля, Схема-Примеры-Задача, бэктики, Self-check | 21-28 |

## Быстрый маршрут

1. Определи цель, вход, выход, ограничения
2. Выбери 5-10 техник (приоритет = эффект x простота)
3. Собери промпт по канонической структуре (glossary -> role -> context -> task -> rules -> output -> examples -> self_check)
4. Добавь self-check блок
5. Проверь: есть ли примеры? Есть ли приоритеты? Есть ли ограничения?
