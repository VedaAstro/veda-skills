<principles>
## 11 КЛЮЧЕВЫХ ПРИНЦИПОВ

1. **Структура = инструкции для внимания модели**
   Форматирование — это не украшение, а способ направить фокус LLM.

2. **XML-теги для модульности**
   `<role>`, `<context>`, `<task>`, `<rules>`, `<output>` — чёткие границы секций.

3. **Разделители между блоками**
   `---` между секциями, `===` между примерами (+16-24% точности).

4. **Таблицы для 5+ элементов**
   Табличная подача данных улучшает аналитику на +40%.

5. **Приоритеты через эмодзи**
   🔴 Критично / 🟡 Важно / 🟢 Желательно — модель видит иерархию.

6. **Псевдокод для условий**
   `IF/ELSE`, `SWITCH/CASE` вместо текста (+36% точность, -87% токенов).

7. **JSON Schema с enum**
   Ограничивай выбор модели (планеты только из списка, оценки 0-10).

8. **Примеры обязательны**
   Минимум 2-3 примера (Few-Shot), разделённые `===`.

9. **Self-check блок**
   Чеклист в конце промпта → модель проверяет сама себя (-50% ошибок).

10. **Схема-Примеры-Задача**
    Универсальная структура: сначала формат, потом примеры, потом задача.

11. **System-2 для 30+ элементов**
    Разбивай большие контексты на чанки с `│` и промежуточными результатами.
</principles>
