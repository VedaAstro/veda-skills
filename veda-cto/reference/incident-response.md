# Что делать если сайт упал

> Простыми словами: сайт - как магазин. Если дверь не открывается, сначала проверяем электричество, потом замок, потом дверь. Не начинаем с ремонта крыши.

## Шаг 1: Определить масштаб (30 секунд)

| Вопрос | Как проверить |
|--------|--------------|
| Какой именно сайт? | Открыть URL в браузере |
| Все сайты или один? | Проверить 2-3 других домена |
| Ошибка 502/503 или просто медленно? | Посмотреть что показывает браузер |

**Если ВСЕ сайты лежат** - скорее всего проблема с сервером или nginx. Идти к Шагу 2.
**Если ОДИН сайт** - проблема в конкретном приложении. Идти к Шагу 3.

## Шаг 2: Сервер и nginx (2 минуты)

```
# Сервер жив?
ssh root@109.73.194.217 "uptime && df -h / && free -m"
```

| Проблема | Признак | Решение |
|----------|---------|---------|
| Диск переполнен | df показывает 100% | Удалить логи: `truncate -s 0 /root/.pm2/logs/*.log` |
| Память кончилась | free показывает 0 available | `pm2 restart all` или найти утечку |
| nginx упал | Все сайты 502 | `systemctl restart nginx` |
| Сервер недоступен | SSH не подключается | Зайти в панель Beget/FirstVDS, перезагрузить |

## Шаг 3: Конкретное приложение (2 минуты)

```
# PM2 статус - какие процессы работают?
pm2 ls

# Логи проблемного приложения
pm2 logs <name> --lines 50
```

| Проблема | Признак в логах | Решение |
|----------|----------------|---------|
| Приложение крашится | `errored` в pm2 ls, рестарты | Читать логи, искать ошибку |
| Порт занят | EADDRINUSE | `pm2 delete <name> && pm2 start` |
| Модуль не найден | MODULE_NOT_FOUND | `cd /path && npm install && pm2 restart` |
| БД недоступна | connection refused :5432 | `systemctl status postgresql && systemctl restart postgresql` |
| Файлы не обновились | Старый контент | PM2 reload не перечитал симлинк → `pm2 delete + start` |

## Шаг 4: После починки

1. Проверить что сайт открывается
2. Проверить что API отвечает: `curl -s https://domain/api/health | head`
3. Посмотреть логи 1 минуту - нет ли повторных ошибок
4. Записать aha-card: что сломалось, почему, как починили

## Ловушка: два домена, один nginx

Если один сайт работает, а другой нет, но оба на одном приложении - проблема может быть в nginx конфиге. ПЕРВЫМ делом сравнить конфиги двух доменов:
```
diff /etc/nginx/sites-enabled/domain1 /etc/nginx/sites-enabled/domain2
```
