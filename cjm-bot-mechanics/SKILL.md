# CJM & Bot Mechanics

## Когда использовать
- Проектирование нового бота или переделка существующего
- Дизайн онбординга, welcome-цепочки
- Настройка авторассылок и nurture-drip цепочек
- Падает retention, нужна re-engagement механика
- Дизайн value ladder (бесплатное → платное)
- Добавление геймификации / вовлечения в бота

## Фреймворк: AARRR × Механики

Каждый этап воронки — отдельная задача с своими механиками и метриками.

| Этап | Задача | Ключевые механики | Метрика |
|------|--------|-------------------|---------|
| **Acquisition** | Привлечь в бота | Реферал, viral loop, лид-магнит, social proof | Новые юзеры/день |
| **Activation** | Довести до первой ценности | Micro-commitments, прогресс-бар, onboarding flow | % дошедших до "aha moment" |
| **Retention** | Вернуть повторно | Drip-цепочка, hook loop, персонализация, расписание контента | D1/D7/D30 retention |
| **Revenue** | Конвертировать в оплату | Value ladder, scarcity, social proof, trial→paid | Conversion rate, ARPU |
| **Referral** | Запустить виральность | Invite reward, share-кнопка, групповые механики | Viral coefficient (K>1) |

## Hook Loop (ядро вовлечения)

Каждое взаимодействие бота — это цикл:

```
ТРИГГЕР (пуш/broadcast) → ДЕЙСТВИЕ (нажал кнопку) → НАГРАДА (персональный контент) → ИНВЕСТИЦИЯ (данные/привычка)
```

**Правила:**
- Действие должно быть максимально простым (1 нажатие, не текст)
- Награда должна быть переменной (не одно и то же каждый раз)
- Инвестиция = юзер вкладывает что-то своё (данные, время, настройки) → повышает switching cost

## Принципы проектирования бот-воронки

### 1. Micro-commitments (лестница "да")
Не проси большое сразу. Цепочка маленьких согласий:
```
Нажми /start → Как тебя зовут? → Дата рождения → Время → Город → [карта готова!]
```
Каждый шаг — маленькое "да". К моменту карты юзер уже инвестировал 5 действий.

### 2. Один CTA на сообщение
Не давай выбор из 8 кнопок. Одно сообщение = одно действие. Меню — только когда юзер сам просит.

### 3. Первая ценность за < 60 секунд
Онбординг должен доставить "aha moment" быстро. Для VEDA: натальная карта + персональный инсайт.

### 4. Переменная награда
Одинаковый контент каждый день = скука → отписка. Варьируй:
- Формат (текст / картинка / голос / видеокружок)
- Глубина (краткий инсайт / развёрнутый разбор)
- Тема (карьера / отношения / здоровье / духовность)

### 5. Scarcity + Urgency (дозированно)
- Лимит сообщений/день → "осталось 3 звезды" (не спам, а ценность)
- Таймер на оффер → "расширенный прогноз доступен 24ч"
- Эксклюзивность → "доступно только подписчикам с картой"

### 6. Social proof в контексте
Не абстрактно "10,000 юзеров", а конкретно: "847 человек с таким же Солнцем уже получили прогноз"

### 7. Re-engagement через ценность
Юзер молчит 3 дня → НЕ "ты куда пропал?", А "У тебя через 2 дня интересный транзит Юпитера. Хочешь разбор?"

### 8. Value ladder (эскалация ценности)
```
Бесплатно: ежедневный прогноз (бот)
     ↓ micro-payment или action
Бесплатно+: расширенный разбор планеты (бот, за реферал или активность)
     ↓
Платно: диагностическая сессия 1-на-1 (60 мин, live)
     ↓
Платно: курс 7Н/5S (полная программа)
```

## Growth Loops (петли роста)

Помимо линейной воронки AARRR — проектируй замкнутые петли, которые компаундят рост без платного трафика.

| Тип петли | Механизм | Пример для VEDA |
|-----------|----------|-----------------|
| **Viral** | Юзер создаёт → делится вовне → новые юзеры приходят | "Поделиться своей картой" → ссылка в соцсети → новый юзер |
| **Usage** | Контент от юзера привлекает других | Публичный прогноз дня в канале → подписка на бота |
| **Collaboration** | Юзер приглашает знакомых для совместного использования | "Совместимость с партнёром" → оба регистрируются |
| **UGC** | Юзеры создают контент, другие приходят за ним | Отзывы/кейсы после диагностики → social proof |
| **Referral** | Прямое приглашение за вознаграждение | "Приведи друга → +3 звезды / бесплатный расширенный прогноз" |

**Как выбрать петлю:**
1. Определи что юзер уже делает (какой контент создаёт?)
2. Где минимальное трение для шеринга?
3. Что получает новый юзер по ссылке? (должна быть мгновенная ценность)
4. Начни с ОДНОЙ петли, доведи до K>0.5, потом добавляй вторую

**Для ботов VEDA самая естественная петля:** Collaboration (совместимость) + Viral (поделиться картой). Контент персонализирован → высокая мотивация шерить.

## Быстрый маршрут: проектирование воронки

1. **Определи "aha moment"** — какой результат заставит юзера сказать "вау"?
2. **Нарисуй путь к нему** — минимум шагов, каждый = micro-commitment
3. **Спроектируй hook loop** — триггер → действие → награда → инвестиция
4. **Добавь retention-цепочку** — что бот присылает на D1, D3, D7?
5. **Заложи re-engagement** — что если юзер молчит 3/7/14 дней?
6. **Встрой value ladder** — где переход free → paid?
7. **Определи метрики** — что мерить на каждом этапе?
