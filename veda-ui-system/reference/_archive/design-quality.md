# Design Quality Principles — Veda UI

Эти принципы применяются к ЛЮБОМУ UI-элементу, странице или компоненту.
Источник: адаптация impeccable (pbakaus/impeccable) под наш стек.

---

## АНТИ-ПАТТЕРНЫ — ЗАПРЕЩЕНО

Никогда без исключений:

| Запрещено | Почему | Вместо |
|-----------|--------|--------|
| Inter / Roboto / Arial вместо system stack | Generic, безликий | Untitled UI использует встроенный font stack |
| Серый текст на цветном фоне | Плохой контраст | `text-primary_on-brand` / `text-white` |
| Карточки внутри карточек | Визуальный шум, путаница иерархии | Flat list или разделитель |
| Большие числа + метрики как hero-блок | Клише дашборда, дёшево | Metric items с контекстом |
| Cyan-on-dark / purple-to-blue градиенты | 2018 стиль | Brand tokens или монохром |
| Glassmorphism как декорация | Дорого, неуместно | Чистый bg-primary / bg-secondary |
| Bounce / elastic easing | Игриво, не профессионально | `ease-out` или `ease-in-out` |
| Emoji в UI-элементах | Уже в SKILL.md | 3D WebP |
| `text-gray-*` / `bg-blue-*` | Уже в SKILL.md | Семантические токены |

## ТИПОГРАФИКА

### Иерархия (не больше 3 уровней на экране)
```
Заголовок: text-display-sm — text-display-xl (bold/semibold)
Тело: text-md — text-lg (regular/medium)
Вспомогательный: text-sm (regular, text-tertiary)
```

### Правила
- Максимум **2 размера шрифта** в одном компоненте
- Line-height для заголовков: `leading-tight` (1.2-1.3)
- Line-height для тела: `leading-normal` (1.5-1.6)
- Не mix bold + italic в одной фразе — выбери одно
- Числа в таблицах: `tabular-nums` (моноширинные цифры)

### Запрещённые размеры
- < 11px (нечитаемо, уже в SKILL.md)
- > 72px без явной причины (hero-only)

## ЦВЕТ И КОНТРАСТ

### Минимальные контрасты (WCAG AA)
- Обычный текст (< 18px): **4.5:1**
- Крупный текст (≥ 18px bold или ≥ 24px): **3:1**
- Иконки / UI-компоненты: **3:1**

### Правила
- Используй `text-primary` на светлом, `text-primary_on-brand` на тёмном/brand фоне
- Никогда pure black (`#000`) на white — используй `text-primary` (off-black)
- Для акцентов: brand tokens, не `text-blue-*`
- Dark mode: всё через семантические токены — они автоматически инвертируются

## ПРОСТРАНСТВО И СЕТКА

### 8-пойнтовая сетка (ОБЯЗАТЕЛЬНО)
Используй только кратные 4: `4 8 12 16 20 24 32 40 48 64 80 96`
Tailwind: `p-2 p-3 p-4 p-5 p-6 p-8 p-10 p-12 p-16 p-20 p-24`

### Breathing room (воздух в дизайне)
```
Между секциями страницы: gap-16 — gap-24
Между группами элементов: gap-8 — gap-12
Между связанными элементами: gap-3 — gap-6
Внутри компонента: gap-2 — gap-4
```

### Визуальная иерархия через spacing
- Близкое = связанное. Далёкое = несвязанное.
- Не делай одинаковые отступы везде — это убивает иерархию

## MOTION И АНИМАЦИИ

### Принципы
- Анимация должна иметь **смысл** — показывать отношения или состояния
- Длительность: `100ms` (hover/toggle) → `200ms` (появление) → `300ms` (сложные переходы)
- Никогда > 400ms без уважительной причины

### Easing
```css
/* Появление */     ease-out   (начинает быстро, замедляется)
/* Исчезновение */  ease-in    (начинает медленно, ускоряется)
/* Перемещение */   ease-in-out
/* Hover/toggle */  linear (100ms)
```

### Staggering (для списков)
- Задержка между элементами: 30-50ms (не больше)
- Направление: top-to-bottom или left-to-right (по чтению)

### Reduced motion (ОБЯЗАТЕЛЬНО)
```tsx
// Tailwind
className="motion-reduce:transition-none motion-reduce:transform-none"
// Framer Motion
const prefersReducedMotion = useReducedMotion()
transition={{ duration: prefersReducedMotion ? 0 : 0.2 }}
```

## UX-ТЕКСТЫ

### Кнопки (action-oriented, конкретно)
```
❌ "ОК" / "Отмена" / "Продолжить"
✅ "Сохранить изменения" / "Отменить" / "Перейти к оплате"

❌ "Нажмите кнопку ниже"
✅ "Зарегистрироваться"
```

### Сообщения об ошибках (конкретно, actionable)
```
❌ "Ошибка" / "Что-то пошло не так"
✅ "Неверный формат email. Проверьте и повторите."
✅ "Не удалось загрузить. Попробуйте ещё раз или свяжитесь с поддержкой."
```

### Placeholder-тексты (пример, не инструкция)
```
❌ "Введите ваше имя"
✅ "Алексей Щекотов"

❌ "Укажите дату рождения"
✅ "01.01.1990"
```

### Empty states (дружелюбно + действие)
```
❌ "Нет данных" / "Пусто"
✅ "Консультаций пока нет. Запишитесь на первую диагностику →"
```

### Loading states (что именно загружается)
```
❌ "Загрузка..."
✅ "Строим вашу карту..." / "Сохраняем изменения..."
```

## ИНТЕРАКТИВНОСТЬ

### Focus states (ОБЯЗАТЕЛЬНО для a11y)
- Все интерактивные элементы должны иметь видимый focus ring
- Untitled UI обеспечивает это автоматически через React Aria
- Никогда `outline: none` без альтернативного индикатора

### Loading patterns
- Кнопка при загрузке: `isLoading` prop (встроено в Untitled UI)
- Страницы: skeleton, не spinner (skeleton = меньше layout shift)
- Данные в таблицах: skeleton rows той же высоты

### Form validation
- Показывай ошибки после blur (не при вводе)
- Исключение: passwordMatch — можно real-time после первого submit
- `isInvalid` + `hint` на Input/Select — встроено в Untitled UI
