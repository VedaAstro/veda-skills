# CEO Consulting -- PGAC Framework

> Каждый вопрос CEO -- шаг управленческого цикла. Ответ ведет к действию, не к новому вопросу.

Методология: **Driver-Based Planning** + **PDCA** + **DMAIC** + **Balanced Scorecard**.
Источник Grove-части: `/Users/alex/Projects/Claude_Docs/high_output_management_playbook.md`

## IF/THEN Router

```
IF "как дела" / "что по бизнесу"    -> GAP-ответ: result план/факт/ГЭП + виновный driver + рекомендация
IF "что по продажам"                 -> декомпозиция по источникам + ГЭП по каждому
IF "почему просело X"                -> RCA-цепочка: result -> driver -> health -> корневая причина
IF "что делать"                      -> ACT: driver + действие + прогноз + задача
IF "у кого проблемы"                 -> зоны ответственности + ГЭП по каждой (секция 4)
IF задача на исполнение              -> обработай как задачу, не анализируй
IF bottleneck / leverage / control / maturity -> READ reference/management-os.md
IF нужен шаблон 1:1 / weekly / review       -> READ reference/templates.md
IF нужен пример формата ответа              -> READ reference/examples.md
IF есть ГЭП                          -> ВСЕГДА переводи в рубли
IF есть просадка                     -> driver + корневая причина + рекомендация
IF нет данных                        -> скажи прямо, не выдумывай цифры
```

## 1. Три типа метрик

**RESULT** (lagging) -- следствие, нельзя влиять напрямую. Выручка, прибыль, LTV, retention.
**DRIVER** (leading) -- рычаг, можно влиять действием. CR шагов, лиды, средний чек, скорость ответа.
**HEALTH** (diagnostic) -- контекст, объясняет почему driver просел. NPS, uptime, % ошибок.

Правило декомпозиции: Result ВСЕГДА раскладывается на Drivers.
```
Выручка = Лиды x CR x Средний чек
```
Result просел -> ищи какой driver. Driver просел -> ищи в health.
**Никогда не "поднимай выручку" -- поднимай конкретный driver.**

## 2. Цикл PGAC (12 шагов, спринт 1-2 нед.)

### PLAN
1. Target: план по result-метрикам
2. Decompose: план по каждому driver (`выручка 500K = 200 лидов x 25% CR x 10K СЧ`)

Без плана нет ГЭПа. Без ГЭПа нет управления.

### GAP
3. План vs Факт -> ГЭП (абсолют и %)
4. Какой driver просел (декомпозиция ГЭПа)
5. Тренд: было -> стало (динамика, не точка)
6. Цена ГЭПа в рублях
7. RCA: корневая причина через health-слой

### ACT
8. Решение: на какой driver влияем
9. Прогноз: driver +X% -> result +Y руб. (фиксируется ДО действия)
10. Задача: конкретная, с ответственным, с дедлайном

### CHECK
11. Факт после спринта vs прогноз из шага 9
12. Вывод: сработало -> масштабировать / нет -> новая гипотеза -> PLAN

## 3. Формула прибыли VEDA

```
Прибыль = Выручка - Расходы

Выручка = Выр_Трафик + Выр_База + Выр_Соцсети + Выр_LTV

Каждый источник: Выр = Охват x C1 x C1.5 x C2a x C2b x СЧ
  Охват .......... бюджет / контент-план / рассылки
  C1 (-> рег.) ... лендинг, креатив, оффер
  C1.5 (-> КЛ) .. квалификация, скрипт бота
  C2a (-> заказ) . вебинар, оффер, диагностика
  C2b (-> оплата)  скрипт ОП, скорость ответа
  СЧ ............. тарифная сетка, апселл
```

Полная таблица метрик и 8 комбинаций источник x канал -> `reference/examples.md`

### Cash flow != декомпозиция прибыли
Автооплаты (рассрочка), дебиторка, мелкочеки -- атрибуты платежа, не элементы воронки.
**Тест**: можно ли повлиять через DRIVER воронки? Нет -> не элемент формулы.

## 4. Зоны ответственности

```
МАРКЕТИНГ (Света):  Охват, C1, C1.5, C2a -> KPI: план горячих заказов
ОД (диагностики):   заявки + конверсия ДС -> KPI: план заказов из диагностик
ОП (продажи):       C2b, ручные заказы    -> KPI: конверсия оплат + план ручных
ФИНАНСЫ (Катерина): дебиторка, рассрочки   -> KPI: % собранной дебиторки
ОПЕРАЦИИ (Наталья): логистика задач        -> KPI: % задач в срок
```

ГЭП по зоне = потери в рублях. Приоритет: зона с максимальными потерями.

---

## 5. Правила ответа

1. **Рубли > проценты**: ГЭП по result всегда в рублях -- CEO думает деньгами
2. **Тренд > точка**: метрика = динамика (было -> стало), не одно число
3. **Действие > констатация**: каждый ответ заканчивается рекомендацией или задачей
4. **Один рычаг**: не "все плохо", а "главный рычаг X, потому что Y"
5. **Цена бездействия**: "надо ли?" -> посчитай потери в день/неделю
