# PGAC Management OS

Источник: `/Users/alex/Projects/Claude_Docs/high_output_management_playbook.md`

## 1. Output функции

- Для каждой роли фиксируй основной результат, единицу измерения и критерий качества
- Activity не заменяет output
- Менеджер оценивается по системе результата, а не по личной перегрузке

## 2. Bottleneck

- Главный рост приходит через снятие ограничения, а не через локальную полировку всего подряд
- Ищи очередь, перегрузку, возвраты, нестабильное качество входа, зависимость от одного человека

## 3. Leverage

- Самые сильные действия менеджера: информация, решение, делегирование с калибровкой, обучение, дизайн процесса
- Вопрос дня: где мой следующий час даст непропорциональный эффект на output системы

## 4. Cadence

- `One-on-one`: 1 раз в 1-2 недели
- `Weekly review`: каждую неделю
- `Decision meeting`: только под выбор
- `Monthly performance review`: 1 раз в месяц
- `Quarterly planning review`: раз в квартал

## 5. Control Modes

- `Market`: когда результат прозрачно измерим
- `Contractual`: когда нужны owner, срок, критерий качества
- `Cultural`: когда полное формализованное описание невозможно, но стандарт должен соблюдаться
- В реальности нужен микс, а не один режим

## 6. Task-Relevant Maturity

- Зрелость определяется на уровне конкретной задачи
- Низкая зрелость -> directive
- Средняя -> coaching
- Высокая -> delegation

## 7. Performance

- Смотри на output, повторяемость, влияние на смежников, скорость обучения
- Трудный разговор строится через факты и бизнес-влияние, а не через эмоции
