# PGAC Examples

## Полный цикл PGAC (пример VEDA)

```
PLAN:
  Result: Выручка = 2M /мес
  Декомпозиция: Трафик 1.2M + База 400K + Соцсети 200K + LTV 200K
  Drivers: Охват 50K, C1 4%, C2a 15%, C2b 70%, СЧ 12K
  По заказам: горячих 80, ручных 30, из ДС 20

GAP:
  Выручка факт 1.6M -> ГЭП -400K
  Декомпозиция ГЭПа: Трафик -300K (охват ок, C2a просел с 15% до 10%)
  Цена: C2a -5pp = -25 заказов x 12K = -300K
  RCA: сменили формат вебинара -> конверсия просела

ACT:
  Driver: C2a (КЛ -> заказ)
  Действие: вернуть старый формат вебинара + добавить кейс
  Прогноз: C2a +3pp -> +15 заказов -> +180K

CHECK:
  C2a факт: 12% (было 10%, прогноз 13%) -> частично сработало
  Следующий цикл: усилить оффер (гипотеза: слабый CTA)
```

## Паттерны ответов CEO

### "Как дела?" -> GAP-ответ
```
Выручка: 1.6M из 2M плана (ГЭП -400K).
Просел C2a (конверсия вебинара): 10% вместо 15%.
Цена: -25 заказов x 12K = -300K.
Рекомендация: вернуть старый формат + усилить кейсы.
```

### "Что по продажам?" -> декомпозиция по источникам
```
Трафик: 1.0M (план 1.2M, ГЭП -200K) - C2a просел
База: 350K (план 400K, ГЭП -50K) - охват ок, CR ниже
Соцсети: 180K (план 200K) - в норме
LTV: 210K (план 200K) - перевыполнение
Итого: 1.74M, ГЭП -260K. Главный рычаг: C2a трафика.
```

### "Почему просело X?" -> RCA-цепочка
```
Выручка просела -> C2a (вебинар -> заказ) просел с 15% до 10%
-> NPS вебинара упал с 8.2 до 6.5 -> сменили формат (убрали кейсы)
Корневая причина: решение от 01.03 убрать блок кейсов.
```

### "Что делать?" -> ACT-ответ
```
Driver: C2a (конверсия вебинара)
Действие: вернуть блок кейсов + добавить social proof
Прогноз: C2a +3pp -> +15 заказов -> +180K/мес
Задача: [конкретное поручение, исполнитель, дедлайн]
```

### "У кого проблемы?" -> зоны + ГЭП
```
Маркетинг (Света): ГЭП по горячим заказам -15 шт -> потери 150K
ОП: C2b 60% при плане 75% -> потери 90K
ОД: конверсия ДС 40% при плане 50% -> потери 60K
Приоритет: маркетинг (макс. потери).
```

## Классификация метрик VEDA (справочник)

| Метрика | Тип | Частота |
|---------|-----|---------|
| Прибыль | RESULT | Месяц |
| Выручка (общая) | RESULT | Неделя / месяц |
| Выручка по источнику | RESULT | Неделя |
| Маржа, ROMI | RESULT | Месяц |
| Охват | DRIVER | День / неделя |
| C1 (охват -> рег.) | DRIVER | Неделя |
| C1.5 (рег. -> КЛ) | DRIVER | Неделя |
| C2a (КЛ -> заказ) | DRIVER | Неделя |
| C2b (заказ -> оплата) | DRIVER | День / неделя |
| СЧ (средний чек) | DRIVER | Неделя |
| Кол-во заказов (горячие/ручные/ДС) | DRIVER | Неделя |
| Возвраты, доля расходов | HEALTH | Месяц |
| Скорость ответа ОП, NPS, % прочтения | HEALTH | По запросу |
