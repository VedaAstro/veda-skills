**Версия:** 1.0
**Дата:** 2026-02-17

## 0. КРИТИЧНОЕ: РЕЧЬ ДОЛЖНА ЛИТЬСЯ

> **Главная ошибка агента — рубленная, телеграфная речь. Этот раздел — приоритет №1.**

### 0.1 Принцип: читай вслух — если запнулся, переписывай

Речь спикера (колонка `notes`) — это живая, льющаяся речь. Каждое предложение СВЯЗАНО со следующим. Нет обрубков. Нет телеграмм. Нет списков из существительных без глаголов.

**Тест качества:** Прочитай notes вслух. Если где-то остановился, чтобы додумать связь — там нет связки. Исправь.

### 0.2 Связки ОБЯЗАТЕЛЬНЫ между предложениями

| Тип связки | Примеры | Когда |
|------------|---------|-------|
| Продолжение | и вот, а теперь, так вот, и знаете что | После утверждения |
| Контраст | но, а, однако, при этом | Когда переворачиваем |
| Причина | потому что, дело в том что, знаете почему? | Объясняем |
| Следствие | поэтому, и тогда, в итоге, и вот что получается | Вывод |
| Вопрос-мост | Знаете почему? Что это значит? И что дальше? | Между блоками |
| Обращение | дорогие, смотрите, понимаете, представьте | Привлечение внимания |

### 0.3 Примеры: ПЛОХО vs. ХОРОШО

❌ **ПЛОХО (телеграфный стиль, обрубки):**
```
Сейчас — научный факт. Закон Бенфорда. Если взять данные — цифра 1 будет первой в 30% случаев. Не 11%. Тридцать. Аудиторы используют. Вселенная по правилу. Дата рождения — тоже.
```
**Проблема:** Точка-стоп после каждой мысли. Нет связок. Читатель спотыкается. Спикер не может произнести это как живую речь.

✅ **ХОРОШО (льётся):**
```
А сейчас — научный факт, который вас удивит. Закон Бенфорда. Смотрите: если взять любые реальные данные — высоты гор, налоговые декларации, население городов — то цифра 1 будет первой в 30% случаев. Не 11%, как по теории вероятности, а тридцать. И знаете, кто этим пользуется? Аудиторы — они ловят мошенников этим законом. Потому что если цифры выдуманы, закон нарушается. Вселенная распределяет хаос по строгому правилу. И ваша дата рождения — тоже не случайность.
```
**Почему хорошо:** «Смотрите:» → «Не 11%, а» → «И знаете, кто» → «Потому что» → «И ваша дата» — каждая фраза цепляется за предыдущую.

---

❌ **ПЛОХО (конкретика жизни, но обрубки):**
```
Зарплата пришла. Ипотека списалась. Коммуналка. Кружок дочке. Зуб. Кроссовки. И ноль на счету.
```

✅ **ХОРОШО (та же конкретика, но льётся):**
```
Вот как это выглядит в жизни. Зарплата пришла тридцатого, а первого уже списалась ипотека. Пятого заплатили за коммуналку, десятого отдали за кружок дочке. Пятнадцатого зуб заболел — три тысячи на пломбу. Двадцатого у ребёнка кроссовки развалились. И вот двадцать пятого вы смотрите в телефон и думаете: как дотянуть эти пять дней до зарплаты? Знакомо?
```

---

❌ **ПЛОХО (абстрактная боль):**
```
Многие женщины чувствуют нереализованность. Финансовые затруднения. Отсутствие баланса. Потеря жизненных ориентиров.
```

✅ **ХОРОШО (конкретная боль, живая речь):**
```
Дорогие, я знаю, что многие из вас сейчас чувствуют одно и то же. Утром встаёте — и первая мысль не «о, новый день», а «господи, опять». Работа, которую не выбирали. Зарплата, которой не хватает. И где-то внутри тихий голос: «Неужели это всё? Неужели так и будет до конца?» Знакомо? Если да — напишите плюс в чат.
```

### 0.4 Антипаттерны речи (notes)

| Антипаттерн | Пример | Почему плохо | Как исправить |
|-------------|--------|--------------|---------------|
| Телеграфный стиль | «Сатурн. Карма. Долги.» | Нет глаголов, нет связи | «Сатурн в вашей карте отвечает за долги. И знаете что? Он их собирает, как строгий кредитор.» |
| Обрубки | «Факт. Бенфорда. Данные.» | Спотыкаешься при чтении | Добавь связки: «А сейчас — факт, который...» |
| Канцелярит | «Данный метод позволяет осуществить» | Убивает живость | «Этот метод работает так:» |
| Абстракции | «Финансовые затруднения» | Не узнаваемо | «Денег не хватает до зарплаты» |
| Незавершённые мысли | «Сатурн забирает» (что?) | Надо додумывать | «Сатурн забирает то, что вы ему должны» |
| Выдуманные цитаты | Любые кавычки без источника | Ложь | Убрать или дать источник |
| Повторы интерактива | «Напишите да» 5 раз подряд | Надоедает | Варьировать: плюс, цифру, одно слово |

### 0.5 Объём notes

| Параметр | Значение |
|----------|----------|
| Слов на слайд | 80-120 (30-50 секунд речи) |
| Цифры | Цифрами: 30%, 5 000₽, 15 лет. Не словами |
| Финальный тест | Прочитай вслух. Запнулся → переписывай |

## 1. TSV-ФОРМАТ: СТРУКТУРА ТАБЛИЦЫ

### 1.1 Колонки

| Колонка | Имя | Что содержит | Обязательно |
|---------|-----|--------------|-------------|
| A | № | Порядковый номер слайда | Да |
| B | раздел | Блок структуры (например, `1.1 Захват`, `2.3 Метафора`, `3.2 Кейс`) | Да |
| C | title | Заголовок слайда | Да |
| D | body | Тело слайда (визуальный текст) | Да |
| E | notes | Речь спикера (полный текст, который произносится) | Да |
| F | img_tag | Имя файла из каталога (например, `image89.png`) | По содержанию |

### 1.2 Переносы строк внутри ячеек

TSV не поддерживает переносы строк внутри ячейки. Для визуальных переносов используется разделитель:

```
 · 
```

Это: `пробел` + `·` (middle dot) + `пробел`.

**Пример body в TSV:**
```
💰 Юпитер наполняет кошелёк · ↓ · 🪨 Сатурн списывает долги · ↓ · 📉 Деньги утекают быстрее, чем приходят
```

**Как отобразится на слайде:**
```
💰 Юпитер наполняет кошелёк
    ↓
🪨 Сатурн списывает долги
    ↓
📉 Деньги утекают быстрее, чем приходят
```

### 1.3 Пример полной строки TSV

```
12	2.3 Метафора	Сатурн забирает твои деньги	💰 Юпитер наполняет кошелёк · ↓ · 🪨 Сатурн списывает долги · ↓ · 📉 Деньги утекают быстрее, чем приходят	Дорогие, знакомая ситуация? Зарабатываете, вроде всё делаете правильно — а на счёте ноль. И вы думаете: я что-то не так делаю, мало стараюсь. А на самом деле это не вы. Это конфликт планет в вашей карте — Юпитер даёт, а Сатурн забирает. И пока вы не увидите, где утечка — деньги будут уходить. Напишите плюс, если узнали себя.	image57.png
```

### 1.4 Нумерация разделов

Раздел в колонке B формируется по шаблону:

```
{номер_блока}.{номер_подблока} {Короткое_название}
```

**Примеры:**
```
1.1 Захват
1.2 Сегодня
1.3 Расскажите
2.1 Про мир
2.3 Метафора
3.1 Вопрос
3.2 Кейс
4.1 Оффер
```

## 2. TITLE (ЗАГОЛОВОК СЛАЙДА)

### 2.1 Требования

| Правило | Пояснение |
|---------|-----------|
| Полное предложение | Подлежащее + сказуемое. Мысль завершена |
| До 10 слов | Короткий, ёмкий, считывается за 1 секунду |
| Вывод, а не тема | Понятен без контекста. Не «О Сатурне», а «Сатурн забирает твои деньги» |
| Разговорный тон | Как сказала бы подруге, а не на конференции |
| КАПС — только акценты | 1-2 слова максимум на ключевом. Не весь заголовок |
| Цифры — цифрами | 3 млн, 8 месяцев, 40%. Не «три», «восемь» |

### 2.2 Хорошие примеры

```
Сатурн забирает твои деньги
Почему ты работаешь на износ, а денег нет
Не ты виновата — это записано в карте
ДОЛГИ 3 МЛН → закрыла за 8 месяцев
Вот как одна планета меняет всё
Ты уже сделала то, что делают астрологи
```

### 2.3 Плохие примеры и исправления

| ❌ Плохо | Почему | ✅ Как исправить |
|---------|--------|-----------------|
| САТУРН И ДЕНЬГИ | КАПС на всё + нет глагола | Сатурн забирает твои деньги |
| ДОЛГИ. КРЕДИТЫ. НИЩЕТА. | КАПС на всё + телеграф | ДОЛГИ 3 МЛН → закрыла за 8 месяцев |
| Информация о Сатурне | Канцелярит, тема а не вывод | Вот как Сатурн влияет на доход |
| О влиянии планет | Абстракция, нет субъекта | Планеты определяют твой доход |
| Причина проблем | Каких проблем? Неясно | Вот почему деньги утекают |
| Важно знать | Что важно? Не информативно | Это меняет всё |
| Финансовый потенциал в карте | Канцелярит + абстракция | Твоя карта показывает, где деньги |

## 3. BODY (ТЕЛО СЛАЙДА)

### 3.1 Требования

| Правило | Пояснение |
|---------|-----------|
| Полные предложения | Каждая строка — законченная мысль с глаголом |
| Максимум 4 строки | Не считая стрелок ↓ |
| Читается за 3 секунды | Взгляд схватывает сразу |
| Стрелки ↓ | Между смысловыми шагами: было → стало, причина → следствие |
| Эмодзи по смыслу | Не для красоты, а для навигации глаза |
| Не помещается → дели | Если контент не влезает в 4 строки — разбей на 2-3 слайда |
| Цифры — цифрами | 40%, 3 000 000₽, 8 месяцев. Не словами |

### 3.2 Паттерны body

**Кейс-трансформация:**
```
👩‍💼 Елена, 43 года, бухгалтер из Самары
❌ Было: долги 3 000 000₽, паника каждый месяц
    ↓
✅ Стало: закрыла долги за 8 месяцев
```

**Статистика:**
```
📊 40% россиян живут без сбережений
👩‍💼 Работают на износ, встают в шесть утра
    ↓
❓ Почему деньги не копятся?
```

**Цитата + решение:**
```
💭 «Работаю на износ — а денег всё равно нет»
    ↓
🔍 Причина спрятана в натальной карте
    ↓
✅ Её можно найти и исправить
```

**Метафора:**
```
💰 Юпитер наполняет твой кошелёк
🪨 Сатурн списывает долги
    ↓
📉 Деньги утекают быстрее, чем приходят
```

**Список-выбор (интерактив):**
```
Какой тип узнали в себе?
1️⃣ Зарабатываю, но не копится
2️⃣ Боюсь тратить на себя
3️⃣ Деньги приходят и сразу уходят
```

### 3.3 Антипаттерн — телеграфный стиль body

| ❌ Плохо | Почему | ✅ Как исправить |
|---------|--------|-----------------|
| 📊 Статистика | Чья? О чём? | 📊 40% россиян живут без сбережений |
| 👩‍💼 Работа. Усталость. | Нет глагола, обрубки | 👩‍💼 Они работают на износ каждый день |
| ❓ Деньги? | Что «деньги»? | ❓ Почему деньги не накапливаются? |
| 🔍 Причина в карте | Причина чего? | 🔍 Причина спрятана в натальной карте |
| ✅ Можно изменить | Что можно? | ✅ Это можно изменить |
| ↓ Результат | Какой результат? | ↓ Долги закрыты за 8 месяцев |

**Правило:** Каждая строка body — законченная мысль. Если убрать эмодзи и прочитать — должно быть понятно без контекста.

## 4. NOTES (РЕЧЬ СПИКЕРА) — РАСШИРЕННЫЕ ПРАВИЛА

> **Этот раздел дополняет §0 «Критичное». Перечитай §0 перед каждым сеансом написания слайдов.**

### 4.1 Структура речи на одном слайде

```
[Обращение / связка с предыдущим] → [Тезис] → [Раскрытие] → [Интерактив / мост к следующему]
```

**Пример:**
```
Дорогие, а теперь — самое важное.             ← обращение + связка
Есть планета, которая буквально забирает       ← тезис
деньги из вашего кошелька. Это Сатурн.
Он работает как строгий кредитор — если вы      ← раскрытие через метафору
ему должны, он будет списывать. Без предупреждений,
без переговоров. И вот что интересно:
большинство людей даже не знают, что у них      ← связка к следующей мысли
есть этот долг.
Напишите плюс, если чувствуете, что деньги      ← интерактив
утекают непонятно куда.
```

### 4.2 Как связывать слайды между собой

Notes текущего слайда должны логически ПОДВОДИТЬ к следующему. Конец notes — это мост.

| Тип перехода | Формула | Пример |
|-------------|---------|--------|
| К следующему факту | «И знаете, что самое интересное?» | «И знаете, что самое интересное? Это работает не только с деньгами.» |
| К кейсу | «Вот вам реальная история.» | «А вот как это выглядит в жизни. Расскажу историю Марины.» |
| К практике | «А давайте проверим прямо сейчас.» | «Хотите увидеть это в своей карте? Давайте попробуем прямо сейчас.» |
| К блоку боли | «Но тут есть нюанс.» | «Но есть один нюанс, о котором обычно не говорят.» |
| К решению | «И вот что с этим делать.» | «Ну хорошо, проблему увидели. А вот что с этим можно сделать.» |
| К офферу | «Именно для этого я создала...» | «Именно поэтому два года назад я создала программу, которая...» |

### 4.3 Частые ошибки и исправления

**Ошибка 1: «Список вместо речи»**
```
❌ Причины финансовых проблем:
   - Неправильная работа
   - Отсутствие сбережений
   - Кредитная нагрузка
   - Негативные установки

✅ Дорогие, давайте разберёмся, почему так получается.
   Первое — вы работаете не на своём месте,
   и каждый день тратите энергию впустую.
   Второе — откладывать не получается, потому что
   расходы всегда догоняют доходы.
   И третье — многие из вас тянут кредиты,
   которые взяли, чтобы закрыть кассовый разрыв.
   А под всем этим — установка, которую вы
   даже не осознаёте. Какая? Сейчас покажу.
```

**Ошибка 2: «Одно предложение без развития»**
```
❌ Сатурн влияет на финансы.

✅ Сатурн — это планета, которая отвечает за ваши
   долги и ограничения. И дело вот в чём: если
   Сатурн стоит в определённом доме вашей карты,
   он буквально «собирает» с вас плату за
   непройденные уроки. Звучит жёстко? Но на самом
   деле это справедливо — и самое главное,
   это можно увидеть и исправить.
```

**Ошибка 3: «Формальный интерактив»**
```
❌ Напишите да, если согласны.

✅ Напишите в чат одно слово — то, которое
   описывает ваше состояние прямо сейчас.
   Просто одно слово. Не думайте долго —
   первое, что пришло в голову.
```

## 5. IMG_TAG (ТЕГИ НА КАРТИНКИ)

### 5.1 Как работают картинки

1. В каталоге `CATALOG_STICKMAN.tsv` лежит реестр всех доступных изображений
2. Каждое изображение имеет: `filename`, `tags`, `description`
3. В колонке `img_tag` TSV-таблицы слайдов указывается имя файла (например, `image57.png`)
4. Скрипт при экспорте в Google Slides подставляет картинку на слайд

### 5.2 Каталог изображений

**Расположение:** `Claude_Docs/Images/CATALOG_STICKMAN.tsv`
**Папка с файлами:** `Claude_Docs/Images/Из презы стикман/`
**Всего изображений:** ~278 файлов (PNG, JPG, GIF)

**Формат каталога:**

| Колонка | Содержание |
|---------|------------|
| filename | Имя файла (`image1.png`, `image12.gif`) |
| tags | Ключевые слова через запятую |
| description | Короткое описание того, что на картинке |

### 5.3 Как подбирать img_tag

**Алгоритм:**

```
1. Определи ЭМОЦИЮ или СИТУАЦИЮ слайда
2. Найди подходящие теги в каталоге по совпадению
3. Выбери картинку, description которой ближе всего к смыслу слайда
4. Поставь filename в колонку img_tag
```

**Примеры подбора:**

| Содержание слайда | Ключевые теги | img_tag |
|-------------------|---------------|---------|
| Боль: «работаю на износ, денег нет» | выгорание, усталость, деньги | image83.png |
| Кейс: трансформация, успех | радость, успех, победа | image28.png |
| Практика: работа с картой | натальная_карта, астрология | image86.png |
| Метафора: путь из А в Б | путь, из_А_в_Б, движение | image58.png |
| Интерактив: «напишите в чат» | рука_вверх, вопрос, внимание | image47.png |
| Боль: «день сурка», рутина | день_сурка, рутина, круг | image117.png |
| Планета Сатурн | Сатурн, планета, астрология | image60.png или image109.png |
| Оффер: подарок, бонус | подарок, радость, бонус | image75.png |
| Возражения: «дорого, не справлюсь» | возражения, барьеры | image66.png |
| Мечта: звёзды, предназначение | мечты, звёзды, предназначение | image45.png |

### 5.4 Правила img_tag

| Правило | Пояснение |
|---------|-----------|
| Один тег на слайд | Только одно имя файла. Не список |
| Не каждый слайд | Если слайд — чисто текстовый (цитата, список), img_tag можно оставить пустым |
| Соответствие смыслу | Картинка усиливает сообщение, а не отвлекает |
| Не повторять подряд | Не ставить одну и ту же картинку на 3 слайда подряд |
| GIF для динамики | GIF-файлы (image12.gif и т.д.) — для энерджайзеров, переходов, демонстраций |
| Эмоция > буквальность | Лучше картинка с правильной эмоцией, чем буквальная иллюстрация |

### 5.5 Специальные изображения

| Тип | Файлы | Когда использовать |
|-----|-------|-------------------|

## 6. ЭКСПОРТ: PIPELINE CLAUDE → GOOGLE SLIDES

### 6.1 Workflow

```
┌──────────────────┐     ┌──────────────────┐     ┌──────────────────┐
│                  │     │                  │     │                  │
│   Claude / LLM   │────▶│    TSV-файл      │────▶│  Google Slides   │
│  (создаёт TSV)   │     │  (таблица)       │     │  (презентация)   │
│                  │     │                  │     │                  │
└──────────────────┘     └──────────────────┘     └──────────────────┘
         │                        │                        │
    Playbook_Slides          Google Sheets            Скрипт экспорта
    + Playbook_Hormozi       tab "slides"             (TSV → Slides)
    + контент-плейбуки       "Таблица → Преза"
```

### 6.2 Шаги

| Шаг | Что делаем | Где |
|-----|-----------|-----|
| 1 | LLM создаёт сценарий по Playbook_Hormozi | Claude / Chat |
| 2 | LLM генерирует TSV по формату §1 этого плейбука | Claude / Chat |
| 3 | TSV вставляется в Google Sheets (tab `slides`) | Google Sheets «Таблица → Преза» |
| 4 | Скрипт читает TSV и создаёт слайды в Google Slides | Apps Script / Python |
| 5 | Скрипт подставляет картинки по `img_tag` из папки Images | Автоматически |
| 6 | Проверка: речь вслух, визуал на слайдах | Вручную |

### 6.3 Правила генерации TSV для LLM

```
КОГДА ГЕНЕРИРУЕШЬ TSV:

1. Первая строка — заголовки: №	раздел	title	body	notes	img_tag
2. Разделитель колонок — TAB (не пробелы, не запятые)
3. Переносы внутри body и notes — через  ·  (пробел-точка-пробел)
4. Никаких кавычек вокруг ячеек (если нет спец. символов)
5. img_tag — ТОЧНОЕ имя файла из каталога (image57.png, не "деньги" или "грусть")
6. Каждый слайд — отдельная строка. Одна строка = один слайд
7. Нумерация № — сквозная по всему вебу (1, 2, 3... 50, 51...)
8. Раздел — наследуется до смены (если 3 слайда подряд в «2.1 Про мир», все три с этим разделом)
```

## 7. УДЕРЖАНИЕ И РИТМ НА УРОВНЕ СЛАЙДОВ

> **Детальная методология — в Playbook_Hormozi.md §3-4, §8.2, §9. Здесь — только то, что касается слайдового production.**

### 7.1 Петли и микро-петли

| Тип | Что это | Где в слайде |
|-----|---------|-------------|
| Петля | Открываем интригу в начале — закрываем в конце | notes: «Сегодня покажу самый простой способ...» |
| Микро-петля | Короткий тизер следующего блока | notes: «Через 5 минут покажу формулу, которая объяснит почему» |
| Netflix-крючок | Интрига за 30 сек до конца блока | notes: «А сейчас — то, ради чего стоило прийти. Но сначала...» |
| Мост между вебами | Петля на следующий веб | notes: «Завтра мы посчитаем Число Года — и вы поймёте, что вас ждёт в 2026» |

**Правило:** Каждый открытый крючок должен быть закрыт. Невыполненное обещание = потеря доверия.

### 7.2 Смена формата

Не держать один формат дольше 10-15 минут. Каждые 3-5 слайдов — смена типа:

| Тип слайда | Функция | Характерный body |
|------------|---------|-----------------|
| Монолог | Объясняет | Тезис + раскрытие |
| История / кейс | Вовлекает эмоционально | ❌ Было → ✅ Стало |
| Статистика | Доказывает цифрами | 📊 Цифра + вывод |
| Практика | Вовлекает действием | Инструкция + «Напишите результат» |
| Интерактив | Активирует | Вопрос + варианты ответа |
| Метафора | Помогает понять | Образ + связь с реальностью |

### 7.3 Интерактивы в слайдах

**Частота:** Каждые 3-5 минут (примерно каждые 4-6 слайдов).

**Правило:** Не повторять одну формулировку больше 2 раз за веб.

| Вместо | Использовать |
|--------|--------------|
| «Напишите да» (в 5-й раз) | «Напишите цифру от 1 до 10 — насколько точно?» |
| «Напишите да» (ещё раз) | «Какой тип узнали в себе?» |
| «Напишите да» (опять) | «Напишите плюс, если откликается» |
| «Напишите да» | «Что увидели в своей карте?» |
| «Напишите да» | «Напишите одно слово — какое ощущение прямо сейчас?» |

## 8. ЧЕКЛИСТЫ

### 8.1 Чеклист одного слайда

- [ ] **title** — полное предложение с подлежащим и сказуемым?
- [ ] **title** — понятен без контекста? (не «О Сатурне», а вывод)
- [ ] **title** — до 10 слов?
- [ ] **title** — КАПС только на 1-2 акцентных словах (или без КАПС)?
- [ ] **body** — каждая строка законченная мысль с глаголом?
- [ ] **body** — ≤4 строки (не считая ↓)?
- [ ] **body** — нет телеграфного стиля? (не «Статистика.», «Деньги?»)
- [ ] **notes** — полные предложения, СВЯЗКИ между ними?
- [ ] **notes** — 80-120 слов?
- [ ] **notes** — прочитан вслух без запинок?
- [ ] **notes** — нет канцелярита, обрубков, абстракций?
- [ ] **img_tag** — файл существует в каталоге?
- [ ] **img_tag** — картинка соответствует эмоции/содержанию?

### 8.2 Чеклист блока (5-10 слайдов)

- [ ] Микро-петля / Netflix-крючок в начале?
- [ ] 2-4 интерактива (разные формулировки)?
- [ ] Минимум 1 кейс или факт (proof)?
- [ ] Разрыв без интерактива ≤5 мин (≤6 слайдов)?
- [ ] Переход к следующему блоку в notes последнего слайда?
- [ ] img_tag не повторяется на соседних слайдах?

### 8.3 Чеклист всего веба

**Структура:**
- [ ] Первые слайды = программа веба (не боль)?
- [ ] Петля открыта в начале, закрыта в конце?
- [ ] Netflix-крючок на каждом переходе между блоками?
- [ ] Смена типа слайда каждые 10-15 мин?

**Речь:**
- [ ] ВСЕ notes прочитаны вслух — нигде не запнулся?
- [ ] Нет телеграфного стиля НИ В ОДНОМ слайде?
- [ ] Интерактивы не повторяются (варьируются)?
- [ ] Связки между слайдами — notes подводят к следующему?

**Визуал:**
- [ ] img_tag проставлены осмысленно?
- [ ] Нет повтора одной картинки на 3+ слайдах подряд?
- [ ] Специальные изображения (фото, лицензии) используются в нужных местах?

**TSV:**
- [ ] Заголовки колонок корректны?
- [ ] Переносы через ` · `, а не `\n`?
- [ ] Нумерация сквозная?
- [ ] Разделы заполнены корректно?
