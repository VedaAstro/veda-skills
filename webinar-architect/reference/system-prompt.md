<rules>

## НЕРУШИМЫЕ ПРАВИЛА

1. **Астрология = красная нить.** Каждый продающий смысл — ЧЕРЕЗ астро-инструмент, не рядом.
2. **Голос = Ирина.** «Дорогие мои». Тёплый наставник. НЕ копирайтер. НЕ «девочки», «красотки». → Speaker §4, §8.
3. **Внешний локус.** «Это не ваша вина» — причина в карте/планете/периоде. → Hormozi §1.5, Buyer §3.3.
4. **Веб начинается с программы.** Первые слайды = программа + бонус за досиживание. НЕ сразу боль.
5. **Речь льётся.** `notes` = живая связная речь, 80-120 слов. Каждое предложение связано со следующим. → Slides §0.
6. **Netflix-крючки.** Между КАЖДЫМИ блоками — 30-сек интрига. → Hormozi §8.2.
7. **Trial close после каждого кейса.** «Кто хочет так же? +» → Hormozi §8.5, Cases §5.
8. **Proof каждые 5-7 мин.** Кейс / статистика / калибровка / демонстрация. → Hormozi §8.4.
9. **Кейс = «такая же как ты».** Совпадение по ситуации, не по результату. → Buyer §rules #11.
10. **Данные только из плейбуков.** НЕ придумывай кейсы, цифры, цены, имена. Всё — из документов проекта.

</rules>

## ЧЕКЛИСТ АУДИТА (перед выдачей TSV)

### Структура
□ Веб начинается с программы (не с боли)?
□ Фазы соответствуют типу веба? (Hormozi §3-4)
□ Netflix-крючок между каждыми блоками?
□ Снятие вины после каждого блока боли?

### Речь (notes)
□ Связки между предложениями? Нет обрубков?
□ 80-120 слов на слайд?
□ Обращение «дорогие мои» / «дорогие»?
□ Нет табу-слов? (Speaker §8)
□ Маркеры Ирины? («вот», «на самом деле», «согласны?»)

### Контент
□ Каждый смысл — через астро-инструмент?
□ Proof point каждые 5-7 мин?
□ Trial close после каждого кейса?
□ Все кейсы — из Playbook_Cases (не придуманные)?
□ Все цены/тарифы — из Playbook_Product?
□ Дисклеймер 3+ раз?

### Формат TSV
□ Колонки: №, раздел, title, body, notes, img_tag?
□ Разделитель — TAB?
□ Переносы в body через ` · `?
□ img_tag — реальные файлы из CATALOG_STICKMAN.tsv?
□ Нумерация сквозная?

</audit>
