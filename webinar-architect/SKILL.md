# Архитектор вебинаров

## Когда использовать

- Создание вебинара с нуля (прогрев / оффер / дожим)
- Написание скрипта / речи спикера
- Создание TSV-таблицы слайдов
- Доработка / аудит существующего веба
- Выбор упражнений, крючков, пруфов для вебинара
- Создание оффера / value stack

## Ключевые правила (10 нерушимых)

1. **Астрология = красная нить.** Каждый смысл -- ЧЕРЕЗ астро-инструмент, не рядом
2. **Голос = Ирина.** "Дорогие мои". Теплый наставник. НЕ копирайтер. НЕ "девочки/красотки"
3. **Внешний локус.** "Это не ваша вина" -- причина в карте/планете/периоде
4. **Веб начинается с программы.** Первые слайды = программа + бонус за досиживание
5. **Речь льется.** notes = живая связная речь, 80-120 слов. Нет обрубков
6. **Netflix-крючки.** Между КАЖДЫМИ блоками -- 30-сек интрига
7. **Trial close после каждого кейса.** "Кто хочет так же? +"
8. **Proof каждые 5-7 мин.** Кейс / статистика / калибровка / демонстрация
9. **Кейс = "такая же как ты".** Совпадение по ситуации, не по результату
10. **Данные только из плейбуков.** НЕ придумывать кейсы, цифры, цены, имена

## Чеклист аудита (из system-prompt.md)

### Структура
- Веб начинается с программы (не с боли)?
- Фазы соответствуют типу веба?
- Netflix-крючок между каждыми блоками?
- Снятие вины после каждого блока боли?

### Речь (notes)
- Связки между предложениями? Нет обрубков?
- 80-120 слов на слайд?
- Обращение "дорогие мои"?
- Нет табу-слов?
- Маркеры Ирины? ("вот", "на самом деле", "согласны?")

### Контент
- Каждый смысл -- через астро-инструмент?
- Proof point каждые 5-7 мин?
- Trial close после каждого кейса?
- Все кейсы из плейбуков (не придуманные)?
- Все цены/тарифы из плейбуков?

### Формат TSV
- Колонки: No, раздел, title, body, notes, img_tag?
- Разделитель -- TAB?
- Переносы в body через " . "?
- img_tag -- реальные файлы из каталога?
