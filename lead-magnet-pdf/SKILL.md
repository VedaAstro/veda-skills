# Лид-магнит PDF

## Когда использовать

- Создание бонусного PDF для вебинара (календарь, шпаргалка, чек-лист)
- Лид-магнит для воронки (гайд, мини-книга, рабочая тетрадь)
- PDF-отчёт для бота (диагностика, натальная карта, разбор)
- Любой брендированный PDF-документ Vedantica

## Ключевые правила

### 1. PDF = продукт, не документ
Каждый PDF решает ДВЕ задачи:
1. **Ценность** — зритель получает реальный инструмент (даты, таблицу, чек-лист)
2. **Gap** — PDF показывает, что есть БОЛЬШЕ, и создаёт желание углубиться

PDF, который только даёт ценность — обесценивает программу.
PDF, который только продаёт — не работает как лид-магнит.
Баланс: 80% ценность, 20% gap-блок в конце.

### 2. Дизайн-система (DESIGN.md)
Все PDF используют единую дизайн-систему Vedantica.
Дизайн-файл: `reference/DESIGN.md`

### 3. Астро-ассеты обязательны
Не рисовать планеты, знаки, дома средствами reportlab.
Использовать готовые WebP/PNG из продакшн-директорий.

### 4. Шрифты с кириллицей
Только проверенные шрифты. Перед генерацией — тест рендеринга кириллицы.
Рабочие шрифты macOS:
- `/System/Library/Fonts/Supplemental/Arial.ttf` (Ar)
- `/System/Library/Fonts/Supplemental/Arial Bold.ttf` (ArB)
- `/System/Library/Fonts/Supplemental/Arial Italic.ttf` (ArI)

Canvas-design шрифты с кириллицей (проверено):
- `Jura-Light.ttf` — геометрический, для заголовков
- `ArsenalSC-Regular.ttf` — для body text
- `PoiretOne-Regular.ttf` — декоративный, для акцентов

### 5. Конвертация ассетов для reportlab
WebP не поддерживается reportlab напрямую. Конвертация:
```python
from PIL import Image
img = Image.open('planet.webp').convert('RGBA')
img.save('/tmp/planet.png')
# Использовать /tmp/planet.png в drawImage()
```

## Быстрый маршрут

### Бонус к вебинару
1. Прочитай этот скилл
2. Открой `reference/DESIGN.md` — палитра, типографика, компоненты
3. Открой `reference/ASSETS.md` — все доступные изображения с путями
4. Определи структуру: обложка → контент (2-3 стр) → gap-блок
5. Напиши скрипт reportlab с правильными шрифтами
6. Тестовый рендер → проверь кириллицу и изображения
7. Финальный PDF

### Лид-магнит для воронки
1. Определи аудиторию (холод vs LTV)
2. Определи ценность (что получает) и gap (что остаётся за кадром)
3. Структура: обложка → «зачем это нужно» → основной контент → gap → CTA
4. Следуй DESIGN.md

## Структура PDF по типам

### Бонус-календарь (как Рохини)
```
Стр 1: Обложка — название, подзаголовок, «бонус от Ирины»
Стр 2: Объяснение — что это, почему важно, 3 свойства (карточки)
Стр 3: Инструмент — таблица (лагны → дома → сферы)
Стр 4: Календарь + gap-блок
```

### Чек-лист / шпаргалка
```
Стр 1: Обложка
Стр 2-3: Контент с чекбоксами / таблицами
Стр 4: Gap-блок + CTA
```

### Мини-гайд (5-8 страниц)
```
Стр 1: Обложка
Стр 2: Оглавление / «для кого»
Стр 3-6: Контент с иллюстрациями (планеты, знаки)
Стр 7: Резюме / чек-лист
Стр 8: Gap-блок + CTA
```

## Чеклист перед сдачей

### Дизайн
- [ ] Палитра из DESIGN.md? (indigo, gold, ivory — ничего другого)
- [ ] Шрифты с кириллицей проверены тестовым рендером?
- [ ] Изображения из ASSETS.md? (не нарисованные, не emoji-заглушки)
- [ ] Единый стиль на всех страницах?

### Контент
- [ ] PDF даёт реальную ценность (не пустышка)?
- [ ] Gap-блок в конце создаёт желание, не давление?
- [ ] CTA = «кнопка на странице эфира» (не «ссылка в чате»)?
- [ ] Данные из плейбуков (не придуманные)?
- [ ] Нет «15 лет», нет «НейроАстролог» (если не в этом запуске)?

### Технический
- [ ] PDF открывается на телефоне?
- [ ] Размер < 5MB?
- [ ] Все страницы A4?
- [ ] Футер на каждой странице?
- [ ] Нет обрезанного текста на краях?
