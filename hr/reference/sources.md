# Sources

## Основной источник: книга «Кто»

**Geoff Smart, Randy Street. «Who: The A Method for Hiring»**
Русский перевод: «Кто. Решите вашу проблему номер один»
Локальный файл: `/Users/alex/Documents/КНИГИ/Кто. Решите вашу проблему номер один.rtf`

### Что взято из книги в этот skill

**A-метод в целом:**
- Определение A-игрока: 90% вероятность достичь результатов лучших 10% кандидатов
- Два параметра: квалификация + воля
- Цена ошибки найма: 15 месячных окладов сотрудника

**Лист целей (Scorecard):**
- 4 части: задача + ожидаемые результаты + профессиональные качества + культурное соответствие
- Приоритет измеримых outcomes над списком обязанностей
- Список базовых компетенций A-игрока

**Sourcing:**
- 5 каналов по приоритету
- Вопрос Патрика Райана для нетворкинга
- Принцип 30 мин в неделю на прозвон
- Система таблицы кандидатов

**4 типа интервью — дословные вопросы:**
1. Отборочное (30 мин, 4 вопроса) — все вопросы дословно
2. Топгрейдинг (3 часа, хронологическое, 5 вопросов по каждой должности) — все вопросы дословно
3. Фокус-интервью (45-60 мин, 3 вопроса) — все вопросы дословно
4. Reference checks (7 звонков, 5 вопросов) — все вопросы дословно

**Техники интервью:**
- Уточняющие зонды: «что / как / расскажите подробнее»
- Техника УПР (угроза проверить рекомендации)
- Три «П» (предыдущий период / план / партнёры)
- Тяни-толкай (вытолкнут / притянут)
- Чтение сигналов рекомендателей (язык кода)

**Closing — 5 рычагов (5 «П»):**
- Привязка (Fit)
- Переживания близких (Family)
- Предоставленная свобода (Freedom)
- Процветание (Fortune)
- Получение удовольствия (Fun)

**Ошибки найма:**
- 10 масок плохого нанимателя
- Системные ошибки (sourcing, интервью, closing, onboarding)
- Red flags кандидата и системы

## Сессия стратегии 2026-04-04 (найм Growth-маркетолога Vedantica)

Живая сессия с owner, зафиксированы уроки которых не было в книжных источниках.

**Файл-артефакт:** `/Users/alex/Projects/platform/specs/ROADMAP-marketer-hiring-2026-04-04.md`

### Что взято из сессии в skill

**KPI-дизайн:**
- Не более 2-3 KPI (перегруженная мотивация не считается)
- Northstar в рублях, не в штуках (защита от демпинга)
- Northstar + guardrail (защита от выжигания актива)
- Привязка только к контролируемой метрике (не к выручке, если C₂b у другой роли)
- Разделение ответственности по границе «заказ → оплата»

**Анти-паттерны killer-questions:**
- «Какой AI используешь» — все скажут ChatGPT, сигнала ноль
- «К чему стремишься в карьере» — заученный ответ
- Правильные вопросы — через датасеты и сценарии, проверяющие мышление

**AI-воронка (новый трек):**
- 4 этапа: Telegram-бот → тестовое-датасет → AI-видеоинтервью → живое интервью
- Свои инструменты вместо TestGorilla/Hireflix (Claude Agent SDK + веб-форма + Whisper)
- Живое интервью — только финалисты (2-3 из 100)
- Первые 20 кандидатов owner калибрует AI вручную

**Тестовое задание (правильный формат):**
- Не «напиши план» — даём датасет с противоречиями и ограничениями
- Задача на выбор + обоснование, не на генерацию списка идей
- Проверяет: приоритизацию, причинно-следственные связи, data literacy, осознание ограничений
- Антипаттерн: «3 гипотезы и как их проверить» — банально, не фильтрует

**Обязательные активности:**
- Кастдевы как ритуал для маркетолога (5-8 в неделю), не разово
- Без кастдевов = mis-hire даже если цифры в плане

**Нейминг ролей:**
- Избегать «оператор» — звучит как машинист
- Брать названия из рыночного лексикона (как ищут сами кандидаты)
- Использовать результат market research, не фантазию

**Правильный формат roadmap документа:**
- См. `memory/feedback_strategic_cycle.md`
- §0 Текущий статус (обновляется каждой сессией)
- §1 Факты верифицированные (с источником и датой)
- §N Handoff-ы для каждой фазы
