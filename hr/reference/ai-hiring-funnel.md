# AI Hiring Funnel — 95% автоотсев

> Третий трек найма (в дополнение к EXPRESS и FULL из hiring-playbook.md).
> Используется для AI-first ролей: Growth-маркетолог, Data-аналитик, Product-менеджер, AI-инженер.
> Основан на сессии стратегии 2026-04-04. Референс-реализация: `/Users/alex/Projects/platform/specs/ROADMAP-marketer-hiring-2026-04-04.md`

## Зачем этот трек

Классический EXPRESS/FULL не подходит когда:
- Роль требует AI-native мышления (нужно проверить глубину работы с AI, а не просто "пользуется ChatGPT")
- У owner ограниченное время на интервью (≤ 5 часов на всех кандидатов)
- Нужен массовый отсев с сохранением качества сигнала
- Можно построить инструмент скрининга своими руками (Claude API, Telegram-бот, веб-форма)

**Принцип:** до owner доходят только 2-3 финалиста из 100 откликов. Все остальные этапы — автоматизированы через AI.

---

## Воронка из 4 этапов

```
ОТКЛИКИ (100)
  ↓  Этап 1: Telegram-бот (AI-скрининг заявки)
ПРОШЛИ АНКЕТУ (30)                         ← отсев 70%
  ↓  Этап 2: Аналитическое тестовое (задача на мышление)
СДАЛИ ТЕСТОВОЕ (10)                        ← отсев 67%
  ↓  Этап 3: AI-видеоинтервью (6-8 вопросов без подготовки)
ПРОШЛИ ВИДЕО (3-5)                         ← отсев 50-70%
  ↓  Этап 4: Живое интервью с owner
ФИНАЛИСТЫ (1-2)                            ← оффер
```

Каждый этап даёт независимый сигнал, решение принимается AI + owner в точке эскалации.

## Этап 1 — Telegram-бот (вместо анкеты hh.ru)

**Зачем:** собрать структурированные данные + DISC-профиль + ссылки на кейсы + оценить релевантность автоматически.

**Что делает бот:**
1. Приветствие + ссылка на полный текст вакансии
2. Спрашивает 5-7 обязательных вопросов (не поверхностных, см. ниже)
3. Просит прислать ссылки на 2-3 реальных кейса с цифрами
4. DISC-опросник (16 пунктов) встроенный в диалог
5. Claude-скоринг кейсов на релевантность по rubric
6. Прошёл → ссылка на тестовое (Этап 2). Не прошёл → вежливый отказ

**Правильные вопросы (не поверхностные):**
- Для AI-first роли: «Пришли ссылку на твой проект/агент в Claude или GPT, который ты довёл до результата. Объясни в 3 предложениях: вход, что делает, выход.»
- Для data-driven роли: «Опиши кейс, где ты принимал решение против интуиции на основе данных. Какие данные были? К чему пришёл?»
- Для скорости: «Сколько времени занимает у тебя запуск нового email-флоу с нуля? Опиши этапы.»
- Для кастдевов: «Сколько custdev-интервью ты провёл за последние 3 месяца? Что самое неожиданное услышал?»

**Запрещены вопросы типа:**
- «Какой AI используешь?» — все скажут ChatGPT
- «Расскажи о себе» — заученный ответ
- «Почему хочешь у нас работать?» — подготовленный ответ

**Хранение:** таблицы `hr_candidates`, `hr_answers`, `hr_scores` в bot_admin DB (см. ROADMAP референс).

**Инструменты:** Claude Agent SDK + aiogram + bot_admin DB. Не нужны внешние ATS (Talantix, Huntflow) — свой бот гибче.

## Этап 2 — Аналитическое тестовое (задача на мышление)

**Принцип:** тестовое проверяет КАК думает кандидат, а не ЧТО он знает. Формат — задача с данными и противоречиями.

### Антипаттерн

Мусорное тестовое: «Напиши план email-маркетинга для Vedantica на 3 месяца. Какие 3 гипотезы проверишь?»

Почему мусор:
- Предсказуемо — все напишут одно и то же (персонализация, A/B, реактивация)
- Не проверяет приоритизацию
- Не проверяет умение читать данные
- Не проверяет осознание ограничений

### Правильный формат

**Даём датасет** с противоречиями и ограничениями. Спрашиваем **выбор + обоснование**.

Пример для Growth-маркетолога:

```
ДАННЫЕ (таблица сегментов базы):
| Сегмент | Размер | OR | CTR | CR→заказ | Ср.чек | Последний запуск |
|---|---|---|---|---|---|---|
| Купили A, не купили B | 1200 | 22% | 4.1% | 2.8% | 88К | 4 мес назад |
| Регистрация без покупки | 18000 | 8% | 1.2% | 0.3% | 42К | 2 нед назад |
| Купили 1 курс <30К | 2400 | 18% | 3.5% | 1.9% | 35К | 6 мес назад |
| Только TG, нет email | 8500 | — | 2.8% | 0.5% | 50К | 1 мес назад |
| Ученики 2023 года | 900 | 25% | 5.2% | 0.4% | 62К | 8 мес назад |

ОГРАНИЧЕНИЯ:
- Бюджет: 0 руб. (только текущие каналы)
- Время: 2 недели (один спринт)
- Есть: Claude, GetCourse, email, TG-бот, менеджер на обзвон (15 звонков/день × 2 недели = 150)
- Цель: максимум квалифицированных заказов

ЗАДАНИЕ:
1. Выбери 2 сегмента для работы. Объясни почему именно эти, а не другие.
2. Для каждого: что запускаешь, в каком порядке, с каким прогнозом (цифры).
3. Какие данные тебе НЕ хватает? Что бы ты запросил?
4. Как распределишь 150 звонков менеджера между сегментами?
```

### Что это выявляет

| Критерий | Хороший ответ | Слабый ответ |
|---|---|---|
| Приоритизация | Выбирает по expected value (размер × CR × чек) | Выбирает "потому что интересно" |
| Причинно-следственные связи | Замечает противоречие: "ученики 2023" — высокий OR, низкий CR — 8 мес молчания, оффер нерелевантен | Игнорирует аномалии |
| Data literacy | Считает прогноз в цифрах: 1200 × 22% × 4.1% × 2.8% = ~3 заказа | Даёт только общие слова |
| Осознание ограничений | Спрашивает недостающие данные (какой был оффер, когда последний контакт) | Не задаёт уточняющих вопросов |
| Ресурсное мышление | Распределяет 150 звонков по expected value | Распределяет поровну или по размеру |

### Rubric AI-скоринга

```
ПРОМПТ ДЛЯ CLAUDE:
Ты — эксперт по найму growth-маркетологов. Оцени ответ кандидата на тестовое задание.

Датасет: [датасет]
Задание: [задание]
Ответ кандидата: [ответ]

Оцени по 5-балльной шкале по каждому критерию:
1. Приоритизация (выбирает по expected value)
2. Причинно-следственные связи (замечает аномалии)
3. Data literacy (считает в цифрах)
4. Осознание ограничений (задаёт уточняющие вопросы)
5. Ресурсное мышление (распределяет 150 звонков осмысленно)

ИТОГ: средний балл.
Порог: ≥ 3.5 из 5 = проход на Этап 3.

Формат: JSON.
```

### Таймбокс

60 минут максимум. Дедлайн — 48 часов с момента отправки. Кто не сдал = автоотказ (это сам по себе фильтр, 30-40% отвалятся).

### Антипаттерн оформления

**Не тратить кандидата** на написание документации, красивых слайдов, презентаций. Только суть — текстовый ответ в Google Doc. Оформление не оценивается.

## Этап 3 — AI-видеоинтервью (6-8 вопросов без подготовки)

**Зачем:** отсечь тех, кто "красиво пишет", но медленно думает. На живой камере не подготовишься к неизвестным вопросам.

### Архитектура

```
Кандидат получает персональную ссылку → /hr/interview/:token
  ↓
Веб-страница: включает камеру, видит вопрос #1
  ↓
MediaRecorder API записывает ответ (таймер 2 минуты)
  ↓
После ответа — следующий вопрос (6-8 штук, кандидат не знает сколько)
  ↓
Видео загружаются на S3 (Timeweb bucket)
  ↓
Whisper транскрипция на сервере
  ↓
Claude анализ по rubric
  ↓
Сводка + оценка в recruiter cockpit
```

**Своими руками**, не сервисами (Hireflix, VCV). Причина: интегрируется с нашим botadmin, хранится у нас, можно дорабатывать rubric.

### Вопросы (пример для маркетолога)

1. **Проект** — «Покажи мне последний проект или агент, который ты построил в Claude или GPT. Что было на входе, что на выходе?»
2. **Диагностика** — «OR email-базы упал с 18% до 9% за 3 месяца. Назови 3 возможные причины и что проверишь первым.»
3. **Действие** — «Ты видишь: сегмент 1200 человек с чеком 88К не получал писем 4 месяца. Что сделаешь в первые 48 часов?»
4. **Культурный фит** — «Что тебе интересно в теме [ниша компании]? Если ничего — скажи честно, это не дисквалифицирует.»
5. **Рабочий ритм** — «Опиши свой идеальный рабочий день: что делаешь, сколько, когда останавливаешься.»
6. **Приоритизация** — «У тебя 3 срочные задачи одновременно. Как выбираешь что делать первым?»
7. **Кастдевы** — «Как часто ты общаешься с живыми клиентами? Что обычно спрашиваешь?»
8. **Самокритика** — «Какой твой последний проваленный эксперимент? Что понял?»

### Rubric видеоинтервью

| Критерий | 1 балл | 3 балла | 5 баллов |
|---|---|---|---|
| Конкретика | Общие слова, абстракции | Есть примеры | Цифры, даты, имена, факты |
| Скорость мысли | Долгие паузы, путается | Думает 5-10 сек → отвечает | Отвечает сразу по существу |
| Глубина | Поверхностный ответ | Есть структура | Несколько уровней причинно-следственных связей |
| Уверенность | Много "ну это", "наверное" | Спокойно отвечает | Говорит как эксперт |
| Релевантность | Ушёл от темы | Частично ответил | Попал точно |

### Промпт AI-анализа

```
Ты — эксперт по найму. Транскрипт видеоинтервью кандидата ниже.

Вопрос: [вопрос]
Ответ кандидата: [транскрипт]

Оцени по 5 критериям (1-5): конкретика, скорость мысли, глубина, уверенность, релевантность.
Укажи 2 сильных сигнала и 2 слабых сигнала.
Итоговый балл: среднее.

Формат: JSON.
```

## Этап 4 — Живое интервью с owner (только финалисты)

**До этого этапа доходят только 2-3 кандидата из 100 откликов.** Owner получает:
- Все ответы на этапе 1
- Тестовое + AI-оценка
- Транскрипты + видео этапа 3
- AI-саммари: сильные стороны, слабые, red flags

**Формат интервью:** 30 минут, 5 вопросов с rubric, подтверждение chemistry и culture fit.

Живое интервью — **не про проверку компетенций** (уже проверены), а про:
1. Химию с owner
2. Культурный фит в команде
3. Ответы на вопросы кандидата (closing)
4. Озвучивание оффера и условий

### 5 вопросов для финального интервью

1. «Расскажи про последний провал в работе. Что именно пошло не так? Что понял?»
2. «Что тебе сейчас важно в следующей работе? В каком порядке приоритетов?»
3. «Был ли у тебя опыт, когда руководитель принимал решение, с которым ты не был согласен? Что сделал?»
4. «Если бы ты пришёл к нам и через 90 дней не выполнил план — что это, по-твоему, означало бы?»
5. «У тебя остались вопросы к нам?»

### Rubric финала

| Критерий | Стоп-сигналы | Зелёные сигналы |
|---|---|---|
| Провалы | "Не помню", "всё было хорошо" | Называет конкретный, видит свою роль |
| Приоритеты | Деньги первыми или только деньги | Задача, команда, рост — деньги во 2-3 очереди |
| Работа с руководителем | "Всегда соглашался" или "уходил" | Говорил напрямую, обосновывал |
| Взгляд на провал | Обвиняет обстоятельства | Берёт ответственность, даёт план |
| Вопросы к нам | Нет вопросов или только про ЗП | Вопросы про продукт, команду, roadmap |

## Антипаттерны AI-воронки

1. **Внешние сервисы** (TestGorilla, Hireflix, Pymetrics) — дороже, хуже интеграция, данные не наши
2. **Живое интервью как 2-й шаг** — owner закопается в 30+ часах разговоров
3. **Сложные статические формы** вместо Telegram-бота — конверсия ниже, нет DISC
4. **Тестовое "напиши стратегию"** — банально, не фильтрует
5. **Вопросы из книги "Кто" в видеоинтервью** — работают для живого, на камере слишком жёсткие
6. **Запуск без калибровки AI** — первые 20 кандидатов owner смотрит вручную и сравнивает с AI

## Когда НЕ использовать AI-воронку

- Для ролей МП и РОП — EXPRESS и FULL треки работают лучше (см. hiring-playbook.md)
- Если нет времени на построение инструмента (2-3 сессии разработки)
- Для массовых разовых задач (например, seasonal saleman)
- Для очень узких экспертов где рынок маленький — лучше headhunting через нетворкинг

## HH-приглашение: правила текста

### Правильный шаблон (бот: @hr_alexbot)
```
Здравствуйте! Посмотрели ваш профиль — интересный опыт.

Приглашаем на первый этап: знакомство с нашим AI-агентом.
Займёт ~10-15 минут, отвечать можно в удобном темпе.

Переходите: https://t.me/hr_alexbot?start=hh_{neg_id}
```

### Правила копирайтинга
- **Всегда приветствие** — без него выглядит как спам
- **"AI-агент"**, не "Telegram-бот" — не обесценивать инструмент
- **Принцип Зейгарник** — не раскрывать всё сразу (не говорить про "аналитическую задачу", "скрининг", этапы). Лёгкий вход → потом внутри бот раскрывает следующие шаги
- **Не "скрининг"** — слово создаёт ощущение отбора/проверки, снижает конверсию в переход

### Антипаттерны
- ❌ `короткий скрининг через Telegram-бота`
- ❌ `задаст вопросы и предложит аналитическую задачу`
- ❌ отсутствие приветствия
- ❌ неверный username бота

## Решения об отказе

### Кто принимает решения
**Всегда Алекс**, не автоматически:
- A/B кандидаты (высокий/средний скоринг) → уведомление с полным scorecard + кнопки `✅ Отправить тестовое | ❌ Не подходит`
- C кандидаты (низкий скоринг, FAIL) → авто-отказ кандидату + короткое уведомление Алексу
- Pre-screen C → авто-отказ через HH API (до входа в бот)

### HH rejection API (правильная последовательность)
```python
# НЕПРАВИЛЬНО — вернёт 403:
PUT /negotiations/discard/{neg_id}
PUT /negotiations/consider/{neg_id}

# ПРАВИЛЬНО — сначала перевести в phone_interview, потом discard:
PUT /negotiations/phone_interview/{neg_id}
PUT /negotiations/discard_by_employer/{neg_id}  # body: {"message": "текст отказа"}
```

## Технические уроки (бот @hr_alexbot)

### Критичные баги и фиксы

**Баг: null content от OpenRouter → crash**
- Симптом: `AttributeError: 'NoneType' has no attribute 'strip'` → бот отвечает "технические неполадки"
- Причина: Claude Sonnet через OpenRouter иногда возвращает `content: null` с `finish_reason: stop`
- Фикс: `_call_agent_text` ретраит 3 раза с exponential backoff; `_openrouter_call` логирует и бросает RuntimeError

**Баг: кандидат перезашёл по /start не ответив на последний вопрос**
- Симптом: history заканчивается на сообщении агента → передаём Claude conversation `[user, assistant, user, assistant]` → null content
- Фикс: если `messages[-1].role == "agent"` → сразу вернуть последний вопрос без API-вызова

**Баг: неверный username бота в приглашениях**
- Правильный: `@hr_alexbot` (не `@veda_hr_bot`)
- Константа `BOT_USERNAME = "hr_alexbot"` в `/root/hr-bot/bot.py`
