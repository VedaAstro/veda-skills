# HR — Найм

## Когда использовать

- Найм, подбор, интервью, оценка кандидатов
- Лист целей / scorecard для новой роли
- Тестовое задание или work sample
- Компенсации и оффер
- Вакансии на hh.ru / Telegram
- Onboarding, 30/60/90
- Разбор, почему найм сломался

## Три трека

| | EXPRESS (МП, таргетолог) | FULL (РОП, Head) | AI-FUNNEL (Growth, Data, AI-first) |
|---|---|---|---|
| Лист целей | Да | Да | Да + профиль AI-глубины |
| Этап 1 | Скрининг 15 мин, 4 вопроса | Скрининг 20-30 мин, 4 вопроса | Telegram-бот: анкета + DISC + кейсы |
| Этап 2 | Тестовое (work sample) | Тестовое (case discussion) | Тестовое с датасетом, AI-скоринг |
| Этап 3 | — | Глубокое интервью 60-90 мин | AI-видеоинтервью 6-8 вопросов, Whisper+Claude анализ |
| Этап 4 | Reference checks (2) | Reference checks (3-4) | Живое интервью с owner (только 2-3 финалиста) |
| Срок | 5-7 дней | 10-14 дней | 10-14 дней (параллельно пока строим инструмент) |

**Когда какой трек:**
- EXPRESS — линейные роли с понятным выхлопом (МП, таргетолог, SMM)
- FULL — ключевые роли с командной ответственностью (РОП, Head)
- AI-FUNNEL — роли где критична AI-глубина и нет времени owner на 30+ часов интервью (Growth-маркетолог, Data-аналитик, AI-инженер)

## Быстрый маршрут

1. Составь **Лист целей** с проверяемыми компетенциями → `hiring-playbook.md §2`
2. Опубликуй на **hh.ru + 5 Telegram-каналов** → `vacancy-templates.md` + `hiring-playbook.md §3`
3. **Скрининг** 15-20 мин по 4 вопросам → `§4`
4. **Тестовое** с rubric → `work-samples.md`
5. Для FULL трека: **глубокое интервью** → `§5`
6. **Reference checks** → `§7`
7. **Close** через задачу + деньги + свободу → `§8`
8. **Компенсация** → `compensation-models.md`
9. **Onboarding** → `§10`

## Ключевые правила

- Нет Листа целей → нет найма
- Подход «красного света»: ищи причину отказать, не причину взять
- Тестовое обязательно для ВСЕХ ролей, до собеседования
- Не озвучивай ЗП по телефону
- Правило «2 на 1»: 2 человека на ИС на 1 вакансию
- Презентация вакансии ПОСЛЕ выявления мотива кандидата
- **KPI в рублях, не в штуках** — защита от демпинга чеком. Для маркетолога: не "50 заказов", а "объём заказов ₽"
- **Northstar + guardrails** — только Northstar = выжигание актива за 3 месяца. Для маркетолога обязателен guardrail "активная база не убывает"
- **Не более 3 KPI** — перегруженная мотивация не считается и не управляется
- **Разделение маркетинг / ОП / ОД по границе "заказ → оплата"** — маркетолог не отвечает за C₂b (см. profit-formula-methodology.md)
- **Не гнать быстро на интервью** — сначала документы, кейсы, тестовое, AI-анализ видео. Живое интервью только с финалистами (2-3 человека)
- **Свои инструменты вместо внешних сервисов** — TestGorilla/Hireflix не нужны, если можно построить бот на Claude SDK и свой recruiter cockpit

## Анти-паттерны killer-questions

Поверхностные killer-questions не фильтруют ничего — только собирают «правильные» ответы, которые знают все. Примеры мусорных вопросов:

- «Какой AI-инструмент используешь?» → все скажут «ChatGPT», не даст сигнала
- «Расскажи про свой опыт в маркетинге» → заученные формулировки
- «Почему ты хочешь у нас работать?» → подготовленный ответ

**Правильные killer-questions проверяют мышление, а не знания:**

- Дать реальный датасет с противоречиями → «Какие 2 сегмента выберешь для работы и почему? Каких данных не хватает?»
- Показать плохой текст → «Что в нём не так?»
- Задать сценарий с ограничениями → «Как распределишь 150 звонков менеджера между 5 сегментами?»
- Попросить кейс с цифрами до/после → «Что именно ты сделал? Какая была гипотеза?»

**Видеоинтервью без подготовки** — вопросы не известны заранее, кандидат отвечает на камеру в реалтайме, 2 минуты на ответ. Whisper → Claude анализ. Фильтрует тех, кто «красиво пишет», но не думает быстро.

## Обязательные активности для маркетолога

- **Кастдевы как ритуал** — 5-8 глубинных интервью в неделю. Не «провели исследование раз в квартал», а системная практика. Без этого маркетолог работает вслепую, офферы наугад.
- Еженедельный инсайт-дайджест для owner (3-5 наблюдений из кастдевов)
- Применение инсайтов в следующем спринте

Это не KPI, это **операционная норма роли**. Отсутствие кастдевов = кандидат не подходит.

## Сценарий живого собеседования (AI-FUNNEL трек, финал)

Живое интервью — только с 2-3 финалистами после AI-скрининга + тестового + видеоинтервью.

### Как запросить план через HR-бот

Reply на уведомление о кандидате (или пока он активен как «последний») и напиши любое из:
`план`, `сценарий`, `план собеса`, `вопросы`, `подготовь вопросы`

Бот сгенерирует план на основе: резюме + прескрининг + briefing AI-скрининга + результаты тестового.

### Структура плана (35-45 мин)

| Блок | Время | Содержание |
|------|-------|-----------|
| Вход | 3 мин | Контекст: «это финал, задам конкретные вопросы по кейсам» |
| Топгрейдинг | 10 мин | 2 последних места: цели → результат → что не получилось → почему ушёл |
| Зоны приоритет 1 | 15 мин | 3-4 поведенческих вопроса по слабым/неясным зонам |
| Кастомные вопросы | 5 мин | По данным скрининга: пробелы и громкие claims |
| Закрытие | 2 мин | «Что скажет руководитель?» + вопросы кандидата |

### Best practices (методология «Кто», топгрейдинг)

**Поведенческий фрейм STAR:**
- Situation → Task → Action → Result
- Каждый вопрос: «Расскажи конкретный случай когда...» / «Дай пример когда...»
- Не принимать обобщения «я обычно делаю» — требуй конкретный кейс

**Топгрейдинг для каждого места работы:**
1. Что привлекло в компанию?
2. Каковы были твои цели / KPI?
3. Что достиг (цифры)?
4. Что не получилось?
5. Почему ушёл?
6. Что скажет о тебе руководитель, если я позвоню?

**Drill-down на пассивный залог и громкие claims:**
- «Мы делали» → «Что ИМЕННО ты делал?»
- «Увеличил конверсию в 3 раза» → «Как считал? Какой был baseline? За какой период?»
- Результат без цифр = не засчитывается

**Ключевые вопросы по зонам (Growth-маркетолог):**

base_monetization:
- «Расскажи про базу, с которой работал. Какие OR/CTR, и что ты делал чтобы их поднять?»
- «Опиши конкретную рассылку с лучшим ROI — что тестировал, какой результат?»

ai_native:
- «Какие AI-инструменты используешь прямо сейчас? Покажи конкретный результат»
- «Опиши агента или автоматизацию, которую ты построил — вход, выход, сколько времени сэкономил?»

analytical_thinking:
- «Последняя воронка, которую ты собирал с нуля — где был главный слив и как нашёл?»
- «Как находишь, где воронка дырявая? Покажи на реальном примере с цифрами»

custdev:
- «Последний разговор с клиентом — что узнал и что изменил в маркетинге после этого?»
- «Какой ритм глубинок у тебя был? Как превращал инсайты в конкретные действия?»

track_record:
- «Назови 3 измеримых результата с последнего места. Верифицируемые?»
- «Что из этого можно проверить — есть ссылки, скрины, могу позвонить?»

**Антипаттерны — не использовать:**
- «Каковы ваши слабые стороны?» — шаблонный ответ
- «Почему хотите у нас работать?» — провоцирует лесть
- «Где видите себя через 5 лет?» — нет связи с задачей
- Halo effect: один яркий ответ не закрывает пробелы в остальном
- Говорить о компании > 30% времени: кандидат должен говорить 70%
- Разные вопросы разным финалистам: нет валидного сравнения

## Оффер senior-кандидату после живого собеса

Если финалист senior-уровня и есть нестыковки с вакансией (совмещение, цена, формат) — НЕ давай готовый оффер. Делегируй постановку.

### Антипаттерны

- Предлагать финальные KPI на первой встрече — owner не знает реального состояния системы лучше кандидата после 3 дней аудита
- Давить форматом (full/part) сразу — решение должно быть на данных, не на интуиции
- Повышать ФОТ выше запроса + снижать ответственность одновременно — худший микс
- Переименовывать роль «консультант» вместо Growth-маркетолога — снимает ответственность без компенсации

### Правильная структура

1. **Аудит (3-4 дня):** даёшь доступы, платишь фикс 15К независимо от исхода, просишь 4 артефакта: аудит внутрянки + план 90 дней + честная оценка формата + её KPI/SLA предложение
2. **Стратегическая встреча:** она приходит с предложением, ты — со своим, договариваются
3. **2 якоря вилки:** нижний (part-time + совмещение) и верхний (full-time по вакансии). Кандидат выбирает ПОСЛЕ погружения, не до.
