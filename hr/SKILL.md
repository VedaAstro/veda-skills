# HR — Найм A-игроков

## Когда использовать

- Найм, подбор, интервью, оценка кандидатов
- Формулировка роли, scorecard / лист целей
- Проектирование тестового задания или work sample
- Построение воронки отбора
- Reference checks, финальное решение
- Компенсационные модели и оффер
- Написание вакансий на hh.ru
- Адаптация, 30/60/90, вывод на KPI
- Разбор, почему найм сломался

## Главный принцип

**A-игрок** — кандидат, с вероятностью 90% способный достичь результатов, доступных лишь 10% из всех кандидатов на эту роль. Нанимай только A-игроков.

Цепочка найма (не пропускать этапы):

```
ЛИСТ ЦЕЛЕЙ → SOURCING → 4 ИНТЕРВЬЮ → CLOSING → ONBOARDING → KPI RAMP
```

## Быстрый маршрут

1. Составь **Лист целей** → `reference/hiring-playbook.md §1`
2. Запусти **sourcing** (нетворкинг прежде всего) → `§2`
3. Проведи **4 интервью** по протоколу → `§3-6`
4. Сделай **reference checks** → `§7`
5. Закрой кандидата через **5 рычагов** → `§8`
6. Выставь компенсацию по **рыночной модели** → `reference/compensation-models.md`
7. Если нужна вакансия на hh.ru → `reference/vacancy-templates.md`
8. Запусти **onboarding 30/60/90** → `§9`

## Ключевые правила

- Нет Листа целей → нет найма. Всегда сначала.
- Оценивай только job-related сигналы. Не "химию", не "энергетику".
- Интервью №2 (Топгрейдинг) — хронологически, от первой должности к последней.
- Красный флаг на любом этапе = немедленная оценка, не откладывать.
- Рекомендателей выбираешь ты, не кандидат.
- Closing начинается с первого контакта, а не с оффера.
- Цена ошибки найма: 15 месячных окладов этого сотрудника.
