# Value Map — экраны app.myveda.ru

> Для каждого экрана: зачем человек здесь, что он должен получить, как измерить успех.
> Используй при проектировании или оценке любого экрана.

## Навигационные экраны

| Экран | Задача пользователя | Главная ценность | Метрика успеха |
|-------|---------------------|------------------|----------------|
| `/dashboard` | Понять "что нового" после входа | Точка входа → быстрый доступ к главному | Клик на карту или создание новой < 5 сек |
| `/chart` (список) | Найти нужную карту или создать новую | Обзор всех карт одним взглядом | Время до клика на карту < 3 сек |
| `/chart/new` | Создать карту = получить "ключ" к себе | Минимум трения при вводе данных | Конверсия: начал → создал > 90% |
| `/consultation` | Найти или начать консультацию | Связь с экспертом/AI | Повторные консультации |
| `/profile` | Управление аккаунтом | Контроль над данными | Частота посещения ≈ 0 (хороший знак) |

## Карта — ядро приложения

| Экран | Задача пользователя | Главная ценность | Метрика успеха |
|-------|---------------------|------------------|----------------|
| `/chart/:id` overview (Simple) | **Новичок**: "Что моя карта говорит обо мне?" | Инсайт о себе через 3-5 ключевых фактов | Фраза "это про меня" (engagement > 30 сек) |
| `/chart/:id` overview (Pro) | **Астролог**: быстрая оценка карты клиента | Вся ключевая информация без скролла | Все данные видны above the fold |
| `?tab=planets` | Где какая планета, в каком статусе | Полная таблица позиций = рабочий инструмент | Время до нахождения нужной планеты < 2 сек |
| `?tab=dashas` | Какой период жизни сейчас и что впереди | Временная карта жизни, текущий момент | Юзер находит текущий период без помощи |
| `?tab=strength` | Какие планеты сильные/слабые | Количественная оценка = аргумент для интерпретации | Понимание "красный = слабый, зелёный = сильный" |
| `?tab=yogas` | Есть ли особые комбинации в карте | "У вас есть Гаджакешари-йога" = wow-фактор | Новичок: понял что йога = хорошо/плохо. Про: нашёл все йоги |
| `?tab=forecast` | Что происходит сейчас по транзитам | Связь космоса с "сейчас" | Юзер соотнёс транзит с событием в жизни |
| `?tab=panchanga` | Какой сегодня день по ведическому календарю | Ритуальная/практическая ценность | Ежедневные возвраты на этот экран |
| `?tab=muhurtas` | Благоприятное время для действий | Практическое руководство "когда делать что" | Юзер выбрал мухурту для реального действия |
| `?tab=varna` | Распределение стихий/варн в карте | Типология: "кто я по природе" | Новичок: узнал свой тип. Про: подтвердил гипотезу |

## Критические инсайты

### 1. Dashboard — не ценный экран сейчас
Три карточки (кол-во карт, последняя консультация, панчанга) — это метрики, не ценность. Юзер не приходит чтобы увидеть "у вас 3 карты". Он приходит посмотреть СВОЮ карту. Dashboard должен быть shortcut к главной карте, а не витрина статистики.

**Решение**: Primary card = карта с `is_primary=true` (мини-дашборд с лагной, текущим периодом, сегодняшней панчангой). Статистика — secondary.

### 2. Chart overview (Simple) — ключевой экран, но пустой по смыслу
Сейчас Simple mode = те же данные что Pro, только меньше. Это WRONG. Simple = другой экран с другой задачей:
- Pro задача: "дай мне все данные быстро"
- Simple задача: "расскажи мне про мою карту"

Без AI-описаний Simple mode неполноценен. Но даже до AI можно: подписи к лагне ("Ваш восходящий знак определяет..."), к текущему периоду ("Сейчас вы в периоде Венеры — это время...").

### 3. Forecast — самый retention-friendly экран
Транзиты меняются каждый день. Это единственный экран с обновляемыми данными. Если довести до ума — это daily hook. Сейчас это статичная таблица. Должен быть "что происходит сегодня / на этой неделе".

### 4. Panchanga — скрытый daily driver
Ежедневная панчанга = причина открыть приложение каждый день. Но сейчас она спрятана в сайдбаре и требует primary chart. Должна быть на dashboard или даже на lock-screen widget.

### 5. Yogas — wow-момент, закопанный в табах
Йоги = единственные данные, которые вызывают эмоцию ("у меня есть йога мудрости!"). Сейчас они badges в подвале overview и таблица на табе. Должны быть hero-элементом для новичка.
