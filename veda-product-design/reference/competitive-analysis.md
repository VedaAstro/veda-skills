# Конкурентный анализ — астрологические приложения

> Не пересказ фич — а design decisions, которые мы берём или отвергаем.

## Матрица позиционирования

```
                    Данные (плотность)
                         ↑
                         │
         Astro-Seek ─────┤──── Astrodienst
         (бесплатный,    │    (стандарт точности,
          UX из 2005)    │     перегружен)
                         │
                         │        ← VEDA Pro mode
                         │
    ─────────────────────┼────────────────────→ Смысл (интерпретация)
                         │
              ← VEDA Simple mode
                         │
         The Pattern ────┤──── Co-Star
         (social,        │    (текст + минимализм,
          закрытый)      │     без данных)
                         │
                    Эмоция (ощущение)
```

VEDA занимает уникальную позицию: единственный продукт с ведической астрологией + dual mode (данные И смысл). Ни один конкурент этого не делает.

## Co-Star

**Подход**: Астрология как текстовый нарратив. Минималистичный чёрно-белый UI. Каждый день — персональный текст про настроение, отношения, работу, духовность.

### Что берём
1. **Текст вместо таблиц для новичков.** Co-Star не показывает "Луна в Раке, 9 дом" — показывает "Ваша эмоциональная природа стремится к глубоким корням и безопасности". Simple mode VEDA должен работать так же.
2. **Ежедневный push.** Одна фраза в уведомлении = причина открыть. У нас есть панчанга — это наш ежедневный hook, но он не используется.
3. **"Про тебя" язык.** Каждый экран Co-Star обращается к "you". Не "Солнце в Овне" — а "Your fundamental self is...". Персонализация через местоимения.

### Что отвергаем
1. **Скрытие данных.** Co-Star не показывает градусы, дома, аспекты. Для нашей Pro-аудитории это неприемлемо.
2. **Western-only.** Тропический зодиак, нет сидерического. Наша ценность — ведическая система.
3. **Чёрно-белый дизайн.** Для эзотерической аудитории VEDA нужно тепло, не холод.

## Astro-Seek

**Подход**: Калькуляторная площадка. Десятки инструментов: натальная карта, транзиты, прогрессии, синастрия, солнечные возвраты. Бесплатно, с рекламой. UI из 2005 года, но всё работает.

### Что берём
1. **Плотность данных на Pro-экранах.** Astro-Seek показывает 12+ столбцов в таблице планет. Наш PlanetsTable — 12 столбцов, это правильно. Не упрощать Pro.
2. **Калькуляторная философия.** Ввёл данные → получил результат → всё. Никаких sign-up wall для базовых функций. Наш chart/new должен быть таким же быстрым.
3. **Мульти-система.** Sidereal + Tropical + разные ayanamsa. Для Pro-юзера это must-have. Мы пока только D1-D12 варги — уже хорошо, но нужно расширять.

### Что отвергаем
1. **UI без иерархии.** Всё одинаковым шрифтом, нет визуального якоря. Наш принцип "один якорь на экран" — антитеза Astro-Seek.
2. **Нет progressive disclosure.** Все инструменты видны сразу. Новичок теряется. Наш dual mode решает это.
3. **Реклама в контенте.** Баннеры между данными. Мы — premium-продукт, реклама исключена.

## Astrodienst (astro.com)

**Подход**: Академическая точность. Swiss Ephemeris — стандарт индустрии. Интерпретации от Liz Greene и Robert Hand (платные). Бесплатные карты и базовые описания.

### Что берём
1. **Расчётная точность как бренд.** Astrodienst = точность. VEDA должна быть "Astrodienst для ведической". Ни одной ошибки в расчётах — это фундамент доверия.
2. **Tiered content.** Бесплатные карты + платные глубокие интерпретации. Наша модель: бесплатная карта в Simple → платная консультация/курс.
3. **Knowledge base.** Astrodienst имеет обширную библиотеку статей. Наш /news + /faq — зачатки этого. Но нужно связать знания с конкретной картой юзера.

### Что отвергаем
1. **Перегруженный UI.** 50+ пунктов меню. У нас 8 пунктов в сайдбаре — и это правильный лимит.
2. **Отсутствие мобильной версии.** astro.com до сих пор desktop-first. Наши 60% юзеров — мобильные.
3. **Холодная академичность.** Тексты написаны для астрологов-интеллектуалов. Наша аудитория A1 ("потерявшая себя") нуждается в тёплом, поддерживающем тоне.

## The Pattern

**Подход**: Астрология без астрологии. Никаких "Луна в 9 доме" — только "You tend to feel emotions deeply and need a sense of home". Social features: сравнение совместимости с друзьями, timing alerts.

### Что берём
1. **Перевод данных в ощущения.** The Pattern не говорит "транзит Сатурна" — говорит "period of growth through challenge". Наш Simple mode может делать то же самое, сохраняя ведическую терминологию для Pro.
2. **Social proof через сравнение.** "See how you connect with friends" — мощный retention hook. Синастрия / совместимость — будущая фича VEDA.
3. **Timing notifications.** "Today is significant for your personal growth". Наш forecast tab + панчанга = raw data для таких нотификаций.

### Что отвергаем
1. **Полное скрытие системы.** The Pattern прячет ВСЮ астрологию. Наши юзеры — ученики школы, они ХОТЯТ учиться. Мы обучаем, не прячем.
2. **Закрытость.** Нет экспорта, нет кастомизации. Наша Pro-аудитория требует контроля.
3. **Только Western.** Та же проблема, что у Co-Star.

## Стратегические выводы для VEDA

### 1. Уникальная ниша: "ведический Co-Star + Astro-Seek в одном"
Ни один конкурент не предлагает dual experience (нарратив для новичка + данные для про) в ведической системе. Это наше конкурентное преимущество. Его нужно усиливать, а не размывать.

### 2. Simple mode = наш Co-Star
Текст, смысл, "про тебя". Не урезанный Pro, а принципиально другой экран. Без этого мы теряем 60% аудитории (A1 + A2).

### 3. Pro mode = наш Astro-Seek
Плотность, скорость, все данные. Без этого мы теряем астрологов-практиков, которые приводят учеников.

### 4. Daily hook = наша панчанга + forecast
Co-Star и The Pattern держат юзеров ежедневными push. У нас есть данные для этого (панчанга, транзиты). Нет delivery mechanism (push + dashboard widget).

### 5. Расчётная точность = фундамент
Astrodienst доказал: точность расчётов = доверие = retention. Наш Go-движок в Docker даёт точность. Любая ошибка в градусах = потеря Pro-юзера навсегда.
