# Emotional Design — как экран ощущается как "про тебя"

> Астрология — самая личная тема. Человек видит карту своего рождения, свою судьбу, свои сильные и слабые стороны. Каждый экран — это зеркало. Задача дизайна — чтобы это зеркало было тёплым, а не холодным.

## Проблема: таблица vs зеркало

Сейчас app.myveda.ru показывает данные в формате "база данных → таблица → экран". Юзер видит:

```
☽ Рак, 9 дом
```

Это технически верно. Но это ощущается как строка из spreadsheet, не как факт о твоей жизни.

Тот же факт, но как зеркало:

```
Ваша Луна в Раке
Эмоциональный мир для вас — это дом, семья, забота о близких.
9 дом усиливает потребность в духовности и глубоком понимании жизни.
```

Разница — не в объёме текста. Разница в том, говорит ли экран С ТОБОЙ или ПРО ДАННЫЕ.

## Принципы эмоционального дизайна

### 1. Обращайся ко второму лицу
**Правило**: Каждый экран, каждый компонент обращается к "вам/тебе", не к данным.

| Холодно (сейчас) | Тепло (цель) |
|---|---|
| Карта #17 | Ваша натальная карта |
| Лагна: Телец | Ваш восходящий знак — Телец |
| Маха-даша: Венера | Сейчас вы в периоде Венеры |
| Шадбала: 1.2 | Ваш Юпитер имеет высокую силу |
| Йоги: Гаджакешари | У вас есть Гаджакешари-йога — комбинация мудрости |

**Где применять**: заголовки секций, подписи к данным, empty states, loading states.
**Где НЕ применять**: таблицы данных в Pro mode (там нужна плотность, не нарратив).

### 2. Факт → значение → "что это для меня"
**Правило**: Каждый астрологический факт подаётся тройкой.

```
ФАКТ:      Солнце в Овне
ЗНАЧЕНИЕ:  Овен — огненный знак лидерства
ДЛЯ МЕНЯ:  Вы от природы стремитесь быть первой, вести за собой
```

В Simple mode показываем все три уровня. В Pro mode — только факт (значение и "для меня" — по hover или expand).

### 3. Один wow-момент на экран
**Правило**: Каждый экран содержит один элемент, который вызывает эмоцию.

| Экран | Wow-момент |
|---|---|
| Chart overview (Simple) | Описание лагны: "Ваш Телец — это..." |
| Chart overview (Pro) | Текущая даша + период жизни |
| Planets tab | Ретроградная планета с объяснением |
| Dashas tab | Визуализация текущего момента на временной шкале всей жизни |
| Yogas tab | Самая значимая йога с описанием: "Это редкая комбинация, которая даёт..." |
| Forecast tab | "Сегодня Юпитер проходит через ваш 5 дом — это день для творчества" |
| Panchanga | "Сегодня — благоприятный день для начинаний" |

### 4. Прогрессивное раскрытие эмоции
**Правило**: Не вываливай всё сразу. Сначала якорь, потом контекст, потом глубина.

```
Level 1 (заголовок):  Гаджакешари-йога
Level 2 (подписка):   Комбинация мудрости и удачи
Level 3 (expand):     Луна и Юпитер в вашей карте формируют эту йогу.
                      Люди с Гаджакешари отличаются хорошей памятью,
                      мудростью и способностью влиять на других.
```

Новичок останавливается на Level 2 ("о, это хорошо"). Про идёт на Level 3 ("ага, Луна-Юпитер, логично").

### 5. Микро-паттерны заботы

#### Loading state
**Холодно**: Спиннер.
**Тепло**: "Рассчитываем вашу карту..." → "Определяем положение планет..." → "Готово!"
Даже если расчёт мгновенный — добавить 1-2 сек с промежуточным текстом создаёт ощущение "для меня считают лично".

#### Empty state
**Холодно**: "У вас пока нет натальных карт."
**Тепло**: "Ваша натальная карта ждёт создания. Введите дату и время рождения — и вы увидите уникальный рисунок, который небо создало в момент вашего появления на свет."

#### First-time experience
**Холодно**: Пустой dashboard с нулями.
**Тепло**: Full-screen приветствие: "Добро пожаловать в VEDA. Давайте создадим вашу первую натальную карту — это займёт 2 минуты."

#### Chart name
**Холодно**: "Карта — Мария Иванова" (или хуже: chart ID).
**Тепло**: "Натальная карта Марии" (имя в родительном падеже, без фамилии — это personal space).

#### Date format
**Холодно**: 15.04.1987 14:30
**Тепло**: 15 апреля 1987, 14:30 (полная дата, человеческий формат — это день рождения, а не timestamp).

### 6. Цветовой эмоциональный код

Астрология связана с планетами, а планеты — с цветами. Используй это:

| Планета | Цвет (традиция) | Применение |
|---|---|---|
| Солнце | Золотой/amber | Подсветка знака солнца, badge |
| Луна | Серебристый/cool-gray | Подсветка знака луны |
| Марс | Красный/error-600 | Ретроградность, предупреждения |
| Меркурий | Зелёный/success-600 | Интеллект, коммуникация |
| Юпитер | Жёлтый/warning-500 | Мудрость, йоги |
| Венера | Розовый/pink | Отношения, даша Венеры |
| Сатурн | Тёмно-синий/blue-800 | Кармические уроки, ограничения |

**Правило**: Не перекрашивать весь UI — использовать как accent color в badges, icons, progress bars. Основной UI остаётся нейтральным (Untitled UI tokens).

## Anti-паттерны (что убивает эмоцию)

### 1. ID вместо имени
```
✗ Карта #47         → ✓ Натальная карта Марии
✗ Консультация #12  → ✓ Консультация от 15 марта
✗ Продукт SKU-234   → ✓ Курс "Основы ведической астрологии"
```

### 2. Технический жаргон без контекста
```
✗ Шадбала: 1.234      → ✓ Сила Юпитера: высокая (1.2 из 1.0 нормы)
✗ Аштакаварга: 28      → ✓ Энергия 5 дома: 28 очков (среднее: 25)
✗ D9                    → ✓ Навамша (карта отношений)
```

### 3. Дата как timestamp
```
✗ 2024-03-15T14:30:00  → ✓ 15 марта 2024, 14:30
✗ 15.04.1987            → ✓ 15 апреля 1987 года
```

### 4. Безличные заголовки
```
✗ Планеты              → ✓ Ваши планеты (Simple) / Планеты (Pro)
✗ Даши                 → ✓ Периоды вашей жизни (Simple) / Даши (Pro)
✗ Текущий период       → ✓ Где вы сейчас (Simple) / Текущая маха-даша (Pro)
```

### 5. Одинаковый тон для Simple и Pro
Simple = разговор с другом ("У вас Луна в знаке заботы — Раке").
Pro = справочник ("☽ Рак, 9 бхава, 15°23', Пушья").
Смешивать нельзя. Каждый режим — свой голос.

## Чек-лист эмоционального аудита экрана

Перед релизом любого экрана:

```
1. Есть ли на экране хоть одно предложение, обращённое к "вам"?
2. Есть ли wow-момент (факт → значение → для меня)?
3. Empty state — тёплый или "нет данных"?
4. Loading — "считаем для вас" или безликий спиннер?
5. Заголовок — личный или технический?
6. Даты — человеческий формат или timestamp?
7. Есть ли ID/номер, который видит юзер? (не должно быть)
8. Tone of voice — соответствует режиму (Simple/Pro)?
```
