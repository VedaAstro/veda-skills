# User Journeys — от входа до ценности

> Два конкретных пути через app.myveda.ru. Для каждого шага: что человек видит, думает, чувствует, и где мы теряем его.

## Мария, 37 лет — новичок

**Контекст**: Прошла вебинар VEDA, купила курс Астро-1. Астрология = интересно, но непонятно. Скачала приложение по ссылке от куратора. Сегмент A1 ("потерявшая себя"). Хочет понять "кто я на самом деле".

### Шаг 1: Регистрация
**Видит**: Форма — имя, email, пароль.
**Думает**: "Ну, стандартно."
**Чувствует**: Нейтрально. Нет ожидания.
**Проблема**: Нет onboarding-обещания. "Зарегистрируйтесь и узнайте свою карту рождения" — этого нет. Просто форма.
**Решение**: Добавить подзаголовок: "Через 2 минуты вы увидите свою натальную карту".

### Шаг 2: Dashboard (пустой)
**Видит**: "Привет, Мария!" + 3 карточки с нулями (0 карт, нет консультаций, панчанга без данных).
**Думает**: "И что дальше? Куда нажимать?"
**Чувствует**: Растерянность. Пустой экран = "я что-то делаю не так".
**Проблема**: Dashboard показывает ОТСУТСТВИЕ данных вместо приглашения. Empty state = дефолтные нули.
**Решение**: Первый визит = full-screen CTA "Создайте свою первую натальную карту" с объяснением зачем. Не 3 пустых карточки.

### Шаг 3: Создание карты
**Видит**: Форма — имя, дата, время, город.
**Думает**: "О, а точное время... Мне мама говорила в 14:30, вроде."
**Чувствует**: Лёгкая тревога ("а если время неточное?").
**Проблема**: Нет подсказки что делать если не знаешь точное время. Нет объяснения почему время важно.
**Решение**: Tooltip: "Если вы не знаете точное время — укажите приблизительное. Основные расчёты будут верны." Quick cities — хорошо, но пять городов не покрывают 80% аудитории (нужно: Москва, СПб, Новосибирск, Екатеринбург, Казань для RU).

### Шаг 4: Chart overview (Simple mode)
**Видит**: Ромбовидный чарт слева, Key Info Panel справа (лагна, солнце, луна, раху/кету), PlanetCards внизу, YogaBadges.
**Думает**: "Красиво, но... Лагна в Тельце. И что? Что это значит для меня?"
**Чувствует**: **Разочарование.** Данные без смысла. Числа без истории. "Я пришла за ответами, а получила таблицу."
**Проблема**: **ЭТО ТОЧКА ПОТЕРИ.** Нет ни одного предложения на экране, которое говорит Марии что-то О НЕЙ. Всё — технические данные.
**Решение**: Simple mode должен показывать:
- "Ваш восходящий знак — Телец. Это означает, что для вас важны стабильность, красота и материальный комфорт."
- "Солнце в Овне (5 дом) — ваша суть — творчество и самовыражение."
- "Сейчас вы в периоде Венеры (2024-2044) — время гармонии, отношений и искусства."

### Шаг 5: Куда кликает дальше?
**Сценарий А** (с текстовыми описаниями): Скроллит, читает про каждую планету. Находит "Луна в 9 доме — глубокая потребность в духовности и путешествиях". Говорит подруге: "Представляешь, у меня луна в доме путешествий!" → Retention.
**Сценарий Б** (текущее состояние): Видит "☽ Рак, 9 дом". Не понимает. Закрывает. Возвращается в Telegram куратору: "А как пользоваться приложением?" → Нагрузка на поддержку.

### Шаг 6: Момент "ага, это про меня"
**Где он сейчас?** Нигде. Ни один элемент текущего UI не создаёт момент узнавания.
**Где он должен быть?** Chart overview (Simple) — первый экран после создания карты. Три предложения о лагне, солнце и луне = три шанса на "это про меня".
**Альтернатива без AI**: Даже статичные описания знаков ("Телец — знак стабильности, красоты и упорства") создадут этот момент. Не нужен GPT, нужна таблица из 12 описаний.

## Елена, 42 года — практикующий астролог

**Контекст**: 5 лет практики ведической астрологии. Работает в Astro-Seek + Excel. Ищет инструмент быстрее и удобнее. Консультирует 3-5 клиентов в неделю. Знает BPHS, Парашару, Джаймини.

### Шаг 1: Логин → создание карты клиента
**Видит**: Форму создания карты.
**Думает**: "Быстро ли это? В Astro-Seek я за 20 секунд создаю карту."
**Чувствует**: Оценивает скорость. Если больше 30 сек — "хуже чем у меня".
**Проблема**: Quick cities — 5 городов, ни один не помогает для клиента из Воронежа. Autocomplete для городов = ОК, но lat/lon ручной ввод — friction.
**Решение**: Autocomplete должен заполнять lat/lon автоматически (уже делает через Nominatim — хорошо). Время создания карты = benchmarked < 20 сек.

### Шаг 2: Chart overview → переключает в Pro mode
**Видит**: Чарт + Key Info + PlanetCards + Yogas.
**Первое действие**: Нажимает "Pro" (через 2 сек — знает что "Обзор" = для новичков).
**Думает**: "Ок, базовые данные на месте. Где полная таблица планет?"
**Чувствует**: Проверяет точность. Сравнит лагну и позиции с тем, что получает в Astro-Seek. Если расхождение > 1° — уйдёт.

### Шаг 3: Сканирует обзор (Pro mode)
**Порядок сканирования** (eye tracking pro-астролога):
1. **Чарт** — 2 сек, оценка общей картины (какие дома заполнены, где скопления)
2. **Лагна** — подтверждение знака и градуса
3. **Текущая даша** — что сейчас у клиента
4. **Йоги** — какие комбинации, сходу видно уровень карты
5. **Планеты** — быстрый скан ретроградности и домов

**Проблема**: Key Info Panel показывает 5 планет без градусов в Simple mode. В Pro — с градусами. Но PlanetCards дублирует ту же информацию в другом формате. Елена вынуждена смотреть в два места.

### Шаг 4: Переключение табов
**Порядок** (типичная консультация):
1. **Планеты** — полная таблица, все 9 грах + точные градусы + накшатры + функциональная природа
2. **Даши** — временная линия, найти текущий период + следующий
3. **Сила** — шадбала, аштакаварга (кто сильный, кто слабый)
4. **Йоги** — полный список с участвующими планетами
5. **Прогноз** — транзиты на текущий момент (опционально)

**Время на каждом табе**: 10-30 сек. Елена сканирует, не читает. Таблицы = быстро. Карточки = медленно.

### Шаг 5: Момент "я нашла что искала"
**Что ищет**: Подтверждение гипотезы. "Клиент жалуется на проблемы в карьере. Сатурн в 10 доме ретроградный + шадбала низкая + нет йог для карьеры = вот почему."
**Где находит**: Таб "Планеты" (Сатурн, 10 дом, R) + таб "Сила" (шадбала < среднего) + таб "Йоги" (нет раджа-йог).
**Проблема**: 3 переключения табов чтобы подтвердить одну гипотезу. В Astro-Seek всё на одном экране.
**Решение**: Pro overview должен быть плотнее. Или: возможность открыть 2 таба рядом (split view).

### Шаг 6: Выход
**Что делает**: Копирует нужные данные в свои заметки (ручками, screenshot). Закрывает.
**Что хотела бы**: Export в PDF / таблицу. Copy-paste из таблицы планет. Шаринг карты клиенту (ссылка).
**Проблема**: Нет экспорта, нет share, нет print view.

## Рекомендации по приоритету

### P0 — блокеры
1. **Simple mode с текстовыми описаниями** — без этого новичок не получает ценности. Даже 12 статичных описаний лагны = прорыв.
2. **First-time experience** — пустой dashboard = потеря. Должен быть onboarding flow: "Создайте карту → вот ваша карта → вот что она значит".

### P1 — retention
3. **Daily hook** — панчанга на dashboard или push-уведомление. Причина вернуться каждый день.
4. **Forecast как живой экран** — "что происходит с вашей картой сегодня" вместо статичной таблицы транзитов.

### P2 — Pro UX
5. **Плотнее Pro overview** — убрать дублирование Key Info / PlanetCards.
6. **Export / Share** — PDF, copy-paste из таблиц, share link.
