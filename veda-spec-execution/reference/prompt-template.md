# Промпт-шаблон для исполнения спек

> Этот документ -- полное руководство по подготовке промпта для реализации спеки в новой Claude-сессии.
> Используй его, когда нужно передать задачу из одной сессии в другую, или когда готовишь промпт для себя.

---

```
(Перечисли этапы из спеки с номерами. Если спека большая -- укажи какие этапы

1. БД (миграция Alembic) -- если есть новые таблицы/колонки

4. BFF route (app/api/*/route.ts)
5. Типы TypeScript (app/types/*.ts)
6. Фронтенд компоненты
7. npm run build (проверка)

1. npm run build -- без ошибок
2. TypeScript -- без ошибок

5. Один этап = один коммит

## Контекст

- Ключевые файлы (прочитай перед началом): {{ФАЙЛЫ}}

- НЕ начинай без чтения спеки целиком
- НЕ пропускай npm run build между этапами
```

## 2. Как заполнять каждый плейсхолдер

### {{ПУТЬ}} -- Путь к файлу спеки

Абсолютный путь к .md-файлу спецификации.

| Пример | Когда |
|--------|-------|
| `~/Projects/Claude_Docs/spec-management-rework-v5.md` | Спека в общей папке |
| `~/Projects/platform/veda-presentation/docs/spec-new-feature.md` | Спека рядом с кодом |

**Правило:** Файл должен существовать и быть доступен для чтения в новой сессии.

---

### {{ОПИСАНИЕ}} -- Что уже сделано

Описывай конкретно: какие файлы уже созданы/изменены, что в них готово, что НЕ готово.

**Хорошо:**
```
Этап 1 (БД) выполнен:
- Миграция: veda-backend/alembic/versions/20260218_add_cases_table.py (применена)
- Модель: veda-backend/db/models.py -- добавлена модель Case (строки 450-490)
- НЕ сделано: API эндпоинты, BFF, фронтенд
```

**Плохо:**
```
Половина сделана.
```

**Если с нуля:**
```
Всё с нуля. Ничего не реализовано.
```

---

### {{ЭТАПЫ}} -- Что делать

Перечисли этапы из спеки с указанием объёма. Если спека большая -- ограничь scope текущей сессии.

**Хорошо:**
```
Этапы из спеки (раздел "Задачи"):
1. [Эта сессия] Задача 1: Создать модель Case + миграция (раздел 3.1 спеки)
2. [Эта сессия] Задача 2: CRUD API для Case (раздел 3.2 спеки)
3. [Эта сессия] Задача 3: BFF + TypeScript типы (раздел 3.3 спеки)
4. [Следующая сессия] Задача 4: Фронтенд -- страница /admin/cases/ (раздел 4 спеки)
5. [Следующая сессия] Задача 5: Интеграция с диагностикой (раздел 5 спеки)
```

**Плохо:**
```
Сделай всё что в спеке.
```

---

### {{СКИЛЛЫ}} -- Какие скиллы прочитать

Укажи скиллы, которые дают контекст для реализации. Claude прочитает их автоматически.

| Скилл | Когда нужен |
|-------|-------------|
| `/veda-backend-patterns` | Любая работа с бэкендом (API, модели, миграции) |
| `/diagnostic-workstation` | Изменения в рабочей станции диагноста |
| `/vedic-astro-knowledge` | Работа с астро-данными, трактовками, картами |
| `/veda-deploy` | Когда нужен деплой на сервер |
| `/veda-spec-execution` | Всегда (этот скилл) |

**Формат:**
```
- /veda-spec-execution (порядок реализации, чеклисты)
- /veda-backend-patterns (Enum gotchas, порядок роутов, BFF-паттерн)
```

---

### {{ФАЙЛЫ}} -- Ключевые файлы для чтения

Перечисли файлы, которые Claude ДОЛЖЕН прочитать перед началом работы. Это файлы, которые будут изменены или от которых зависит реализация.

**Категории файлов:**

| Категория | Примеры |
|-----------|---------|

| BFF-роуты | `veda-presentation/app/api/diagnostic/route.ts` |

**Формат:**

## 4. Чеклист подготовки промпта

Перед тем как писать промпт, пройди этот чеклист:

### 4.1 Понимание задачи

- [ ] Спека прочитана ЦЕЛИКОМ (не по диагонали)
- [ ] Понятна цель: что должно работать в результате
- [ ] Понятен scope: что делать, а что НЕ делать
- [ ] Если спека большая (>100 строк) -- определи, какие этапы в эту сессию

### 4.2 Определение слоев

| Вопрос | Если "Да" | Слой |
|--------|-----------|------|
| Нужны новые таблицы/колонки? | Alembic миграция | БД |
| Нужны новые модели/Enum'ы? | db/models.py | Модель |
| Нужны новые API-эндпоинты? | api/routes/*.py | Бэкенд |
| Фронтенд вызывает бэкенд? | app/api/*/route.ts | BFF |
| Нужны новые типы на фронте? | app/types/*.ts | Типы |
| Нужны новые компоненты/страницы? | app/components/ или app/*/page.tsx | Фронтенд |

### 4.3 Определение скиллов

| Что в спеке | Какой скилл |
|-------------|-------------|
| Новые API-эндпоинты, модели, миграции | `/veda-backend-patterns` |
| Изменения в рабочей станции диагноста | `/diagnostic-workstation` |
| Работа с астро-данными, трактовками | `/vedic-astro-knowledge` |
| Нужен деплой на сервер | `/veda-deploy` |
| Любая спека (всегда) | `/veda-spec-execution` |

### 4.4 Список файлов

Чтобы определить ключевые файлы, задай себе:

1. **Какие файлы будут ИЗМЕНЕНЫ?** -- перечисли все
2. **Какие файлы -- ОБРАЗЦЫ паттерна?** -- если создаёшь новый роутер, укажи существующий похожий
3. **Какие файлы -- ЗАВИСИМОСТИ?** -- контексты, сторы, типы, от которых зависит код

### 4.5 Финальная проверка промпта

- [ ] Путь к спеке указан и файл существует
- [ ] Описание "что сделано" конкретное (файлы, строки) или "Всё с нуля"
- [ ] Этапы пронумерованы и ограничены scope сессии
- [ ] Скиллы указаны
- [ ] Файлы для чтения перечислены (5-10 штук)
- [ ] Секция "Что НЕ делать" заполнена
- [ ] Нет противоречий между этапами

## 5. Типичные ошибки и как их избежать

### Ошибка 1: "Сделай всё что в спеке" без конкретики

**Проблема:** Claude начнёт с первого раздела и может не дойти до конца, потеряв контекст.

**Решение:** Всегда указывай конкретные этапы и их scope.

```
-- Плохо:
Сделай спеку ~/Projects/Claude_Docs/spec-management-rework-v5.md

-- Хорошо:
Выполни спеку. Путь: ~/Projects/Claude_Docs/spec-management-rework-v5.md
В этой сессии делай ТОЛЬКО разделы 2-3 (РНП + Спринты).
Разделы 4-6 (Задачи, Дашборд, AI) -- следующая сессия.
```

---

### Ошибка 2: Не указаны файлы для чтения

**Проблема:** Claude начнёт менять файлы без понимания текущего состояния кода. Будут конфликты, дубликаты, сломанные импорты.

**Решение:** Перечисли 5-10 ключевых файлов, которые Claude ДОЛЖЕН прочитать перед началом.

```
-- Плохо:
(никаких файлов)

-- Хорошо:
Файлы (прочитай перед началом):
- veda-backend/db/models.py -- модели Management* (строки ~600-750)
- veda-backend/api/routes/management.py -- существующий роутер
- veda-presentation/app/types/management.ts -- типы
```

---

### Ошибка 3: Забыт BFF-слой

**Проблема:** Бэкенд API создан, фронтенд компонент создан, но между ними нет BFF-роута. Фронтенд не может достучиться до API (Next.js не делает прямые запросы к FastAPI, всё через BFF).

**Решение:** В этапах ВСЕГДА упоминай BFF-слой, если есть бэкенд + фронтенд.

```
-- Плохо:
1. Бэкенд API
2. Фронтенд компонент

-- Хорошо:
1. Бэкенд API (api/routes/cases.py)
2. BFF route (app/api/cases/route.ts)
3. TypeScript типы (app/types/cases.ts)
4. Фронтенд компонент
```

---

### Ошибка 4: Пропущен npm run build между этапами

**Проблема:** Ошибки TypeScript накапливаются. К 5-му этапу 20 ошибок, и непонятно где началась проблема.

**Решение:** В промпте явно укажи обязательность билда.

```
## Обязательные проверки после КАЖДОГО этапа
1. npm run build -- без ошибок
2. TypeScript -- без ошибок
```

### Ошибка 5: Один гигантский коммит на всю спеку

**Проблема:** Если что-то сломалось -- невозможно откатить конкретный этап. Всё или ничего.

**Решение:** Один этап = один коммит с осмысленным сообщением.

```
-- Плохо:
git commit -m "реализация спеки"

-- Хорошо:
git commit -m "feat(management): переименование Ежедневный ввод -> РНП"
git commit -m "feat(management): PDCA-таблица спринта вместо 5 табов"
git commit -m "feat(management): раскрытие строки с RCA и задачами"
```

### Ошибка 6: Описание "что сделано" слишком расплывчатое

**Проблема:** Если предыдущая сессия уже что-то сделала, Claude не понимает с какого места продолжать. Может начать заново и перетереть готовый код.

**Решение:** Перечисли конкретные файлы и что в них готово.

```
-- Плохо:
Бэкенд частично готов.

-- Хорошо:
Этап 1 (БД + модель) ВЫПОЛНЕН:
- Миграция: veda-backend/alembic/versions/2026_02_18_add_cases.py (применена на сервере)
- Модель Case: veda-backend/db/models.py, строки 450-490
- Enum CaseResultType: veda-backend/db/models.py, строки 445-448

Этап 2 (API) НЕ выполнен -- начинай с него.
```

### Ошибка 7: Не указан scope сессии для больших спек

**Проблема:** Спека на 300 строк, 8 задач. Claude начинает с первой и к середине теряет контекст, пропускает детали, торопится.

**Решение:** Разбей на сессии по 2-4 задачи. Явно укажи границы.

```
-- Плохо:
Сделай все 8 задач из спеки.

-- Хорошо:
## Что делать (сессия 1 из 3)
Задачи 1-3 из спеки (разделы 3.1-3.3):
1. Консолидация натальных карт
2. Подключение PNG-планет
3. Обновление CollapsibleLeftPanel

НЕ делать: задачи 4-8 (следующие сессии).
```

### Ошибка 8: Не учтены зависимости между задачами

**Проблема:** Задача 3 зависит от результата задачи 2, но Claude пытается делать их параллельно или в неправильном порядке.

**Решение:** Укажи зависимости явно.

```
## Порядок (ВАЖНО -- зависимости)
1. Сначала: утилита chartPlanetsToPlacement() (от неё зависят шаги 2, 3, 4)
2. Потом: миграция SlideNatalChart (использует утилиту из шага 1)
3. Потом: миграция SlideLoop (использует утилиту из шага 1)
4. Только после 2+3: удаление VedicChartSVG (все потребители мигрированы)
```

## 6. Шаблоны для частых ситуаций

### Продолжение предыдущей сессии

```
Продолжи реализацию спеки.

## Спека
Путь: {{ПУТЬ}}

## Что сделала предыдущая сессия
{{КОНКРЕТНЫЙ СПИСОК файлов и изменений}}

## Что делать в этой сессии
{{ОСТАВШИЕСЯ ЭТАПЫ}}

## Контекст
- Скиллы: {{СКИЛЛЫ}}
- Файлы: {{ФАЙЛЫ}}

## Проверь перед началом
1. Прочитай спеку -- разделы {{НОМЕРА}} (уже сделано) и {{НОМЕРА}} (текущие задачи)
2. Прочитай файлы, изменённые предыдущей сессией (проверь что всё корректно)
3. npm run build -- убедись что всё собирается до начала работы
```

```
## Файлы для проверки
{{ФАЙЛЫ, где скорее всего баг}}

## После фикса
```
