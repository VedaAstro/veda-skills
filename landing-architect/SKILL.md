# Архитектор лендингов

## Когда использовать

- Создание лендинга регистрации на бесплатный вебинар / марафон

- Аудит существующего лендинга на конверсию

### 3. Mental Shifts Journey (шаблон — заполнить перед кодом)

Лендинг = 8–15 внутренних сдвигов. Прописать в комментарии к `page.tsx`:

```
1. «Это про меня» (узнавание себя) → Hero + Pre-headline
2. «Об этом пишут серьёзные СМИ» (trust) → Media logos
3. «Они знают мою боль дословно» → Problem / VoC-цитаты
4. «А что если бы было иначе...» → Dream / BAB-after
5. «Я вижу инструмент — это работает» → Method / Demo с мокапом
6. «Ниша реальна» → Market / источники
7. «Такие как я уже заработали» → Social Proof / кейсы
8. «Это для меня даже с моим барьером» → ForWhom / сегменты
9. «Я увижу результат на каждом этапе» → Program / модули через benefit
10. «Я получаю больше, чем плачу» → Value Stack / Hormozi
11. «Цена посильна» → Tariffs
12. «Попробовать не страшно» → Guarantee
13. «Доверяю эксперту» → About Expert
14. «Все мои «а если» закрыты» → Objections / FAQ
15. «Сейчас или потом дороже» → Final CTA + urgency
```

Каждый блок — ОДИН сдвиг. Повторов не должно быть. Если в блоке нет конкретного сдвига — блок убрать.

### 6. Anti-patterns (ловушки из сессии 17.04.2026)

Эти фразы и подходы НИКОГДА не попадают в лендинг:

| ❌ Запрещено | ✅ Правильно |
|---|---|
| «снимает ваш страх», «закрывает страхи» | Конкретика: «первые 30 тысяч за 10 дней», «без 5 лет в Индии» |
| «JTBD», «value stack» (лейблы на ленде) | «Что вы сможете», «Что входит» — человеческий язык |
| «ст. 32 ЗоЗПП», «ст. 219 НК РФ», «ст. 152.1 ГК РФ» | «Если не зайдёт — вернём деньги», «Государство вернёт 13% через Госуслуги» |
| «Три щита, чтобы вы НЕ ПОТЕРЯЛИ деньги» | «Начинайте спокойно. Всё работает на вас» — не сажать страх |
| «готовит ЧЕРНОВИК консультации» | «готовит PDF-консультацию на 20 страниц, которую отдаёте клиенту как готовый продукт» — не обесценивать |
| «3 нейротехнологии», «7 направлений» | Через JTBD байера: что он сможет, кому продаст, как изменится |
| «Для тех, кто хочет снять страхи» | «Для тех, кто устал тратить 5 дней на одну консультацию» (боль дословно) |
| «Рассчитан на тех, кто...» (канцелярит) | «Вы прошли курсы, умеете строить карту, но клиентов нет...» (зеркало) |
| Цветовая: «плохо» и «хорошо» в одном контрасте | Старое = серое/приглушённое (saturate 0.7), новое = тёплый глоу, зелёные галочки |
| Копипаст существующего ленда + переписать контент | Всегда начинать с Mental Shifts Journey под конкретный продукт |
| `<Footer />` полный на каждом ленде | `PnaFooterMini` — одна строка (ИНН, лицензия) + 4 ссылки (оферта, политика, согласие, сведения). Юр-минимум для АНО ДПО |

### 7. High-ticket продажа — через заявку, а не прямую оплату

Для чеков **70К+** прямая оплата через GC-виджет проигрывает заявочной модели в 1,5–2 раза:

**Правильный паттерн:**
- Показывать цену: «от 2 916 ₽/мес · всего 69 990 ₽ · рассрочка 24 мес без %» (обе цены по ст. 435 ГК)
- Кнопка: «Оставить заявку», не «Оформить заказ»
- Форма: имя + телефон + email + выбор тарифа (Профи/VIP/«помогите выбрать») + согласие ПД
- API пишет лид в GC (`addUser` с тегом) + POST в Telegram бот → уведомление РОПу
- TY-экран: «Позвоним в течение 15 минут»

**Критичные условия, иначе заявки сгорят:**
- РОП звонит в <15 мин в рабочие часы (иначе конверсия −3–5×)
- Скрипт менеджера: 3 квалифицирующих вопроса + обработка 8 возражений (взять из блока Objections ленда)
- Минимум 2 тачки если не взяли трубку: SMS через 10 мин + WA/TG через час

**Запретный паттерн** (проверено): форма с GC-лайт-виджетом в iframe — ломает SPA, белый экран, низкая конверсия. См. `getcourse-widget-integration.md`.

Перед deploy — пройти весь лендинг сверху вниз, ВСЛУХ прочесть каждый заголовок и каждый абзац. Отметить и пересобрать каждое слово, которое байер не произнёс бы в реальном разговоре с подругой.

Чек-лист аудита:
- [ ] Нет юридических статей, номеров ФЗ, «ст. X ЗоЗПП» в body-тексте (они только в оферте и футере)
- [ ] Нет слов «закрывает страх / снимает тревогу / JTBD / value stack / mental shift»
- [ ] Нет заголовков-самообесценивания («черновик», «подсказка», «помощник»)
- [ ] Все цифры подкреплены источником (авторитетный URL ИЛИ плашка «внутренняя аналитика Академии, индивидуально»)

- [ ] Цветовая кодировка: старое/проблема = desaturated, новое/решение = тёплый глоу + зелёные чеки

## Ключевые правила (8 нерушимых)

1. **Трансформация, не продукт.** Заголовок = кем станет человек, не что получит. "Стань экспертом" > "Курс по астрологии"
2. **Данные только из плейбуков.** Цены, модули, кейсы, имена -- из `veda-offer` и `buyer-persona`. НЕ придумывать
3. **Голос = Ирина.** Теплый наставник. "Дорогие мои". НЕ копирайтер. НЕ "девочки". НЕ длинное тире
4. **Внешний локус вины.** "Это не ваша ошибка -- причина в карте/планете/периоде". Снятие вины = разрешение купить
5. **CTA видна без скролла.** На десктопе И на мобайле. Hero = 5 элементов
6. **Proof каждые 2-3 экрана.** Кейс / отзыв / цифра / скриншот. Без proof > 3 экранов = потеря доверия
7. **Один лендинг = одно действие.** Нет навигации, нет внешних ссылок
8. **Mobile first. ВСЕГДА.** 90% трафика с телефонов. Мобильная версия -- приоритет #1. Кнопки >= 44px, шрифт >= 16px. Hero контент видим БЕЗ скролла на 375px экране.
9. **Reveal-анимации: SSR visible, hidden только ниже fold.** Любой ScrollReveal/FadeIn в SSR отдаёт `opacity:1`. Скрывать можно только ПОСЛЕ mount через `useEffect` + `getBoundingClientRect()` для элементов ниже fold. ЗАПРЕЩЕНО `opacity: inView ? 1 : 0` — на медленной мобильной сети JS грузится 10+ сек, пока не смонтируется — пользователь видит чёрный экран. Реальный инцидент 2026-04-18: лендинг потерял весь мобильный трафик. Эталон: `platform/vedantica-site/components/ui/scroll-reveal.tsx`.
10. **Sticky bottom CTA: safe-area-inset-bottom обязателен.** `fixed bottom-0` на iOS перекрывается home-indicator и зоной показа Safari toolbar — первый тап по кнопке система перехватывает как «показать тулбар», кнопка кажется некликабельной. Всегда `paddingBottom: calc(env(safe-area-inset-bottom, 0px) + 12px)`. Инцидент 2026-04-18: sticky CTA на iPhone требовал двойного тапа. Эталон: `platform/vedantica-site/components/neuroastrolog/mobile-sticky-cta.tsx`.
11. **Countdown/таймер: SSR-состояние = нейтральный placeholder, НЕ «Время вышло».** `useState(null)` + ветка `if (!left) return "Время вышло"` попадает в SSR-HTML как алярма ещё до первого тика useEffect. На медленном мобильном клиенте пользователь видит «Время вышло» секунды или минуты. Правильно: initial state = `{d:0,h:0,m:0,s:0}` + отдельный флаг `ticked`, и только `ticked && !left` триггерит «Время вышло». Инцидент 2026-04-18: маркетолог увидел «Время вышло» на рабочем лендинге. Эталон: `platform/vedantica-site/components/neuroastrolog/tariffs.tsx` `Countdown`.
12. **Двойной padding между секциями одного фона.** `py-24 lg:py-32` + `py-24 lg:py-32` подряд с одинаковым `background` = ~256px пустоты, которая выглядит как баг. Если следующая секция того же цвета что предыдущая — используй `pb-*` вместо `py-*` (или `pt-*`), чтобы не дублировать отступ на стыке. Инцидент 2026-04-18 на neuroastrolog_0426.

## Копирайтинговые фреймворки

| Фреймворк | Когда | Формула |
|-----------|-------|---------|
| **Hook-Story-Offer** (Brunson) | Универсальный скелет любого лендинга | Hook (любопытство) -> Story (трансформация) -> Offer |
| **PAS** | Болевой вход (основной для астро) | Problem -> Agitate -> Solution |
| **BAB** | Трансформационный продукт | Before -> After -> Bridge (продукт = мост) |
| **Proof-Promise-Plan** (Hormozi) | Оффер / value stack | Proof ПЕРВЫМ -> Promise -> Plan |
| **VoC / Message Mining** (Wiebe) | Тексты и заголовки | Формулировки из слов реальных клиентов (9520 анкет!) |

Детали и примеры для астро-ниши: `reference/copywriting-frameworks.md`

## Формулы заголовков

### Регистрационный лендинг
```
[Конкретный результат] за [формат] -- бесплатно
"Узнай 3 планеты, которые сейчас управляют твоей судьбой -- бесплатный мастер-класс"

Как [желаемое] даже если [возражение]
"Как читать натальную карту, даже если ты полный новичок"

[Число] [секрет/ошибка/шаг] которые [результат]
"5 точек в карте, которые покажут твое предназначение"
```

### Офферный лендинг
```
[Результат] без [главный страх] за [срок]
"Первые 3 клиента через 30 дней -- без холодных продаж"

Из [текущее] в [желаемое] через [механизм]
"Из увлеченного астролога -- в эксперта, которому платят"

Метод [результат] для [аватар]
"Система, которая превращает астрологические знания в стабильный доход"
```

## Юридическая безопасность (баланс конверсия / риск)

Триггеры и сильные формулировки = ОК, КРОМЕ двух зон:

| Зона | Что НЕЛЬЗЯ | Что МОЖНО |
|------|-----------|-----------|
| **Гарантия денег** | "Заработаете 100К", "окупите за месяц", "гарантируем доход" | "Наши ученицы зарабатывают от X" (кейс, не обещание). "Потенциал X разборов = Y руб" |
| **Сверхрезультаты** | "Излечивает", "решает все проблемы", "100% результат" | "Помогает разобраться", "дает инструмент", "ученицы отмечают" |
| **Статус документа** | "Диплом государственного образца", "гособразца", "государственный сертификат", "официальный государственный документ" | "Диплом установленного образца", "Сертификат установленного образца", "сведения вносятся в ФИС ФРДО" (только для ДПО от 250 ак.ч.) |

**ЗАПРЕТ «гособразца»:** АНО ДПО не является госвузом. Любое упоминание «государственного образца» на лендинге = ст. 14.7 КоАП + ст. 10 ЗоЗПП, Роспотребнадзор, претензии, удар по лицензии. Правильные формулировки:
- ДО-тариф: «Сертификат установленного образца (не вносится в ФИС ФРДО)»
- ДПО-тариф (250+ ак.ч., при наличии СПО/ВО): «Диплом о профессиональной переподготовке установленного образца с присвоением квалификации, сведения вносятся в ФИС ФРДО»
- Эталон и дословные формулировки: `shared/veda-skills/legal/SKILL.md` раздел «Итоговые документы»

**Правило:** кейсы = факт ("Ольга заработала 108K"), обещания = потенциал ("инструмент, с которым ученицы зарабатывают"). Между "она заработала" и "вы заработаете" -- юридическая пропасть.

**Disclaimer под кейсами:** "Результаты учениц индивидуальны и зависят от вложенных усилий."

### React Hydration Error #418 — таймеры обратного отсчёта

// В рендере: показывать таймер только когда now !== null

## Интеграция с GetCourse (регистрация + мессенджеры)

### Два критичных правила (не забывать!)

**1. GC идентифицирует контакт по EMAIL.** Телефон — НЕ ключ. На одном номере в GC может быть 1000+ контактов. Любая форма **обязана собирать email**, иначе дубли и сломанный ретеншн. Никогда не генерировать синтетические email-ы вроде `phone-{digits}@wheel.vedantica.ru` — реальный контакт пользователя не найдётся.

**2. Мгновенный бот требует ДВА endpoint-а подряд:** `/process` + `/delay`. Без `/delay` задержка 2-6 минут. Подробности ниже.

### Как работает связка "лендинг → GC → мгновенный процесс → бот"

```
Наш лендинг (vedantica.ru)
  ├─ Кастомная тёмная форма (ОБЯЗАТЕЛЬНО email + телефон, country selector)
  ├─ POST /api/lead → серверный сабмит GC-виджета:
  │    1. addUser (тег pna_reg) — best-effort для тегов
  │    2. fetch HTML виджета → парсинг form action + CSRF + cookies
  │    3. POST /pl/lite/block-public/process → GC создаёт заказ + запускает
  │       процесс формы (но первый шаг ждёт привязки TG → 2-6 мин задержка)
  │    4. Извлекаем invitationHash из json.data.preParts[].html
  │    5. POST /pl/lite/block-public/delay?code={hash} → триггер "обработчиков
  │       кнопки мессенджера" в UI виджета → сообщения шлются МГНОВЕННО
  │    6. Возвращаем tgLink/maxLink клиенту
  └─ TY экран "Остался 1 шаг" с кнопками TG/Max (хеши из виджета)
       └─ Клик → бот → юзер получает сообщение мгновенно (уже в очереди)
```

**КРИТИЧНО `/delay`:** обработчики кнопки мессенджера в UI виджета GC (настраивает Люба) НЕ срабатывают от нашего `<a href="t.me/...">` — они ждут POST на `/pl/lite/block-public/delay?id={blockId}&code={hash}` с form data и сессионными cookies. Наш сервер делает этот POST сразу после `/process`, используя тот же сессионный cookie jar. Без `/delay` GC-процесс уходит в блок "ждать привязки TG" и пересчитывается 2-6 мин — юзер не дождётся и уйдёт.

**КРИТИЧНО процесс:** GC запускает процесс мгновенно ТОЛЬКО из формы/виджета (`/process` или клик в iframe). Deals API (`/pl/api/deals`) и событийные триггеры (добавлен в группу, создан заказ) = задержка 10-15 мин. Подтверждено тех.лидом GC (Люба) и тестами 09-11.04.2026.

### Что нужно для нового лендинга с GC-регистрацией

**В GC (делает техник):**
1. Создать **продукт** (например, "ПНА Акшая Трития")
2. Создать **бесплатное предложение** (цена = 0) внутри продукта

**В коде (делает разработчик):**

2. Компонент формы: после сабмита показывает TY с хеш-ссылками из ответа API

### Обязательные поля формы (минимум)

**email + телефон** — всегда оба, email ОБЯЗАТЕЛЬНО.

- Email = первичный ключ контакта в GC. Без email = дубли.
- Телефон = канал для отдела продаж + идентификация мессенджера при клике.
- Name = опционально (если нет — заглушка `"Гость"` серверсайд).
- Country selector — если есть международный трафик (70+ стран на `/wheel`, `/akshaya`).

### КРИТИЧНО: Валидация full_name в GC (баг-ловушка)

GC валидирует поле `formParams[full_name]` и отклоняет заказ с ошибкой `"Неверные символы в фамилии"` при:
- Прочерке `-` или пустоте
- Цифрах или точках
- Email-подобных строках (`user.name`, `test@ya.ru`)
- Словах короче 2 букв

При этом GC возвращает HTTP 200 + `success: true`, но внутри:
```json
{ "data": { "formProcessed": false, "error": "Неверные символы в фамилии" } }
```

**Заказ НЕ создаётся, контакт НЕ формируется, процесс НЕ запускается** — но снаружи всё выглядит успешным. Легко пропустить.

**Обязательно проверять:**
```typescript
if (!json.data?.formProcessed) {
  throw new Error(json.data?.error || 'Form not processed')
}
```

**Заглушка имени:** если форма не собирает имя (чтобы максимизировать CR), использовать `"Гость"` — проходит валидацию. НЕ использовать `"-"`, `""`, `"User"` + digits, email local-part.

| **Подписная форма** | Сбор email в рассылку | Только контакт с тегом | НЕТ |

### Что НЕ работает (проверено, не повторять)

- **Phone-only форма без email** — GC идентифицирует по email, телефон не ключ. Синтетический email (`phone-{digits}@...`) создаёт дубликаты контактов.
- **GC Deals API для мгновенного процесса** — создаёт заказ, но процесс стартует через 10-15 мин
- **GC API addUser** — создаёт контакт, но НЕ генерирует хеш. Нельзя получить персональную ссылку на бота
- **Событийные триггеры GC** (добавлен в группу, создан заказ, подключился к боту) — всё с задержкой 10-15 мин
- **`/pl/api/processes`** — endpoint НЕ существует в GC API
- **GC-страницы в iframe** — `X-Frame-Options: DENY`
- **GC lite-виджет напрямую в React SPA** — скрипт ломает SPA, белая форма, redirect. НЕ встраивать
- **full_name = `-`, `""`, `"test"`, цифры, точки** — GC отвергает с `formProcessed: false`, заказ тихо не создаётся
- **Парсинг `Vedantica_bot?start=...` regex'ом из HTML виджета** — отдаёт гостевой хеш, не per-user. Использовать только `invitationHash` из `preParts` ответа сабмита
- **Только `/process` без `/delay`** — задержка 2-6 мин на приход сообщения в бот, юзер уйдёт

## Психотриггеры для астро/эзо ниши

| Триггер | Как работает | Где на лендинге |
|---------|-------------|-----------------|
| **Трансформационная идентичность** | Покупают новую версию себя | Hero, Dream section |
| **Предназначение** | "Это мое призвание" > "хочу зарабатывать" | Problem, Bridge |
| **Снятие вины** | "Не твоя ошибка -- карма/планеты" | Problem, Agitate |
| **Разрешение** | "Мне можно потратить на себя" | CTA, Guarantee |
| **Кейс = "такая же как ты"** | Совпадение по ситуации | Social proof |
| **Эксклюзивность** | Закрытое сообщество, ограниченный поток | Urgency |

## Формула покупки

```
Покупка =
  Боль (узнала себя)
  + Снятие вины (не моя ошибка -- карма)
  + Разрешение (мне можно потратить на себя)
  + Снятие риска (рассрочка / гарантия / возврат)
```

## Hormozi Value Equation (для оффера)

```
Ценность = (Dream Outcome x Вероятность достижения)
         / (Время до результата x Усилие)

Задача лендинга:
  Числитель ↑ = яркий результат (headline) + proof (кейсы, цифры)
  Знаменатель ↓ = быстро (сроки) + легко (формат, поддержка)
```

## Быстрый маршрут

### Создание регистрационного лендинга (вебинар/марафон)
1. Определи мероприятие: вебинар / марафон / мастер-класс
2. Сегмент: навык `buyer-persona` -> A1/A2/B1/B2
3. Headline: конкретный результат (не тему!) через формулу выше
4. Структура 8 блоков: `reference/registration-landing.md`
5. Форма: максимум 2-3 поля
6. Social proof: 2-3 отзыва из плейбука
7. Urgency: реальный дедлайн / лимит мест
8. Чеклист: `reference/conversion-checklist.md`

### Создание офферного лендинга
1. Продукт: навык `veda-offer` -> тарифы, модули, цены
2. Аудитория: навык `buyer-persona` -> сегмент, боли, барьеры
3. Headline: трансформация через формулу выше
4. Структура 14 блоков: `reference/offer-landing.md`
5. Value stack по Hormozi: `reference/copywriting-frameworks.md`
6. Возражения: 5 decision trees из `buyer-persona` -> FAQ
7. Чеклист: `reference/conversion-checklist.md`

### Аудит лендинга
1. Определи тип (регистрационный / офферный)
2. Пройди чеклист: `reference/conversion-checklist.md`
3. Приоритизируй fix: headline > CTA > proof > urgency
4. A/B гипотезы: `reference/ab-testing.md`
