# Архитектор лендингов

## Когда использовать

- Создание лендинга регистрации на бесплатный вебинар / марафон
- Создание офферного лендинга продукта (Академия, НейроАстролог, любой инфопродукт)
- Аудит существующего лендинга на конверсию
- Написание текстов / заголовков / CTA для лендинга

## Два типа лендингов

| | Регистрационный | Офферный |
|---|-----------------|----------|
| **Цель** | Зарегистрироваться на бесплатное | Купить / оставить заявку |
| **Длина** | 1-3 экрана (короткий) | 10-14 блоков (длинный) |
| **CR warm** | 30-50% | 3-8% |
| **CR cold** | 15-25% | 0.5-2% |
| **Барьер** | "Зачем мне это?" | "Почему именно это и сейчас?" |
| **Детали** | `reference/registration-landing.md` | `reference/offer-landing.md` |

## Ключевые правила (8 нерушимых)

1. **Трансформация, не продукт.** Заголовок = кем станет человек, не что получит. "Стань экспертом" > "Курс по астрологии"
2. **Данные только из плейбуков.** Цены, модули, кейсы, имена -- из `veda-offer` и `buyer-persona`. НЕ придумывать
3. **Голос = Ирина.** Теплый наставник. "Дорогие мои". НЕ копирайтер. НЕ "девочки". НЕ длинное тире
4. **Внешний локус вины.** "Это не ваша ошибка -- причина в карте/планете/периоде". Снятие вины = разрешение купить
5. **CTA видна без скролла.** На десктопе И на мобайле. Hero = 5 элементов
6. **Proof каждые 2-3 экрана.** Кейс / отзыв / цифра / скриншот. Без proof > 3 экранов = потеря доверия
7. **Один лендинг = одно действие.** Нет навигации, нет внешних ссылок
8. **Mobile first.** 55%+ трафика с телефонов. Кнопки >= 44px, шрифт >= 16px

|--------|--------|

## Копирайтинговые фреймворки

| Фреймворк | Когда | Формула |
|-----------|-------|---------|
| **Hook-Story-Offer** (Brunson) | Универсальный скелет любого лендинга | Hook (любопытство) -> Story (трансформация) -> Offer |
| **PAS** | Болевой вход (основной для астро) | Problem -> Agitate -> Solution |
| **BAB** | Трансформационный продукт | Before -> After -> Bridge (продукт = мост) |
| **Proof-Promise-Plan** (Hormozi) | Оффер / value stack | Proof ПЕРВЫМ -> Promise -> Plan |
| **VoC / Message Mining** (Wiebe) | Тексты и заголовки | Формулировки из слов реальных клиентов (9520 анкет!) |

Детали и примеры для астро-ниши: `reference/copywriting-frameworks.md`

## Юридическая безопасность (баланс конверсия / риск)

Триггеры и сильные формулировки = ОК, КРОМЕ двух зон:

| Зона | Что НЕЛЬЗЯ | Что МОЖНО |
|------|-----------|-----------|
| **Гарантия денег** | "Заработаете 100К", "окупите за месяц", "гарантируем доход" | "Наши ученицы зарабатывают от X" (кейс, не обещание). "Потенциал X разборов = Y руб" |
| **Сверхрезультаты** | "Излечивает", "решает все проблемы", "100% результат" | "Помогает разобраться", "дает инструмент", "ученицы отмечают" |

**Правило:** кейсы = факт ("Ольга заработала 108K"), обещания = потенциал ("инструмент, с которым ученицы зарабатывают"). Между "она заработала" и "вы заработаете" -- юридическая пропасть.

**Disclaimer под кейсами:** "Результаты учениц индивидуальны и зависят от вложенных усилий."

**Цель:** < 2 секунд TTFB (каждая секунда задержки = -7% конверсии).

- [ ] Все изображения в WebP (не PNG/JPG) — экономия 60-90%

## Интеграция с GetCourse (регистрация + мессенджеры)

### Как работает связка "лендинг → GC → мгновенный процесс → бот"

```
Наш лендинг (vedantica.ru)
  ├─ Кастомная тёмная форма (телефон + email)
  ├─ POST /api/lead → серверный сабмит GC-виджета:
  │    1. addUser (тег pna_reg)
  │    2. fetch HTML виджета → парсинг form action + CSRF + cookies
  │    3. POST формы → GC создаёт заказ + МГНОВЕННО запускает процесс
  │    4. Из JSON-ответа → invitationHash → персональные TG/Max ссылки
  └─ TY экран "Остался 1 шаг" с кнопками TG/Max (хеши из виджета)
       └─ Клик → бот → GC привязывает мессенджер + процесс уже запущен → приветствие
```

**КРИТИЧНО:** GC запускает процесс мгновенно ТОЛЬКО из формы/виджета. Deals API и событийные триггеры = 10-15 мин задержка. Подтверждено тех.лидом GC и тестами 09.04.2026.

### Что нужно для нового лендинга с GC-регистрацией

**В GC (делает техник):**
1. Создать **продукт** (например, "ПНА Акшая Трития")
2. Создать **бесплатное предложение** (цена = 0) внутри продукта
3. Создать **виджет заказа** (НЕ подписную форму!)
   - Привязать предложение
   - Поля: email, имя, телефон
   - Тег: нужный (например, `pna_reg`)
   - **Обработчик формы** ("Что делать после заполнения"): "Запустить процесс" с приветственным сообщением
   - Части формы: добавить "Кнопки входа через соцсети" (TG + Max)
4. Получить **ID виджета**

**В коде (делает разработчик):**
1. `/api/lead/route.ts` — единый endpoint: addUser + серверный сабмит виджета + извлечение хешей
2. Компонент формы: после сабмита показывает TY с хеш-ссылками из ответа API

### Ключевые параметры (копировать и менять)

```typescript
// app/api/lead/route.ts
const GC_WIDGET_ID = '1588448'  // ID виджета из GC
const GC_OFFER_ID = '8270763'   // ID предложения
const GC_BASE = 'https://course.vedantica.ru'

// invitationHash из JSON ответа GC:
// json.data.preParts[].html → /"invitationHash"\s*:\s*"([a-f0-9]+)"/
// → t.me/Vedantica_bot?start={hash}__s4
```

### Типы виджетов GC (НЕ путать!)

| Тип | Когда | Создаёт | Мессенджер-кнопки |
|-----|-------|---------|-------------------|
| **Виджет заказа** | Регистрация на событие, покупка | Заказ + контакт | ДА (на странице спасибо) |
| **Подписная форма** | Сбор email в рассылку | Только контакт с тегом | НЕТ |
| **API addUser** | Программная регистрация | Только контакт | НЕТ (нет хешей) |

**Для мессенджер-кнопок с авто-привязкой нужен ВИДЖЕТ ЗАКАЗА** — подписная форма и API их не дают.

### GC-боты для привязки мессенджеров Vedantica

| Мессенджер | Бот | Для чего |
|-----------|-----|---------|

| ~~@teamchayka_bot~~ | Квиз/диагностики | **НЕ использовать для GC-регистрации** |

### Что НЕ работает (проверено, не повторять)

- **GC Deals API для мгновенного процесса** — создаёт заказ, но процесс стартует через 10-15 мин
- **GC API addUser** — создаёт контакт, но НЕ генерирует хеш. Нельзя получить персональную ссылку на бота
- **Событийные триггеры GC** (добавлен в группу, создан заказ, подключился к боту) — всё с задержкой 10-15 мин
- **`/pl/api/processes`** — endpoint НЕ существует в GC API
- **GC-страницы в iframe** — `X-Frame-Options: DENY`
- **GC lite-виджет напрямую в React SPA** — скрипт ломает SPA, белая форма, redirect. НЕ встраивать

| **Трансформационная идентичность** | Покупают новую версию себя | Hero, Dream section |
| **Предназначение** | "Это мое призвание" > "хочу зарабатывать" | Problem, Bridge |
| **Снятие вины** | "Не твоя ошибка -- карма/планеты" | Problem, Agitate |
| **Разрешение** | "Мне можно потратить на себя" | CTA, Guarantee |
| **Кейс = "такая же как ты"** | Совпадение по ситуации | Social proof |
| **Эксклюзивность** | Закрытое сообщество, ограниченный поток | Urgency |

## Формула покупки

```
Покупка =
  Боль (узнала себя)
  + Снятие вины (не моя ошибка -- карма)
  + Разрешение (мне можно потратить на себя)
  + Снятие риска (рассрочка / гарантия / возврат)
```

## Hormozi Value Equation (для оффера)

```
Ценность = (Dream Outcome x Вероятность достижения)
         / (Время до результата x Усилие)

Задача лендинга:
  Числитель ↑ = яркий результат (headline) + proof (кейсы, цифры)
  Знаменатель ↓ = быстро (сроки) + легко (формат, поддержка)
```

## Быстрый маршрут

### Создание регистрационного лендинга (вебинар/марафон)
1. Определи мероприятие: вебинар / марафон / мастер-класс
2. Сегмент: навык `buyer-persona` -> A1/A2/B1/B2
3. Headline: конкретный результат (не тему!) через формулу выше
4. Структура 8 блоков: `reference/registration-landing.md`
5. Форма: максимум 2-3 поля
6. Social proof: 2-3 отзыва из плейбука
7. Urgency: реальный дедлайн / лимит мест
8. Чеклист: `reference/conversion-checklist.md`

### Создание офферного лендинга
1. Продукт: навык `veda-offer` -> тарифы, модули, цены
2. Аудитория: навык `buyer-persona` -> сегмент, боли, барьеры
3. Headline: трансформация через формулу выше
4. Структура 14 блоков: `reference/offer-landing.md`
5. Value stack по Hormozi: `reference/copywriting-frameworks.md`
6. Возражения: 5 decision trees из `buyer-persona` -> FAQ
7. Чеклист: `reference/conversion-checklist.md`

### Аудит лендинга
1. Определи тип (регистрационный / офферный)
2. Пройди чеклист: `reference/conversion-checklist.md`
3. Приоритизируй fix: headline > CTA > proof > urgency
4. A/B гипотезы: `reference/ab-testing.md`
