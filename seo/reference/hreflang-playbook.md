# Hreflang Playbook

## Когда открывать

- Есть несколько языков, стран или доменов
- Планируется международная структура URL
- Нужно проверить `ru` / `en` / региональные версии без дублирования

## Что проверить сначала

```text
□ каждая версия страницы имеет self-canonical
□ hreflang не конфликтует с canonical
□ у каждой пары есть reciprocal links
□ языковые и региональные коды валидны
□ есть x-default там, где нужен глобальный fallback
□ HTTP/HTTPS и www/non-www не смешаны
```

## Модель

- `canonical` отвечает за основную версию конкретной страницы.
- `hreflang` отвечает за языковую/региональную альтернативу.
- Они не должны спорить друг с другом.

## Типовые ошибки

- `hreflang` указывает на URL, который каноникалится в другую страницу
- одна из альтернатив отсутствует или закрыта `noindex`
- есть `en-EN`, `ru-RU` и другие невалидные или бессмысленные коды
- локали смешаны между поддоменами и поддиректориями без единой модели

## Для VEDA

- Не вводить hreflang, пока нет устойчивого контентного процесса для второй локали.
- Русский рынок основной: сначала сильный `ru`, потом international expansion.
- Если английская версия машинная и слабая, лучше не индексировать её как полноценную альтернативу.
