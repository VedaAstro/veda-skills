# Visual QA Playbook

## Когда открывать

- Нужна визуальная SEO-проверка первого экрана
- Нужно проверить mobile/desktop рендер и above-the-fold
- Есть риск, что HTML нормальный, а реальный экран ломает CTR и engagement

## Что проверить

### Desktop

```text
□ H1 виден на первом экране
□ основной смысл страницы понятен без скролла
□ CTA или следующий шаг виден
□ нет перекрытий, обрезаний, layout shift на старте
```

### Mobile

```text
□ нет горизонтального скролла
□ текст читаем без zoom
□ CTA и ключевой контент доступны
□ меню/навигация не ломают first screen
□ touch targets не микроскопические
```

## Инструменты

- Используй Playwright snapshot/screenshots, если доступен.
- Если браузерного рантайма нет, не выдумывай визуальные выводы по одному HTML.

## Для SEO

Визуальный QA нужен не ради “красоты”, а ради:
- snippet satisfaction после клика
- поведенческого качества
- first-screen comprehension
- мобильной пригодности коммерческих и UGC-страниц
