# Competitor Pages Playbook

## Когда открывать

- Нужно сделать `X vs Y`
- Нужно сделать `alternatives to X`
- Нужно закрыть comparison intent без рискованного SEO-мусора

## Цель

Поймать comparison intent и довести до конверсии без кликбейтного или недостоверного контента.

## Минимальная структура

- прямой answer-first блок: кому и в каком сценарии подходит наш продукт
- таблица сравнения по 5-8 реальным критериям
- блок "когда выбрать нас"
- блок "когда выбрать не нас"
- FAQ
- CTA

## Правила

- сравнивать только по проверяемым критериям
- не рисовать фальшивые рейтинги и вымышленные цифры
- явно разделять факт, интерпретацию и маркетинговое мнение
- использовать `Product` / `Service` / `FAQPage` / `BreadcrumbList` schema только если соответствует странице

## Красные флаги

- 50 страниц `X vs Y` без реального знания продукта Y
- один шаблон без уникальных сценариев выбора
- страницы существуют только ради long-tail, без конверсионной роли

## Для VEDA

- comparison pages работают лучше для:
  - `натальная карта онлайн vs консультация астролога`
  - `ведическая астрология vs западная`
  - `AstroGuru vs [альтернатива]`
- не использовать агрессивный негатив против конкурента; это бьёт по trust.
