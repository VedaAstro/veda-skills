# Programmatic SEO Playbook

## Когда открывать

- Планируются десятки и сотни SEO-страниц
- Есть шаблонный генератор, база данных или AI batch publishing
- Нужно решить, какие страницы индексировать, а какие нет

## Цель

Построить масштабируемые страницы без thin content, cannibalization и index bloat.

## Обязательный порядок

1. Проверить `search-policies.md`.
2. Определить query cluster и тип страницы.
3. Описать источник уникальной ценности на страницу.
4. Спроектировать internal linking и sitemap coverage.
5. Поставить quality gates до публикации батча.

## Quality gates

```text
□ у каждой страницы есть самостоятельный answer-first блок
□ различия между страницами не сводятся к замене 1-2 сущностей
□ есть уникальные данные, примеры, FAQ, proof или UGC
□ есть self-canonical
□ есть место в навигации и связях сайта
□ есть понятное правило noindex для слабых/пустых страниц
□ batch не публикуется без sample review первых 20 страниц
```

## Indexation model

- Индексировать:
  - страницы с завершённым ответом на запрос
  - страницы с уникальными данными и внутренней ролью в архитектуре
- Не индексировать:
  - пустые заготовки
  - страницы с одним шаблоном и без дополнительной ценности
  - низкоспросовые комбинации без смысла для пользователя

## VEDA-специфика

- Для астрологических комбинаций нужен не только шаблон трактовки, но и:
  - сценарии проявления
  - коррекции/рекомендации
  - FAQ
  - переходы в калькулятор, диагностику, community
- Для UGC/community индексировать только mature threads, а не весь поток.

## IndexNow интеграция

При деплое батча programmatic страниц — автоматический POST в IndexNow:
```
POST https://yandex.com/indexnow
Content-Type: application/json
{
  "host": "app.myveda.ru",
  "key": "<indexnow-key>",
  "urlList": ["https://app.myveda.ru/...", ...]
}
```

- Поддерживают: Яндекс, Bing, DuckDuckGo
- Google НЕ поддерживает — для Google используй sitemap + GSC
- Один POST уведомляет все поддерживающие движки
- Скрипт: `scripts/notify-indexnow.ts`, вызывается после deploy

## Tier-система индексации

Не все programmatic страницы заслуживают индексации.

| Tier | Критерий | Действие |
|------|---------|----------|
| 1 | answer-first 200+ слов, общий контент 1200+ | `index` — всегда |
| 2 | 800-1200 слов | `index` — мониторить поведенческие |
| 3 | < 800 слов или answer-first < 200 | `noindex` — доработать |

Реализация: условный `<meta name="robots" content="noindex">` в шаблоне.
Лучше 350 сильных страниц в индексе, чем 400 с тонкими хвостами.

## noindex policy

Обязательно иметь автоматическое правило:
- Если контент < 800 слов → noindex
- Если answer-first блок < 200 слов → noindex
- Если FAQ отсутствует или < 3 вопросов → review перед индексацией

## Красные флаги

- "Сейчас нагенерим 10 000 страниц, потом разберёмся"
- один и тот же H1/title pattern без уникального первого блока
- все страницы одинаковой длины и структуры
- нет политики архивирования, noindex и удаления мусора
- нет видимого автора/бренда на странице (E-E-A-T провал)
- нет datePublished на странице (AI-search и Яндекс Нейро не цитируют)
- массовый деплой без IndexNow/sitemap (Яндекс может не заметить)
- шаблонные passages одной длины (AGC-сигнал)
