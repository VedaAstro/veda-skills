## ЧТО ЭТО ЗА ПРОЕКТ

Академия Ирины Чайки — крупнейшая школа ведической астрологии в России (10 000+ учеников). Мы создаём продающие вебинарные серии (3-5 вебов) для продажи курса «Ведическая Астрология — 7 направлений» с чеком ~100 000₽.

**Аудитория:** Женщины 28-45 лет, Россия. Ищут себя, выход из кризиса, доп.доход.
**Спикер:** Ирина Чайка (46 лет, путь от 5 000₽/мес до основателя академии).
**Метод:** Структура по Hormozi + астрология как красная нить + психология байера.

## БЫСТРЫЙ МАРШРУТ: «МНЕ НУЖНО…»

### Создать вебинар с нуля

```
1. Прочитай Playbook_Hormozi.md → §11 (операционный протокол, 8 шагов)
2. Бери структуру: Hormozi §3-4 (фазы, тайминги)
3. Бери продукт: Product §2-5 (7Н/5S, модули, тарифы)
4. Бери аудиторию: Buyer §2 (сегменты, боли)
5. Бери контент: Astro §планеты (связки для слайдов)
6. Бери голос: Speaker §4 (тон, маркеры, табу)
7. Бери упражнения: Interactive (выбери 8-12 по фильтрам)
8. Бери пруфы: Proofs (по таблице барьеров)
9. Формируй слайды: Slides §1-5 (TSV, img_tag, notes)
```

### Написать речь спикера (notes)

```
1. Speaker §4 → тон, маркеры, обращение («дорогие мои»)
2. Speaker §5 → метафоры Ирины
3. Speaker §8 → что НЕЛЬЗЯ говорить
4. Slides §0 → КРИТИЧНОЕ: льющаяся речь, не телеграф
5. Buyer §vocabulary → словарь ЦА, антисловарь
```

### Создать оффер / value stack

```
1. Product §3 → тарифы и цены (точные данные)
2. Product §5-6 → опции и сводная таблица
3. Hormozi §6 → КАК подавать (каскад, 4 блока, дефицит)
4. Buyer §5.5 → блок разрешения (ДО цен)
5. Speaker §7 → как ГОВОРИТЬ о НейроАстрологе
```

### Закрыть возражение

```
1. Buyer §3.1 + DT-1..5 → какой барьер, глубинный страх, алгоритм
2. Hormozi §7 → на каком вебе какой барьер закрывать
3. Cases §2.1 → кейс по конкретному барьеру + готовый скрипт
4. Speaker §6 → готовые цитаты Ирины по барьерам
5. Proofs → таблица «Быстрый доступ по барьерам» → скрипт с фактом
```

### Сделать диагностику / колесо жизни на вебе

```
1. Product §2.3-2.5 → 5 сфер (5S), шкалы с симптомами
2. Product §2.4 → маппинг 5S → модули 7Н
3. Hormozi §4 → фаза «практика» на вебинаре
4. Interactive → упражнения для диагностики
```

### Вставить кейс / социальное доказательство

```
1. Cases §2.1 → по барьеру (какой кейс закрывает какое возражение)
2. Cases §2.2 → по сегменту («такая же как ты» — Buyer #11)
3. Cases §2.3 → по фазе веба (прогрев / оффер / дожим)
4. Cases §4 → готовые наборы из 3-4 кейсов по задаче
5. Каждый кейс = готовый скрипт по формуле Хормози + trial close
```

### Добавить факт / пруф для легитимизации

```
1. Proofs → таблица «Быстрый доступ по барьерам» → номер факта
2. Proofs → каждый факт имеет готовый скрипт (notes)
3. Proofs → Матрица «Факт × Веб» → какие факты на каком вебе
4. Hormozi §8.4 → правило: proof point каждые 5-7 мин
```

### Выбрать интерактивное упражнение

```
1. Interactive → сводная таблица → фильтры: тип веба, фаза, сегмент
2. Interactive → готовые наборы по типам вебов
3. Hormozi §9.4 → правило: 1 упражнение на 10 мин контента
```

## КЛЮЧЕВЫЕ ПРАВИЛА (запомни навсегда)

### Астрология = красная нить
Все смыслы доносятся ЧЕРЕЗ астрологический инструментарий, а не рядом с ним. Астрология — не тема вебинара, а его ядро.

### Голос = Ирина
- Обращение: «дорогие мои», «дорогие». НИКОГДА: «девочки», «красотки», «подружки»
- Тон: тёплый наставник. НЕ рекламный копирайтер
- Маркеры: «вот», «на самом деле», «согласны?», «откликается?»

### Внешний локус
«Это не ваша вина» — причина в карте, планете, периоде. Не «ты виновата».

### Веб начинается с программы
Первые слайды = программа веба + бонус за досиживание. НЕ сразу боль.

### Речь льётся
Колонка `notes` (речь спикера) = живая, связная речь. Каждое предложение связано со следующим. Нет обрубков. Нет телеграмм. (→ Slides §0)

### Netflix-крючки
Между КАЖДЫМИ блоками — 30-секундный крючок-интрига. Каждый крючок закрыть в следующем блоке.

### Proof каждые 5-7 мин
Без доказательства утверждение = голословное. Кейс, статистика, калибровка, демонстрация.

### Нет дублей между плейбуками
Каждый плейбук = слой. Если нужен продукт — иди в Product. Если голос — в Speaker. Не копируй содержание из одного в другой.
