# Проектирование курсов: модели и таксономия

## Матрица типов продуктов

| Тип | Длина | Формат | Глубина |
|-----|-------|--------|---------|
| Мини-курс | 3–7 уроков | видео + задания | поверхностная |
| Онлайн-курс | 4–12 модулей | видео + материалы + практика | средняя |
| Флагманская программа | 3–6 месяцев | живые встречи + курс + коучинг | глубокая |
| Мастермайнд | 6–12 мес | группа + индивидуальное | трансформационная |
| Воркшоп/интенсив | 1–5 дней | практика + теория | концентрированная |
| Марафон | 7–21 день | ежедневные задания | поведенческая |

```
ЕСЛИ цель = «познакомиться с темой» → мини-курс или воркшоп
ЕСЛИ цель = «освоить навык» → курс с практическими заданиями
ЕСЛИ цель = «трансформация» → флагманская программа + сопровождение
ЕСЛИ цель = «поддержание практики» → марафон или мастермайнд
```

## ADDIE — классическая модель

**A — ANALYSE (Анализ)**
- Кто аудитория? Боли, мечты, уровень знаний
- Разрыв: что знают сейчас → что должны знать/уметь
- Ограничения: время, формат, технические возможности

**D — DESIGN (Дизайн)**
- Цели обучения по Блуму (уровень и домен)
- Структура курса: модули, уроки, последовательность
- Стратегия оценки
- Форматы контента

**D — DEVELOP (Разработка)**
- Создание контента модуль за модулем
- Материалы поддержки (PDF, рабочие тетради)
- Задания и оценочные инструменты

**I — IMPLEMENT (Внедрение)**
- Загрузка на платформу
- Пилотный запуск с малой группой
- Техническое тестирование

**E — EVALUATE (Оценка)**
- Уровень 1: Реакция (доволен ли студент?)
- Уровень 2: Обучение (усвоены ли знания/навыки?)
- Уровень 3: Поведение (применяет ли в жизни?)
- Уровень 4: Результаты (достигнуты ли цели бизнеса?)

## SAM — итеративная модель (для быстрых запусков)

```
Фаза 1: Подготовка → сбор информации → первый прототип → оценка
Фаза 2: Итеративный дизайн → дизайн → прототип → обзор (3–5 итераций)
Фаза 3: Итеративная разработка → альфа → обзор → бета → финал
```

**Когда SAM**: MVP-запуск, живые программы с ведущим, нужна быстрая обратная связь.
**Когда ADDIE**: масштабные программы, асинхронные курсы, корпоративное обучение.

## Модель разрыва в обучении (Dirksen)

```
ТЕКУЩЕЕ СОСТОЯНИЕ ──→ [РАЗРЫВ] ──→ ЦЕЛЕВОЕ СОСТОЯНИЕ

Тип разрыва → Метод:
1. Знания («не знаю») → контент, лекции
2. Навыки («не умею») → практика, повторение
3. Мотивация («не хочу») → истории, ценности, смыслы
4. Среда («условия не позволяют») → изменение внешних условий
```

**КРИТИЧНО**: правильно диагностируй тип. Разрыв мотивации не лечится лекциями.

## Таксономия Блума — когнитивный домен

| Уровень | Суть | Глаголы для целей обучения |
|---------|------|---------------------------|
| 1. Запомнить | Воспроизвести факты | назвать, перечислить, определить, описать |
| 2. Понять | Объяснить смысл | объяснить, интерпретировать, классифицировать |
| 3. Применить | Использовать в новых ситуациях | применить, рассчитать, выполнить, продемонстрировать |
| 4. Анализировать | Разбить на части, найти связи | проанализировать, сравнить, различить |
| 5. Оценить | Вынести суждение | оценить, обосновать, критиковать, выбрать |
| 6. Создать | Создать что-то новое | разработать, составить, спроектировать, создать |

## Аффективный домен (для коучинга/психологии)

| Уровень | Суть | Пример |
|---------|------|--------|
| Восприятие | Открыт к новому | Слышит идею без отторжения |
| Реагирование | Активно участвует | Выполняет практики |
| Оценивание | Принимает ценность | «Эта практика важна для меня» |
| Организация | Интегрирует в систему | Объединяет новые и старые убеждения |
| Характеризация | Живёт по новым ценностям | Новое поведение стало привычным |

## Формат цели обучения

```
[Студент] + [глагол Блума] + [объект] + [критерий/контекст]

✓ «По завершении модуля студент СОСТАВИТ личную астро-карту на год, используя 3 основных транзита»
✓ «Студент ПРИМЕНИТ технику работы с сопротивлением в реальной коуч-сессии»
✗ «Студент поймёт андрагогику» — нет наблюдаемого поведения
✗ «Студент познакомится с инструментами» — слабый глагол, нет критерия
```

## Выравнивание (Alignment) — золотое правило

```
Цель → Контент → Активности → Оценка (все согласованы)

Пример:
Цель: «Создаст план работы с клиентом на 3 месяца»
  ↓ Контент: методы составления планов, примеры
  ↓ Активность: составить план для условного клиента
  ↓ Оценка: чеклист качества плана (5 критериев)
```
