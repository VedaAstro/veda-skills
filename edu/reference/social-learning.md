# Peer-learning, community, accountability

> Этот файл покрывает слой 4 (социальный) из модели удержания.
> Открывай, когда нужно спроектировать peer-взаимодействие, community, accountability-механики.

---

## §1. Социальное обучение — почему это работает

### 1.1 Теоретическая база

```
БАНДУРА (Social Learning Theory):
  Люди учатся через наблюдение за другими + моделирование поведения.
  → Реализация: демо-сессии, кейсы студентов, peer-shаring

ВЫГОТСКИЙ (Zone of Proximal Development):
  Человек может сделать с помощью более опытного то, что не может один.
  → Реализация: peer-менторство, buddy-система, групповые задания

ЛЕЙВ & ВЕНГЕР (Communities of Practice):
  Обучение = постепенное вхождение в сообщество практиков.
  → Реализация: community, наставничество, стажировка

АШЕР (Social Presence Theory):
  Чем сильнее ощущение присутствия других — тем выше вовлечённость.
  → Реализация: live-элементы, видео-обратная связь, реальные имена и фото
```

### 1.2 Влияние социального слоя на метрики

```
МЕТРИКА                    БЕЗ СОЦСЛОЯ    С СОЦСЛОЕМ    ПРИРОСТ
Completion rate            40-50%         65-80%         +25-30 п.п.
NPS                        50-60          70-85          +20-25
Повторная покупка          15-20%         30-45%         +15-25 п.п.
Скорость прохождения       медленнее      быстрее        ~20%
Глубина усвоения           поверхностная  глубокая       значительно
```

## §2. Механики peer-learning — каталог

### 2.1 Buddy-система (пара)

```
ЧТО: Двое студентов объединяются в пару на весь курс / модуль.

ФОРМАТ:
• Регулярные check-in (1-2 раза в неделю, 15-30 мин)
• Обмен заданиями для peer-review
• Совместная практика (роль клиент/консультант)
• Взаимная поддержка в чате

ПРАВИЛА ФОРМИРОВАНИЯ ПАР:
• Близкий уровень знаний (разрыв max 1 уровень)
• Разный опыт (обогащают друг друга)
• Общий часовой пояс (±2 часа)
• Ротация каждые 2-4 недели (если программа длинная)

КОГДА:
• Курсы по навыкам (коучинг, консультирование)
• Корпоративное обучение (buddy = ментор)
• Длинные программы (3+ месяцев)

АНТИПАТТЕРН:
• Пара без структуры → «мы не знаем, о чём говорить» → перестают встречаться
• Решение: дать agenda для каждой встречи (3-5 вопросов для обсуждения)
```

### 2.2 Малые группы (3-5 человек)

```
ЧТО: Устойчивая группа на весь курс для совместной работы.

ФОРМАТЫ ГРУППОВОЙ РАБОТЫ:
• Мастермайнд: каждый по очереди представляет задачу, остальные дают feedback
• Case study: группа анализирует один кейс, каждый с разной ролью
• Проект: совместный финальный продукт
• Hot seat: один участник — в фокусе, остальные — coaches/observers

ПРАВИЛА:
• Чёткие роли (лидер, таймкипер, note-taker — ротируются)
• Фиксированное расписание встреч
• Структурированная agenda (не «просто поболтаем»)
• Групповая ответственность (результат группы = результат каждого)

ОПТИМАЛЬНЫЙ РАЗМЕР:
• 3 человека: глубокое взаимодействие, но хрупко (один выпал — пара)
• 4-5 человек: устойчиво, разнообразие перспектив
• 6+: слишком большая — теряется персональность

КОГДА:
• Групповые программы с live-элементом
• Трансформационные программы (безопасное пространство)
• Проектные курсы
```

### 2.3 Peer-review (взаимная оценка)

```
ЧТО: Студенты оценивают работы друг друга по заданным критериям.

ПРЕИМУЩЕСТВА:
• Развивает критическое мышление (оценка = обучение)
• Снимает нагрузку с куратора
• Даёт множество перспектив
• Учит давать и принимать feedback

УСЛОВИЯ РАБОТЫ:
• Чёткие критерии оценки (чеклист / rubric) — обязательно
• Анонимность ИЛИ психологическая безопасность — одно из двух
• Обучение peer-review: показать пример хорошего и плохого отзыва
• Модерация: куратор проверяет качество отзывов

ШАБЛОН PEER-REVIEW:
  □ Что сделано хорошо (конкретно, min 2 пункта)
  □ Что можно улучшить (конкретно, min 1 пункт)
  □ Соответствие критериям (чеклист из задания)
  □ Общая рекомендация (принять / доработать)

КОГДА:
• Задания с открытым ответом (эссе, проект, план, стратегия)
• Навыковые курсы (разбор записей сессий)
• Группы 10+ человек (где куратор физически не может дать feedback всем)
```

### 2.4 Teach-back (обучение других)

```
ЧТО: Студент объясняет пройденный материал другому студенту или группе.

ПОЧЕМУ ЭФФЕКТИВНО:
• Пирамида обучения: teach others = 90% retention (vs lecture = 5%)
• Выявляет пробелы: не можешь объяснить → не понял
• Укрепляет уверенность (я эксперт → компетентность по SDT)

ФОРМАТЫ:
• Mini-presentation: 5 мин доклад в группе
• Объясни новичку: продвинутый студент → начинающий
• Запись короткого видео-объяснения
• Пост в чате: «объясните тему своими словами»

КОГДА:
• После ключевых теоретических блоков
• Подготовка к сертификации / экзамену
• В buddy-парах (по очереди объясняют друг другу)
```

## §3. Community — проектирование сообщества

### 3.1 Структура community

```
ЯДРО (Core)
  Кто: выпускники, активные студенты, менторы
  Роль: создают контент, помогают новичкам, задают культуру
  Размер: 5-10% от общей численности
  Инструменты: отдельный канал, модераторские права, exclusive-контент

АКТИВНЫЕ УЧАСТНИКИ (Active)
  Кто: текущие студенты, вовлечённые в процесс
  Роль: задают вопросы, делятся опытом, участвуют в дискуссиях
  Размер: 20-30%
  Инструменты: общий чат, тематические каналы, мероприятия

НАБЛЮДАТЕЛИ (Lurkers)
  Кто: читают, но не пишут
  Роль: потенциальные активные — нужен триггер для вовлечения
  Размер: 50-60%
  Инструменты: реакции, опросы, анонимные вопросы
  Стратегия: давать «безопасные» точки входа (голосования, реакции, анонимные вопросы)

НЕАКТИВНЫЕ (Inactive)
  Кто: зарегистрированы, но не заходят
  Размер: 10-20%
  Стратегия: → churn-prevention.md §4 (реактивация)
```

### 3.2 Ритуалы community

```
ЕЖЕДНЕВНЫЕ:
• Morning check-in: «Какая цель на сегодня?»
• Вопрос дня (от куратора или бота)
• Шеринг micro-win: «Что получилось сегодня?»

ЕЖЕНЕДЕЛЬНЫЕ:
• Weekly win: лучший результат / прогресс недели
• Q&A live (30-60 мин)
• Тема недели: обсуждение в чате
• Challenge недели: мини-задание + общий результат

MONTHLY:
• Celebration: отмечаем достижения месяца
• Guest-expert (если актуально)
• Retrospective: что работает / что улучшить
• Showcase: студенты показывают свои проекты

EVENT-DRIVEN:
• Welcome-ритуал для новых участников
• Graduation / completion ceremony
• Alumni meetup (для выпускников)
```

### 3.3 Модерация и культура

```
ПРАВИЛА ЗДОРОВОГО COMMUNITY:

1. CODE OF CONDUCT (правила сообщества)
   • Уважение, конструктивность, конфиденциальность
   • Простые, на 1 экран
   • Подтверждение при вступлении

2. ТОНАЛЬНОСТЬ
   • Задаётся ядром + куратором
   • Позитивная, поддерживающая, без критиканства
   • Куратор моделирует желаемое поведение

3. МОДЕРАЦИЯ
   • Реагирование на токсичность (в течение часов, не дней)
   • Redirect: если оффтопик → «отличный вопрос, давай в канал [X]»
   • Запрет: персональные нападки, спам, продажи

4. CONTENT SEEDING (засев контента)
   • Первые 2 недели: куратор / ядро создаёт 80% контента
   • Потом: постепенная передача инициативы участникам
   • Если тишина > 24ч → куратор инициирует обсуждение
```

## §4. Accountability — системы ответственности

### 4.1 Уровни accountability

```
УРОВЕНЬ 1: САМООТВЕТСТВЕННОСТЬ (Self)
  Инструменты: трекер привычек, дневник, чеклист, personal deadline
  Эффективность: низкая (30-40% выполнения обязательств)
  Когда: self-paced курсы, продвинутые студенты

УРОВЕНЬ 2: PEER-ОТВЕТСТВЕННОСТЬ (Buddy/Group)
  Инструменты: buddy check-in, групповые дедлайны, публичные обязательства
  Эффективность: средняя (60-70%)
  Когда: групповые программы, марафоны

УРОВЕНЬ 3: ВНЕШНЯЯ ОТВЕТСТВЕННОСТЬ (Coach/Curator)
  Инструменты: проверка заданий, 1-on-1 check-in, персональный feedback
  Эффективность: высокая (80-90%)
  Когда: флагманские программы, корпоративное обучение с руководителем

УРОВЕНЬ 4: ПУБЛИЧНАЯ ОТВЕТСТВЕННОСТЬ (Public)
  Инструменты: публичный трекер, шеринг в соцсетях, публичный commitment
  Эффективность: высокая для экстравертов, рискованная для интровертов
  Когда: марафоны, challenge-программы
```

### 4.2 Публичный commitment — механика

```
ПРИНЦИП (Cialdini — Commitment & Consistency):
  Люди стремятся быть последовательными после публичного обязательства.

РЕАЛИЗАЦИЯ:
• В начале курса: «Напишите в чат свою цель» (публичный commitment)
• Еженедельно: «Что вы обязуетесь сделать на этой неделе?»
• Visual: доска обязательств (Trello, Miro, стикеры в чате)
• Check-in: «Кто выполнил обязательство прошлой недели? Поделитесь!»

ПРАВИЛА:
• Обязательство должно быть конкретным и достижимым
• Не наказывать за невыполнение — это убивает психологическую безопасность
• Праздновать выполнение (social recognition)
• Дать возможность скорректировать обязательство (не «сделал / не сделал»)
```

### 4.3 Social proof в обучении

```
ТИПЫ SOCIAL PROOF:

PROGRESS PROOF
  «67% вашей группы уже сдали задание 3»
  Эффект: нормализация + urgency

SUCCESS STORIES
  «Ваша однокурсница [имя] уже применила метод и получила [результат]»
  Эффект: «если она смогла — значит, и я смогу»

EXPERT ENDORSEMENT
  «Этот метод используют в [авторитетная организация]»
  Эффект: доверие к контенту

QUANTITY PROOF
  «10 000+ студентов уже прошли этот курс»
  Эффект: снижение сомнений

PEER PROOF (самый мощный в контексте обучения)
  «Посмотрите, как [имя] решил это задание» (с разрешения)
  Эффект: моделирование (Бандура) + норма группы
```

## §6. Self-check: социальный слой

```
□ Определён уровень accountability (1-4) для данного продукта?
□ Выбраны peer-механики, соответствующие формату и размеру группы?
□ Community имеет структуру (ядро, активные, наблюдатели)?
□ Есть ритуалы (ежедневные / еженедельные / monthly)?
□ Code of conduct определён и прост?
□ Content seeding спланирован на первые 2 недели?
□ Social proof встроен в key moments (onboarding, milestone, плато)?
□ Наблюдатели (lurkers) имеют «безопасные» точки входа?
□ Buddy/группы сформированы по правилам (уровень, timezone, ротация)?
□ Peer-review обеспечен чеклистом / rubric?
```
