# Андрагогика и когнитивная наука для удержания

> Этот файл покрывает слои 1 (когнитивный) и 2 (мотивационный) из модели удержания.
> Открывай, когда проблема — в усвоении материала или потере мотивации.

---

## §1. Когнитивный слой: как мозг усваивает

### 1.1 Три памяти — три задачи дизайнера

| Память | Объём | Задача дизайнера |
|--------|-------|-----------------|
| **Сенсорная** | доли секунды | Захватить внимание (hook, surprise, конфликт) |
| **Рабочая** | 4±1 чанка, минуты | Не перегружать (один концепт за раз, chunking) |
| **Долгосрочная** | без ограничений | Обеспечить перенос (повторение, применение, связи) |

### 1.2 Перенос в долгосрочную память (5 механизмов)

```
1. SPACED REPETITION — интервальные повторения
   Оптимальные интервалы: 1 день → 3 дня → 7 дней → 21 день → 60 дней
   Реализация: recap в начале урока, quiz через неделю, интеграционное задание через месяц

2. RETRIEVAL PRACTICE — извлечение из памяти
   Самотестирование в 3 раза эффективнее перечитывания
   Реализация: quiz, flashcards, «вспомни без подсказки», «объясни другому»

3. ELABORATION — углубление через объяснение
   «Почему это так?», «Как это связано с тем, что ты уже знаешь?»
   Реализация: рефлексивные вопросы, peer-объяснения, teach-back

4. INTERLEAVING — чередование типов задач
   Не 10 однотипных задач подряд, а разные типы вперемешку
   Реализация: смешивай форматы в одном модуле

5. DUAL CODING — двойное кодирование
   Визуал + текст/аудио одновременно (но НЕ текст + аудио того же текста)
   Реализация: схемы + объяснение голосом, инфографика + краткий текст
```

### 1.3 Когнитивная нагрузка — управление

```
ТИПЫ НАГРУЗКИ:
├── Intrinsic (сложность самого материала)    → управляй через chunking и последовательность
├── Extraneous (шум, плохой дизайн)           → убирай всё лишнее
└── Germane (формирование новых схем)         → это и есть обучение — поддерживай

ПРАКТИЧЕСКИЕ ПРАВИЛА:
• Один концепт за раз (не два параллельно)
• Контекст → концепт → пример → практика (всегда в этом порядке)
• Знакомые метафоры для нового (снижает intrinsic load)
• Убирай декоративные элементы (снижает extraneous load)
• Видео max 10 мин (правило Медины — после 10 мин внимание падает на 50%)
• Если материал объективно сложный — дроби на микро-порции + scaffolding
```

### 1.4 Scaffolding (леса) — постепенное снятие поддержки

```
ЭТАП 1: Полная поддержка
  → Демонстрация, пошаговая инструкция, шаблон с заполненным примером

ЭТАП 2: Частичная поддержка
  → Шаблон с пустыми полями, подсказки, чеклист

ЭТАП 3: Минимальная поддержка
  → Открытое задание с критериями оценки

ЭТАП 4: Самостоятельность
  → Реальная задача без шаблона
```

**Ошибка**: бросить студента на этапе 4 без прохождения этапов 1-3. Типичный симптом: «задание слишком сложное, я не справляюсь».

### 1.5 Chunking — правила дробления

```
ПРАВИЛО 7±2 (Miller): рабочая память удерживает 4-7 элементов одновременно

СЛЕДСТВИЯ ДЛЯ ДИЗАЙНА:
• Модуль = 3-5 уроков (не 12)
• Урок = 1 ключевой концепт + 2-3 поддерживающие идеи
• Слайд = 1 мысль
• Список = max 5-7 пунктов (группируй, если больше)
• Видео = 5-10 мин сегмент с одной идеей

ГРУППИРОВКА:
Если нужно передать 15 элементов → сгруппируй в 3-4 категории по 4-5 элементов.
Каждая категория = один чанк в рабочей памяти.
```

## §2. Мотивационный слой: почему человек продолжает

### 2.1 Self-Determination Theory (Deci & Ryan) — три базовые потребности

```
АВТОНОМИЯ                    КОМПЕТЕНТНОСТЬ               СВЯЗНОСТЬ
«Я сам решаю»               «Я справляюсь»               «Я часть чего-то»
                             
Реализация:                  Реализация:                  Реализация:
• Выбор порядка модулей      • Нарастающая сложность      • Групповые задания
• Выбор формата задания      • Quick wins в начале         • Peer-feedback
• «Вы можете» вместо         • Ясные критерии успеха      • Общий чат / форум
  «Вы должны»               • Обратная связь после        • Совместные проекты
• Несколько путей через        каждого задания             • Ритуалы группы
  материал                   • Видимый прогресс            • Шеринги опыта
• Гибкие дедлайны            • Сертификаты / бейджи       • Менторство

ДИАГНОСТИКА:
Автономия нарушена → «Меня заставляют», «Не мой темп», «Не моя тема»
Компетентность нарушена → «Слишком сложно», «Я не справлюсь», «Не понимаю»
Связность нарушена → «Одиноко», «Никому нет дела», «Зачем стараться»
```

### 2.2 Модель ARCS (Keller) — мотивационный дизайн

```
A — ATTENTION (внимание)
    Захвати и удержи.
    Инструменты: surprise, конфликт, вопрос, история, демо проблемы
    Когда: начало каждого урока/модуля, после перерыва, при смене темы
    Частота: каждые 8-10 минут — новый стимул внимания

R — RELEVANCE (актуальность)
    Сделай важным лично для студента.
    Инструменты: «Как это применишь завтра», кейсы из жизни ЦА, персонализация
    Когда: до начала каждого блока (принцип Need to Know)
    Проверка: если студент не может ответить «зачем мне это» → блок не пройдёт

C — CONFIDENCE (уверенность)
    Студент верит, что справится.
    Инструменты: ясные цели, достижимые задания, нарастающая сложность, ранние победы
    Когда: при каждом задании, особенно при повышении сложности
    Красный флаг: если >20% студентов не справляются → задание слишком сложное

S — SATISFACTION (удовлетворение)
    Награда за усилие.
    Инструменты: обратная связь, признание, реальный результат, внутреннее «я сделал это»
    Когда: после каждого задания, в конце модуля, при milestone
    Правило: внутренняя мотивация > внешняя (но внешняя работает как стартер)
```

### 2.3 Шесть принципов андрагогики Knowles (через призму удержания)

| Принцип | Суть | Влияние на удержание | Если нарушен |
|---------|------|---------------------|--------------|
| Need to Know | Зачем мне это? | Без «зачем» — отвал на старте | «Скучно», «Не вижу смысла» |
| Self-Concept | Я — взрослый, не ученик | Без автономии — сопротивление | «Меня как ребёнка учат» |
| Prior Experience | Мой опыт — ресурс | Без связи с опытом — отторжение | «Это не про меня» |
| Readiness | Мне это нужно сейчас | Без привязки к реальности — откладывание | «Когда-нибудь пригодится» |
| Orientation | Я решаю задачу, не изучаю предмет | Без практики — потеря смысла | «Теория без дела» |
| Motivation | Внутренние мотиваторы сильнее | Без внутренней ценности — поверхностность | «Делаю для галочки» |

### 2.4 Теория потока (Csikszentmihalyi) — зона оптимального вовлечения

```
СЛОЖНОСТЬ
    ↑
    │  ТРЕВОГА          ПОТОК
    │  (слишком         ═══════
    │   сложно)        (вызов ≈ навык)
    │
    │  СКУКА            АПАТИЯ
    │  (слишком         (нет ни вызова,
    │   легко)          ни навыка)
    └────────────────────────→ НАВЫК

ПРАВИЛО: задания должны быть на ~10-15% сложнее текущего уровня студента.
Слишком легко → скука → отвал.
Слишком сложно → тревога → отвал.

РЕАЛИЗАЦИЯ:
• Адаптивная сложность (разные уровни заданий)
• Опциональные «challenge» задания для продвинутых
• «Минимальное» и «продвинутое» задание в каждом модуле
• Scaffolding для тех, кто не справляется
```

### 2.5 Quick Wins — ранние победы

```
ПРИНЦИП: Первый ощутимый результат — в первые 48 часов.

ПОЧЕМУ:
• Подтверждает правильность решения купить / начать
• Формирует commitment (sunk cost + positive reinforcement)
• Создаёт momentum для продолжения

ПРИМЕРЫ QUICK WINS (по типу продукта):
• Курс по навыку → мини-результат, который можно показать другим
• Корпоративное обучение → применение в работе в тот же день
• Марафон → заметное изменение в ощущениях / поведении на 1-й день
• Онбординг → первая успешно выполненная рабочая задача

ПРАВИЛО: Quick win должен быть РЕАЛЬНЫМ, не искусственным.
Плохо: «Поздравляем, вы прошли первый урок!» (бессмысленный бейдж)
Хорошо: «Вы только что составили свой первый [результат] — покажите коллеге/другу»
```

### 2.6 Прогрессия сложности — принцип «от простого к сложному»

```
УРОВЕНЬ 1: Воспроизведение (Bloom 1-2)
  → Повтори по шаблону, ответь на вопрос, выбери правильный ответ
  → Scaffolding: полный

УРОВЕНЬ 2: Применение (Bloom 3)
  → Примени в новой ситуации, реши задачу по алгоритму
  → Scaffolding: частичный (чеклист, подсказки)

УРОВЕНЬ 3: Анализ и оценка (Bloom 4-5)
  → Сравни, найди ошибку, обоснуй выбор
  → Scaffolding: минимальный (только критерии)

УРОВЕНЬ 4: Создание (Bloom 6)
  → Создай свой продукт / решение / проект
  → Scaffolding: нет (самостоятельная работа)

ПРАВИЛО: каждый модуль поднимает на один уровень.
Модули 1-2 → уровень 1-2. Модули 3-5 → уровень 3. Финальный → уровень 4.
```

## §3. Обратная связь как механизм удержания

### 3.1 Timing обратной связи

```
НЕМЕДЛЕННАЯ (секунды)     → автоматические quiz, inline-проверки
  Эффект: подкрепление правильного поведения, коррекция ошибок
  Когда: знаниевые задания, тесты, drill

БЫСТРАЯ (часы, до 24ч)   → куратор, AI-проверка, peer-review
  Эффект: поддержание momentum, ощущение «меня видят»
  Когда: практические задания, домашние работы

ОТЛОЖЕННАЯ (дни)          → развёрнутый feedback на проект
  Эффект: глубокое осмысление, рост
  Когда: итоговые проекты, портфолио, сертификация
```

### 3.2 Качество обратной связи (формула)

```
НАБЛЮДЕНИЕ: «В твоей работе [конкретный элемент]...»
ЭФФЕКТ:     «Это [создаёт / не создаёт] [конкретный результат]...»
НАПРАВЛЕНИЕ: «Попробуй [конкретное действие]...»

Плохо:  «Хорошая работа!» (не обучает)
Плохо:  «Нужно переделать» (не даёт направления)
Хорошо: «Ты сформулировал 3 из 5 критериев — добавь критерии X и Y, и план станет полным»
```

### 3.3 Обратная связь как антидот оттока

```
ПРОБЛЕМА                    РЕШЕНИЕ ЧЕРЕЗ FEEDBACK
Студент сомневается         → «Ты на правильном пути, вот подтверждение: [...]»
  в себе
Студент застрял             → «Вот где ты сейчас, вот следующий шаг: [...]»
Студент не видит            → «Посмотри, что ты умел на старте и что умеешь сейчас»
  прогресса
Студент потерял             → «Вот как это применит [похожий на тебя человек]: [...]»
  смысл
```

## §4. Эмоциональный дизайн обучения

### 4.1 Эмоции и запоминание

```
ПОЗИТИВНЫЕ ЭМОЦИИ:
• Расширяют когнитивные ресурсы (broaden-and-build, Fredrickson)
• Повышают креативность и открытость к новому
• Снижают тревогу при сложных заданиях
→ Реализация: истории успеха, юмор, surprise-элементы, празднование побед

НЕГАТИВНЫЕ ЭМОЦИИ:
• Страх и тревога сужают фокус → блокируют усвоение
• Стыд → отвал (студент уходит, чтобы не чувствовать себя глупым)
• Фрустрация → может быть продуктивной (если есть поддержка) или деструктивной (если нет)
→ Реализация: психологическая безопасность, нормализация ошибок, scaffolding

ЭМОЦИОНАЛЬНАЯ БЕЗОПАСНОСТЬ (для трансформационных программ):
• «Здесь нет глупых вопросов»
• «Ошибка — часть процесса»
• «Вы не одни — другие тоже через это проходили»
• Анонимность при деликатных темах
```

### 4.2 Нормализация сложности

```
ПРИНЦИП: Когда студент чувствует «я не справляюсь», проблема не в нём — 
проблема в дизайне. Но пока дизайн не исправлен, студенту нужна нормализация.

ФОРМУЛЫ НОРМАЛИЗАЦИИ:
• «На этом этапе 70% студентов чувствуют то же самое — это нормально»
• «Этот модуль объективно сложнее предыдущих — дайте себе больше времени»
• «Ощущение "я ничего не понимаю" — признак того, что мозг перестраивает схемы»

АНТИПАТТЕРН:
• «Это же просто!» → студент чувствует себя глупым → уходит
• Отсутствие предупреждения перед сложным блоком → шок → уходит
```
