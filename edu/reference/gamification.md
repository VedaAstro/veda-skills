# Геймификация, награды, прогрессия

> Этот файл покрывает слой 3 (поведенческий) из модели удержания.
> Открывай, когда нужно спроектировать систему геймификации, наград, прогрессии, streak-механик.

---

## §1. Фреймворк: Octalysis (Yu-kai Chou) — 8 драйверов мотивации

```
ДРАЙВЕР                        ТИП          ИНСТРУМЕНТЫ
1. Эпический смысл              внутренний   миссия курса, «ты часть чего-то большего»
2. Развитие и достижение        внутренний   бейджи, уровни, прогресс-бар, milestones
3. Наделение творчеством        внутренний   открытые задания, выбор формата, кастомизация
4. Владение и обладание         внешний      коллекции, виртуальная валюта, кастомизация профиля
5. Социальное влияние           внешний      лидерборды, peer-recognition, шеринг достижений
6. Дефицит и нетерпение         внешний      ограниченный доступ, countdown, эксклюзив
7. Непредсказуемость            внешний      random rewards, easter eggs, surprise-контент
8. Избегание потери             внешний      streak, deadline, «потеряешь прогресс»
```

### Правило выбора драйверов

```
ВНУТРЕННИЕ (1, 2, 3) → долгосрочная мотивация, глубокое вовлечение
ВНЕШНИЕ (4-8) → быстрый старт, краткосрочный engagement

СТРАТЕГИЯ:
• Внешние для старта (первые 1-2 недели) → переход к внутренним
• Никогда не строй систему ТОЛЬКО на внешних → overjustification effect
  (внешняя награда убивает внутреннюю мотивацию)
```

## §2. Механики геймификации — каталог

### 2.1 Прогресс и достижения

```
ПРОГРЕСС-БАР
  Что: визуальная полоса / процент завершения
  Когда: всегда (базовый элемент)
  Нюанс: показывай прогресс по модулю И по курсу отдельно
  Психология: endowed progress effect — если показать 10% уже сделанным → выше completion
  Реализация: начинай прогресс-бар не с 0%, а с 5-10% (регистрация = первый шаг)

MILESTONES (вехи)
  Что: значимые точки на пути (не каждый урок, а ключевые достижения)
  Когда: каждые 20-25% курса
  Нюанс: milestone = ощутимый результат, не «ты прошёл модуль 3»
  Хорошо: «Ты провёл первую консультацию», «Ты создал свой первый [продукт]»
  Плохо: «Ты прослушал 10 уроков» (описывает потребление, не результат)

УРОВНИ / РАНГИ
  Что: иерархическая система статусов
  Когда: длинные программы (от 4 недель)
  Нюанс: уровень должен отражать реальную компетенцию, не время в системе
  Пример: Новичок → Практик → Мастер → Наставник
  Антипаттерн: 50 уровней без содержательной разницы
```

### 2.2 Streak и привычки

```
STREAK (серия)
  Что: непрерывная цепочка действий (дней / заданий / входов)
  Когда: марафоны, ежедневные практики, онбординг
  Психология: loss aversion — «не хочу потерять серию из 14 дней»
  
  ПРАВИЛА ДИЗАЙНА STREAK:
  • Freeze / skip — дай 1-2 «пропуска» (иначе одна ошибка → полный отвал)
  • Минимальное действие для сохранения streak = очень маленькое (1 мин, не 30 мин)
  • Визуализируй streak наглядно (цепочка, календарь, огонёк)
  • Награда за streak milestones (7 дней, 14, 21, 30)
  
  АНТИПАТТЕРН: streak без freeze → студент пропустил 1 день → серия обнулилась →
  «какой смысл продолжать» → отвал

HABIT LOOP (петля привычки, Duhigg)
  ТРИГГЕР → ДЕЙСТВИЕ → НАГРАДА
  
  Триггеры:
  • Временные: «каждый день в 9:00» (push-уведомление)
  • Контекстные: «после утреннего кофе», «перед рабочим днём»
  • Социальные: «когда коллега поделился прогрессом»
  
  Действие: минимальное (micro-learning, 5 мин)
  
  Награда: немедленная (streak +1, прогресс-бар, micro-feedback)
```

### 2.3 Бейджи и достижения

```
ТИПЫ БЕЙДЖЕЙ:

Completion badges (за прохождение)
  «Модуль 1 завершён», «Курс пройден»
  Ценность: низкая (констатация факта)
  Когда: базовый набор, есть всегда

Skill badges (за навык)
  «Первая консультация», «Сложный кейс решён»
  Ценность: средняя (отражает реальный навык)
  Когда: при навыковых курсах

Effort badges (за усилие)
  «7 дней подряд», «Все доп.задания выполнены»
  Ценность: средняя (отражает вовлечённость)
  Когда: для поддержания momentum

Social badges (за вклад в группу)
  «Помог 3 однокурсникам», «Лучший ответ недели»
  Ценность: высокая (социальное признание)
  Когда: при наличии community

Surprise badges (неожиданные)
  «Easter egg найден», «Первый ответ в 6 утра»
  Ценность: emotional engagement
  Когда: для разнообразия, не как основа

ПРАВИЛА БЕЙДЖЕЙ:
• Max 15-20 на весь курс (инфляция обесценивает)
• Каждый бейдж = понятный и достижимый критерий
• Визуально различимы (не 20 одинаковых иконок)
• Можно показать / поделиться (social proof)
• Не заменяют реальный результат — дополняют
```

### 2.4 Очки, валюта, лидерборды

```
ОЧКИ / XP
  Когда полезно: длинные программы с множеством активностей разного типа
  Когда вредно: короткие курсы, трансформационные программы (обесценивает глубину)
  Правило: очки должны отражать ЦЕННОСТЬ действия, не его количество
  Плохо: 10 XP за просмотр видео, 10 XP за сложное задание (уравнивает)
  Хорошо: 5 XP за просмотр, 50 XP за практическое задание, 100 XP за проект

ВИРТУАЛЬНАЯ ВАЛЮТА
  Когда: если есть «магазин» (доп.материалы, бонусы, доступ)
  Нюанс: валюта работает только при наличии ценных «товаров»
  Антипаттерн: валюта без возможности потратить → обесценивается

ЛИДЕРБОРДЫ
  ⚠️ ОСТОРОЖНО — самый рискованный инструмент
  
  Плюсы: стимулирует top-20% (соревновательных студентов)
  Минусы: демотивирует bottom-50% («я никогда не догоню»)
  
  БЕЗОПАСНЫЕ АЛЬТЕРНАТИВЫ:
  • Относительный лидерборд: «ты в top-30% по активности» (без имён)
  • Командный лидерборд: соревнуются группы, не индивиды
  • Персональный рекорд: «твой лучший результат — X, побей его»
  • Лидерборд за неделю (обнуляется) — даёт шанс каждому
```

## §3. Антипаттерны геймификации

```
АНТИПАТТЕРН                          ПОЧЕМУ ПЛОХО                         ЧТО ДЕЛАТЬ
─────────────────────────────────────────────────────────────────────────────────────────
Pointsification                      Очки за всё подряд →                  Очки только за
(очки ради очков)                    обесценивание                         ценные действия

Overjustification                    Внешняя награда убивает               Внешние для
(избыток внешней мотивации)          внутреннюю мотивацию                  старта → внутренние

Одинаковая механика                  Привыкание → скука →                  Чередуй типы 
на весь курс                         «игнор»                               наград и механик

Наказание за неучастие               Страх → избегание →                   Награда за участие,
(потеря всего прогресса)             полный отвал                          не наказание за неучастие

Геймификация вместо                  «Весело, но ничему                    Геймификация поверх
качественного контента               не научился»                          хорошего контента

Сложная система правил               Когнитивная нагрузка                  Max 3-4 механики
(10 типов очков, 5 валют)            на систему, а не на обучение          одновременно
```

## §4. Триггеры возврата (re-engagement)

```
ТИПЫ ТРИГГЕРОВ:

PUSH-УВЕДОМЛЕНИЯ
  • Напоминание о незавершённом задании (через 24ч)
  • Streak под угрозой (через 20ч без активности)
  • Новый контент доступен
  • Одногруппник завершил задание (social trigger)
  Правило: max 1 push в день, лучше 3-4 в неделю

EMAIL
  • Еженедельный дайджест прогресса
  • «Что ты пропустил» (FOMO, мягкий)
  • Персональная рекомендация следующего шага
  • История успеха похожего студента
  Правило: не чаще 2-3 раз в неделю

IN-APP
  • Cliffhanger в конце урока («в следующем уроке...»)
  • «Продолжить с того места, где остановился»
  • Предпросмотр следующего модуля
  • Countdown до дедлайна / события
```

### 4.1 Оптимальная частота триггеров по типу продукта

```
                    Push/день   Email/нед   In-app
Марафон (7-21д)     1           3-5         всегда
Курс (4-12 нед)     0-1         2-3         да
Флагман (3-6 мес)   0-1         1-2         да
Корпоратив          0-1         1-2         да
Мини-курс           0           1           да
```

## §5. Проектирование системы геймификации — алгоритм

```
ШАГ 1: ОПРЕДЕЛИ ЦЕЛЕВОЕ ПОВЕДЕНИЕ
  Что конкретно студент должен делать регулярно?
  (Смотреть уроки? Делать задания? Практиковать навык? Помогать другим?)

ШАГ 2: ВЫБЕРИ ДРАЙВЕРЫ (Octalysis)
  Какие 2-3 драйвера подходят твоей ЦА?
  (Достижение? Социальное? Творчество? Избегание потери?)

ШАГ 3: ПОДБЕРИ МЕХАНИКИ
  Из каталога (§2) выбери 3-5 механик, не больше.
  Проверь: нет ли антипаттернов (§3).

ШАГ 4: СПРОЕКТИРУЙ ПРОГРЕССИЮ
  Неделя 1-2: простые reward-loops (streak, первые бейджи)
  Неделя 3-4: социальные механики (peer, лидерборд)
  Неделя 5+: внутренняя мотивация (проект, реальный результат)

ШАГ 5: ОПРЕДЕЛИ ТРИГГЕРЫ (§4)
  Какие триггеры, с какой частотой, в каких каналах.

ШАГ 6: ПРОВЕРЬ
  □ Геймификация поддерживает обучение (не заменяет)?
  □ Система не сложнее, чем сам контент?
  □ Нет overjustification (внешние награды не убивают внутреннюю мотивацию)?
  □ Есть «пропуски» / safety nets (streak freeze, гибкие дедлайны)?
  □ Bottom-50% студентов не демотивированы системой?
```
