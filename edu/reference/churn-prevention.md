# Борьба с оттоком, onboarding, реактивация

> Этот файл покрывает диагностику и предотвращение оттока на всех этапах.
> Открывай, когда студенты отваливаются, нужен onboarding-дизайн или план реактивации.

---

## §1. Кривая отвала — диагностика точек

### 1.1 Универсальные точки отвала

```
ТОЧКА 0: ДО СТАРТА (0-24 часа после покупки)
  Проблема: buyer's remorse, не нашёл где начать, технические проблемы
  Потеря: 5-15%
  Решение: → §2 (onboarding)

ТОЧКА 1: ПЕРВАЯ НЕДЕЛЯ (дни 1-7)
  Проблема: «не то, что ожидал», слишком сложно/просто, нет quick win
  Потеря: 15-30%
  Решение: → §2 (onboarding) + quick wins (andragogy-retention.md §2.5)

ТОЧКА 2: ПЛАТО УСТАЛОСТИ (недели 2-4)
  Проблема: новизна прошла, результат не очевиден, рутина
  Потеря: 10-20%
  Решение: → §3 (early warning) + gamification.md (streak, milestone)

ТОЧКА 3: СЛОЖНЫЙ БЛОК (varies)
  Проблема: резкий рост сложности, студент «застрял»
  Потеря: 5-15%
  Решение: → scaffolding (andragogy-retention.md §1.4) + нормализация (§4.2)

ТОЧКА 4: ФИНИШНАЯ ПРЯМАЯ (последние 15-20%)
  Проблема: «основное уже понял, зачем доделывать»
  Потеря: 5-10%
  Решение: → финальный milestone + сертификация + social commitment
```

### 1.2 Расчёт cumulative completion rate

```
Если на каждой точке теряем по среднему:
  Start: 100%
  T0 (-10%): 90%
  T1 (-20%): 72%
  T2 (-15%): 61%
  T3 (-10%): 55%
  T4 (-7%):  51%
  
  Итого: ~50% completion rate (среднее по рынку для платных онлайн-курсов)
  
  Целевой: 70-80% для хорошо спроектированного курса
  Отличный: 85%+ (обычно только с live-сопровождением)
```

## §2. Onboarding — первые 48 часов (критический период)

### 2.1 Цели onboarding

```
ЦЕЛЬ 1: Снять buyer's remorse
  → Подтвердить правильность решения
  → Показать, что ждёт (roadmap)
  → Дать первый micro-результат

ЦЕЛЬ 2: Устранить технические барьеры
  → Доступ к платформе (логин, навигация)
  → Тест связи / оборудования (если live)
  → FAQ по частым проблемам

ЦЕЛЬ 3: Создать commitment
  → Публичное обязательство (представиться в чате)
  → Первое задание (маленькое, 100% выполнимое)
  → Контакт с группой / куратором

ЦЕЛЬ 4: Сформировать привычку
  → Установить расписание / режим обучения
  → Настроить уведомления
  → Первый streak-день
```

### 2.2 Чеклист onboarding (48 часов)

```
БЛОК «ДОСТУП» (0-2 часа после покупки):
□ Welcome-сообщение (email + messenger) — в течение 5 минут
□ Ссылка на платформу + инструкция входа
□ Контакт поддержки при проблемах
□ FAQ: 5 самых частых вопросов новичков

БЛОК «ОРИЕНТАЦИЯ» (2-12 часов):
□ Welcome-видео от спикера/куратора (max 3 мин)
□ Roadmap курса: что будет, когда, в каком порядке
□ «Что от вас потребуется»: время, усилия, формат
□ Знакомство с навигацией платформы

БЛОК «ПЕРВЫЙ ШАГ» (12-24 часа):
□ Micro-задание (5-10 мин, 100% выполнимое)
□ Результат задания виден сразу (feedback / прогресс-бар)
□ Представление в чате / группе («расскажите о себе»)

БЛОК «ЗАКРЕПЛЕНИЕ» (24-48 часов):
□ Первый полноценный урок + практика
□ Quick win: ощутимый результат, которым можно поделиться
□ Напоминание о расписании / следующем шаге
□ Check-in от куратора: «Как первый день? Есть вопросы?»
```

### 2.3 Onboarding по типам продуктов

```
МАРАФОН (7-21 день):
  Onboarding = вечер перед стартом + первое утро
  Ключевое: максимально простой первый день, zero friction
  Особенность: нет времени на долгую раскачку

КУРС (4-12 недель):
  Onboarding = модуль 0 (ориентация) + первая неделя
  Ключевое: roadmap + первый quick win + знакомство с группой
  Особенность: можно дать 2-3 дня на раскачку

ФЛАГМАНСКАЯ ПРОГРАММА (3-6 мес):
  Onboarding = неделя 0 + welcome-сессия live
  Ключевое: глубокое знакомство + commitment + personal goal setting
  Особенность: live-элемент критичен

КОРПОРАТИВНОЕ ОБУЧЕНИЕ:
  Onboarding = briefing от руководителя + первая сессия
  Ключевое: связь обучения с рабочими задачами
  Особенность: показать применимость в работе в первый же день

ВНУТРЕННЕЕ ОБУЧЕНИЕ:
  Onboarding = intro-модуль + buddy-система
  Ключевое: навигация + знакомство с командой + первая задача
  Особенность: buddy (наставник) ускоряет адаптацию в 2-3 раза
```

## §3. Early Warning Signals — раннее обнаружение оттока

### 3.1 Поведенческие индикаторы

```
КРАСНЫЙ (высокий риск отвала):
• Не заходил на платформу 5+ дней (для курса) / 2+ дня (для марафона)
• Не сдал 2+ задания подряд
• Не открыл доступный материал 7+ дней
• Написал в чат «не успеваю» / «слишком сложно» / «не понимаю»

ЖЁЛТЫЙ (средний риск):
• Снижение активности 3+ дня подряд (заходит, но не делает)
• Сдаёт задания с опозданием 2+ раза
• Перестал писать в чат (был активен → замолчал)
• Задания формальные (минимум усилий)

ЗЕЛЁНЫЙ (низкий, но следить):
• Смотрит уроки, но не делает практику
• Активен в чате, но не сдаёт задания
• Задаёт вопросы «а нужно ли это?» / «а зачем?»
```

### 3.2 Протокол реагирования

```
КРАСНЫЙ → немедленное действие (в течение 24ч):
  1. Личное сообщение от куратора (не шаблонное!)
  2. «Я заметил, что тебя давно не было. Всё в порядке?»
  3. Предложить конкретный следующий шаг (не «вернись», а «сделай вот это одно»)
  4. Если не ответил за 48ч → звонок / голосовое сообщение

ЖЁЛТЫЙ → проактивное действие (в течение 48ч):
  1. Check-in в общем чате или лично
  2. Напоминание о прогрессе: «Ты уже прошёл X% — осталось Y до milestone»
  3. Social proof: «Другие студенты на этом этапе тоже замедлялись — это нормально»

ЗЕЛЁНЫЙ → мониторинг + мягкое вмешательство:
  1. Ответить на вопрос «зачем» (вернуть актуальность)
  2. Пригласить в групповую активность
  3. Предложить лёгкое задание для восстановления momentum
```

## §4. Реактивация — возврат отвалившихся

### 4.1 Принципы реактивации

```
ПРИНЦИП 1: Не вини
  ❌ «Вы пропустили 3 модуля»
  ✅ «Мы подготовили план, как наверстать»

ПРИНЦИП 2: Снизь барьер возврата
  ❌ «Вернитесь и пройдите всё с начала»
  ✅ «Вот одно задание на 10 минут — начните с него»

ПРИНЦИП 3: Покажи прогресс
  ❌ «Вы отстаёте от группы»
  ✅ «Вы уже прошли 40% — до следующего milestone осталось 2 урока»

ПРИНЦИП 4: Используй social proof
  ❌ «Многие уже закончили»
  ✅ «Ваша однокурсница [имя] вернулась после 2-недельного перерыва и успешно закончила»

ПРИНЦИП 5: Дай причину вернуться СЕЙЧАС
  ❌ «Возвращайтесь, когда будете готовы»
  ✅ «На этой неделе live-сессия по [актуальная тема] — не пропустите»
```

### 4.2 Шаблоны реактивации (по каналам)

```
EMAIL (через 5-7 дней неактивности):
  Тема: «[Имя], ваш прогресс ждёт вас»
  Тело: 
  - Напоминание о прогрессе (что уже сделано)
  - Один конкретный следующий шаг
  - Мотивация: кейс / результат / факт
  - CTA: «Продолжить с [конкретный урок]»

MESSENGER / PUSH (через 3-5 дней):
  «[Имя], до milestone [X] осталось [Y] — продолжим?»

КУРАТОР / ЛИЧНОЕ (через 7-10 дней):
  1. «Привет! Заметил, что тебя давно не было. Как дела?»
  2. [Выслушать причину]
  3. «Понимаю. Давай сделаем так: [конкретный план возврата]»
  4. Назначить конкретный следующий шаг + дедлайн
```

### 4.3 Программа «мягкого возврата»

```
ДЛЯ ТЕХ, КТО ПРОПУСТИЛ 1-2 НЕДЕЛИ:
  • Recap-урок: ключевое за пропущенный период (5-10 мин)
  • Сокращённое задание (50% от полного)
  • Подключение к buddy / peer для catch-up

ДЛЯ ТЕХ, КТО ПРОПУСТИЛ 3+ НЕДЕЛИ:
  • Личная встреча с куратором (15 мин)
  • Индивидуальный план: что пропустить, что сделать обязательно
  • «Fast track» — минимальный путь до следующего milestone
  • Отдельный чат / группа «догоняющих» (снимает стыд)

ДЛЯ ТЕХ, КТО НЕ НАЧАЛ ВООБЩЕ:
  • Звонок / голосовое от куратора
  • «Новый старт» — пройти onboarding заново
  • Подключить к следующему потоку (если есть)
```

## §5. Дедлайны и темп — управление ритмом

### 5.1 Типы дедлайнов

```
ЖЁСТКИЙ ДЕДЛАЙН
  «Задание принимается до пятницы 23:59»
  Плюсы: создаёт urgency, синхронизирует группу
  Минусы: стресс, отвал при неуспевании
  Когда: live-программы с групповой работой, корпоративное обучение

МЯГКИЙ ДЕДЛАЙН
  «Рекомендуемый срок — пятница. Принимаем до конца модуля»
  Плюсы: гибкость, снижение стресса
  Минусы: прокрастинация, десинхрон группы
  Когда: курсы для взрослых с разным темпом жизни

АДАПТИВНЫЙ ДЕДЛАЙН
  «У вас 7 дней с момента открытия модуля»
  Плюсы: каждый в своём темпе, но ритм сохраняется
  Минусы: сложнее организовать peer-работу
  Когда: self-paced курсы, внутреннее обучение

ДЕДЛАЙН С GRACE PERIOD
  «Срок — пятница. Продление на 3 дня без штрафа по запросу»
  Плюсы: баланс дисциплины и гибкости
  Минусы: некоторые всегда просят продление
  Когда: оптимальный компромисс для большинства форматов
```

### 5.2 Ритм подачи контента

```
DRIP CONTENT (капельная подача)
  Что: модули открываются по расписанию
  Плюсы: удерживает ритм, предотвращает «проглотил всё за день и забыл»
  Минусы: ограничивает быстрых студентов
  Когда: групповые программы, марафоны

FULL ACCESS (полный доступ)
  Что: весь контент доступен сразу
  Плюсы: автономия, быстрые могут идти вперёд
  Минусы: нет ритма, высокий отвал
  Когда: self-paced, корпоративные библиотеки знаний

ГИБРИДНЫЙ
  Что: первые 1-2 модуля доступны сразу, остальные по расписанию
  Плюсы: быстрый старт + ритм
  Когда: оптимально для большинства курсов

РЕКОМЕНДАЦИЯ: drip content + возможность «забежать вперёд» для быстрых
(открыт текущий модуль + следующий)
```
