# Чеклисты курсостроения

> Каждый пункт трассируется до источника. Тег = книга/автор/принцип.

---

## 1. Чеклист: Проектирование курса (до создания контента)

```
ТРАНСФОРМАЦИЯ [Wiggins/McTighe: Backward Design]
[ ] Трансформация описана в 1 предложении: "Приходит как ___, уходит как ___"
[ ] 3-7 milestone'ов (модулей), каждый = измеримый навык
[ ] Для каждого модуля: что ученик ДЕЛАЕТ после (не знает, а делает) [Moore: Action Mapping]
[ ] Финальный assessment спроектирован ДО контента [UbD: Stage 2 before Stage 3]

УЧЕНИК [Knowles: Andragogy]
[ ] Целевой ученик описан (JTBD: когда ___, хочу ___, чтобы ___) [Olsen: PMF]
[ ] Уровень входа определён (что уже знает/умеет)
[ ] "Зачем?" отвечено для каждого модуля [K6: Need to Know]
[ ] Первый quick win за < 48 часов [Ericsson: motivation through progress]

СТРУКТУРА [Merrill: First Principles]
[ ] Курс problem-centered (по задачам, не по темам) [Merrill P1]
[ ] Сложность нарастает постепенно (scaffolding) [Csikszentmihalyi: Flow]
[ ] Ratio теория/практика ≤ 20/80 [Ericsson: Deliberate Practice]
[ ] Каждый урок проходит цикл Kolb: CE → RO → AC → AE

ФОРМАТ
[ ] Выбран формат (self-paced / cohort / blended) с обоснованием
[ ] Определена длительность модуля (оптимум: 1-2 недели)
[ ] Определена длительность урока (оптимум: 45-60 мин эффективного времени)
[ ] Видео-сегменты ≤ 8 мин [Clark/Mayer: Segmenting]
```

---

## 2. Чеклист: Дизайн модуля

```
РЕЗУЛЬТАТ
[ ] Learning outcome в формате: "После этого модуля ученик сможет [глагол] + [контекст]"
[ ] Глагол по Bloom: Apply/Analyze/Evaluate/Create (не Remember/Understand)
[ ] Outcome связан с общей трансформацией курса

КОНТЕНТ [Clark/Mayer: Multimedia Principles]
[ ] Нет декоративных элементов без учебной функции [Coherence]
[ ] Текст НЕ дублирует голос [Redundancy]
[ ] Подписи на схемах, не отдельно [Spatial Contiguity]
[ ] Разговорный стиль [Personalization]
[ ] Ключевое выделено визуально [Signaling]

ВОВЛЕЧЕНИЕ [Eyal: Hooked + Knowles]
[ ] Hook в начале: проблема / история / провокация [System 1 → System 2]
[ ] Активация предыдущего опыта [K2 + Merrill P2]
[ ] Bridge в конце: preview следующего модуля (cliffhanger)
[ ] Переменность: разные форматы внутри модуля

ПРАКТИКА [Ericsson + Brown/Roediger/McDaniel]
[ ] Задание на применение (не quiz на запоминание)
[ ] Scaffolding: пример → шаблон → самостоятельно
[ ] Обратная связь < 48 часов
[ ] Рубрика дана ДО задания [UbD]

КОГНИТИВНАЯ НАГРУЗКА [Sweller]
[ ] ≤ 4 новых концепций на урок [Chunking]
[ ] Pre-training: термины объяснены ДО использования
[ ] Progressive disclosure: детали по запросу, не все сразу
```

## 3. Чеклист: Дизайн урока (45-60 мин)

```
СТРУКТУРА (7 элементов)
[ ] 1. HOOK (2-3 мин): проблема/история/вопрос → "зачем?"
[ ] 2. АКТИВАЦИЯ (5 мин): "попробуй/вспомни" [Kolb CE + Generation]
[ ] 3. ФРЕЙМВОРК (10-15 мин): модель/принцип [Kolb AC]
[ ] 4. ДЕМОНСТРАЦИЯ (5-10 мин): пример/кейс [Merrill P3]
[ ] 5. ПРАКТИКА (15-20 мин): задание [Merrill P4 + Kolb AE]
[ ] 6. РЕФЛЕКСИЯ (5 мин): "что получилось/что сложно" [Mezirow]
[ ] 7. МОСТ (2 мин): preview следующего урока

ВИДЕО-КОНТЕНТ
[ ] Сегменты ≤ 8 мин (перебивка/задание между)
[ ] Talking head для эмоционального, screencast для процесса
[ ] Не читает текст со слайдов [Redundancy]
[ ] 1 идея на слайд, минимум текста

ЗАДАНИЕ ВНУТРИ УРОКА
[ ] Конкретная задача в реальном контексте [Moore: Action Mapping]
[ ] Критерии успеха ясны
[ ] Есть шаблон или пример для опоры
```

## 4. Чеклист: Дизайн ДЗ

```
ФОРМУЛИРОВКА
[ ] Контекст (ситуация) + задача (действие) + ограничения + критерии
[ ] Задание на уровне Apply+ по Bloom (не Remember)
[ ] Связано с реальной жизнью ученика [K3: Readiness, K4: Orientation]
[ ] Объём адекватный (30-60 мин работы, не 5 часов)

ПОДДЕРЖКА
[ ] Scaffolding: шаблон / пример / подсказки
[ ] Рубрика оценки (4 уровня) дана заранее
[ ] Возможность задать вопрос (чат/форум)

ОБРАТНАЯ СВЯЗЬ
[ ] Механизм feedback определён (auto / peer / mentor)
[ ] Срок получения feedback ≤ 48 часов
[ ] Feedback = конкретные рекомендации, не "хорошо/плохо"
[ ] Возможность доработки после feedback [Growth Mindset]
```

## 5. Чеклист: Assessment (промежуточный и финальный)

```
ПРОМЕЖУТОЧНЫЙ (formative)
[ ] Retrieval quiz в начале каждого модуля (по прошлому) [Roediger: Retrieval Practice]
[ ] Spaced: вопросы по материалу 1-2 модуля назад [Spacing Effect]
[ ] Interleaving: миксуй типы вопросов [Kornell]
[ ] Цель: обучение, не оценка (ошибки нормальны)

ФИНАЛЬНЫЙ (summative)
[ ] Уровень Bloom 5-6 (Evaluate/Create)
[ ] Authentic assessment: максимально близко к реальной задаче
[ ] Рубрика с 4 уровнями по каждому критерию
[ ] Минимум 3 грани понимания (из 6 UbD) задействованы
[ ] Включает рефлексию ("что изменилось в моём подходе")
```

## 6. Чеклист: Оффер и запуск

```
ОФФЕР [Hormozi: $100M Offers]
[ ] Dream Outcome ясен в 1 предложении
[ ] Value stack: ядро + 3-4 бонуса
[ ] Каждый бонус усиливает Value Equation (↑ likelihood или ↓ effort/time)
[ ] Гарантия снижает anxiety
[ ] 3 тарифа: базовый (anchor) / оптимальный (target) / премиум
[ ] Naming: результат + timeframe, не абстракция

LANDING PAGE [Miller: StoryBrand]
[ ] Герой = ученик, не автор курса
[ ] Проблема описана на 3 уровнях (external, internal, philosophical)
[ ] План в 3 шага (простой, понятный)
[ ] Success и Failure sections
[ ] Social proof: кейсы, отзывы, цифры
[ ] CTA ясный и один (не 5 кнопок)

LAUNCH SEQUENCE [Walker: PLF]
[ ] PLC1: Opportunity (7 дней до открытия)
[ ] PLC2: Transformation + кейсы (5 дней до)
[ ] PLC3: Ownership + Q&A (3 дня до)
[ ] Cart Open: 3-7 дней
[ ] Cart Close: scarcity + urgency
[ ] Email sequence на каждый этап
```

## 7. Чеклист: Post-Launch (после запуска)

```
ПЕРВЫЕ 48 ЧАСОВ
[ ] Welcome email + ожидания
[ ] Quick win — первый результат за 15 мин
[ ] Подключение к community
[ ] Accountability partner (если cohort)

МЕТРИКИ (трекать еженедельно)
[ ] Completion rate по модулям
[ ] Assignment submission rate
[ ] Engagement rate (активные / всего)
[ ] NPS после каждого модуля
[ ] Drop-off points (где бросают)

ИТЕРАЦИЯ [Ries: Build-Measure-Learn]
[ ] Собран feedback после каждого модуля
[ ] Определены top-3 проблемы
[ ] Исправления внесены ДО следующей когорты
[ ] A/B тест ключевых изменений

RE-ENGAGEMENT
[ ] Автоматическое сообщение при 3 днях молчания
[ ] Ценность, не guilt-trip ("вот что тебя ждёт", не "ты пропустил")
[ ] Check-in call для учеников в зоне риска (cohort)
```

## 8. Quality Review — финальная проверка всего курса

```
АНДРАГОГИКА
[ ] Каждый модуль отвечает на "зачем?" [K6]
[ ] Опирается на опыт ученика [K2]
[ ] Ориентирован на задачу, не на запоминание [K4]
[ ] Даёт выбор внутри модуля [K1]
[ ] Показывает прогресс [K5]

НАУКА ОБУЧЕНИЯ
[ ] Retrieval practice встроена [Make It Stick]
[ ] Spaced repetition: ключевые концепции возвращаются [Ebbinghaus]
[ ] Практика > теории (80/20) [Ericsson]
[ ] Feedback loop < 48 часов [Deliberate Practice]
[ ] Scaffolding от высокого к низкому [Vygotsky ZPD]

ИНСТРУКЦИОННЫЙ ДИЗАЙН
[ ] Backward design: assessment ДО контента [UbD]
[ ] Problem-centered, не topic-centered [Merrill P1]
[ ] Multimedia principles соблюдены [Clark/Mayer]
[ ] SAM: пилот перед масштабированием [Allen]

ПРОДУКТ
[ ] PMF validated (pre-sale или waitlist) [Ries]
[ ] Оффер по Value Equation [Hormozi]
[ ] Landing по StoryBrand [Miller]
[ ] Launch sequence готов [Walker]
[ ] Метрики определены и трекаются
```
