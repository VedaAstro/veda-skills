# EDU — проектирование и удержание в обучении

> Один скилл = два вопроса одной задачи.
> **ЧТО учить** (структура, цели, контент) + **КАК довести до результата** (удержание, мотивация, закрепление).

## Когда использовать

- «Спроектируй курс / программу / марафон / онбординг»
- «Людей теряем на 3-й неделе — что делать?»
- «Нужна система геймификации / бейджей / прогрессии»
- «Как встроить peer-learning / community?»
- «Ревизия курса: низкая доходимость, плохие отзывы»
- «Напиши структуру модуля / урока / ДЗ»
- Любое образовательное проектирование, где важен completion rate

---

## Ключевые принципы — нельзя забыть

### Закон 1: Backward Design (Wiggins & McTighe)
Проектируй от результата, не от контента.
```
ТРАНСФОРМАЦИЯ ученика → Доказательства трансформации (assessment) → Опыт обучения
НЕ: «есть 40 часов материала» → «запихну в 8 модулей»
```

### Закон 2: 4 слоя удержания

```
СЛОЙ 1: КОГНИТИВНЫЙ    — материал усваивается (нагрузка, spaced repetition, chunking)
СЛОЙ 2: МОТИВАЦИОННЫЙ  — студент хочет продолжать (автономия, компетентность, SDT)
СЛОЙ 3: ПОВЕДЕНЧЕСКИЙ  — привычка учиться сформирована (триггеры, streak, награды)
СЛОЙ 4: СОЦИАЛЬНЫЙ     — среда поддерживает (peers, accountability, community)
```

**Правило диагностики**: определи слой проблемы ПЕРЕД тем как проектировать решение.
Ошибка: лечить мотивационную проблему геймификацией (слой 3 вместо слоя 2).

### Закон 3: Deliberate Practice (Ericsson)
Навык = целенаправленная практика + обратная связь + постепенное усложнение.
```
Знание ≠ Навык. Посмотрел видео ≠ научился.
Сделал → получил фидбек → скорректировал → сделал лучше = научился.
```

### Формула удержания
```
RETENTION = f(Quick Win) × f(Progress Visibility) × f(Social Bond) × f(Effort/Reward Balance)

Если ЛЮБОЙ множитель ≈ 0 → retention падает, независимо от остальных.
```

## Быстрый маршрут

```
ЗАДАЧА: Спроектировать обучающий продукт от начала до конца

ШАГ 1: ТИП ПРОДУКТА
  → Курс / марафон / флагман / корпоративное / онбординг?
  → Async / sync / blended?
  → Длительность, размер группы, ЦА (самоорганизация, опыт)
  → Текущие метрики: completion rate, точки отвала?

ШАГ 2: СТРУКТУРА (ЧТО учить)
  → Трансформация: «приходит как ___, уходит как ___»
  → Декомпозиция на 3-7 milestones (что ученик ДЕЛАЕТ, не знает)
  → Assessment: как доказать трансформацию?
  → Контент: минимум теории для каждой практики
  → Scaffolding: пример → шаблон → самостоятельно
  → reference/course-design.md, reference/andragogy.md

ШАГ 3: УДЕРЖАНИЕ (КАК довести)
  Когнитивный слой:
  → Spaced repetition встроен в структуру
  → Chunking: один концепт за раз, 10-min правило
  → «Зачем» объяснено перед каждым блоком (Knowles K6)
  → reference/andragogy-retention.md

  Мотивационный слой:
  → Quick win в первые 48 часов
  → SDT: автономия / компетентность / связность
  → Баланс усилия/награды (не слишком легко, не слишком сложно)
  → reference/andragogy-retention.md §мотивация

  Поведенческий слой:
  → Механики геймификации под тип продукта (3-5, не больше)
  → Прогрессия: внешние для старта → внутренние к концу
  → Триггеры возврата: push / email / in-app
  → reference/gamification.md

  Социальный слой:
  → Формат peer-взаимодействия (buddy / группа / peer-review)
  → Accountability-механика (уровень 1-4)
  → Community: структура + ритуалы
  → reference/social-learning.md

ШАГ 4: ЗАЩИТА ОТ ОТТОКА
  → Onboarding первые 48 часов: quick win + commitment + технический доступ
  → Early warning signals: красный / жёлтый / зелёный
  → Протокол реактивации
  → reference/churn-prevention.md

ШАГ 5: НИШЕВАЯ СПЕЦИФИКА
  → Коучинг / астро / психология / корпоратив
  → reference/niche-specifics.md
```

## Диагностика: какой слой не работает?

```
СИМПТОМ                              → СЛОЙ         → REFERENCE
«Материал сложный / непонятный»     → когнитивный  → andragogy-retention.md §1
«Не вижу смысла продолжать»         → мотивационный → andragogy-retention.md §2
«Забываю заходить / откладываю»     → поведенческий → gamification.md
«Одиноко, не с кем обсудить»        → социальный    → social-learning.md
«Отвалился и не вернулся»           → отток         → churn-prevention.md §4
«Падают на первой неделе»           → onboarding    → churn-prevention.md §2
«Доходят до 80% и бросают»          → финальный     → churn-prevention.md §1.1 T4
```

## Форматы — доходимость и цена

| Формат | Completion | Цена | Когда |
|--------|-----------|------|-------|
| Self-paced | 3-15% | $ | Информационный продукт, масштаб |
| Blended | 30-50% | $$ | Масштаб + качество |
| Cohort-based | 60-85% | $$$ | Трансформация, accountability |
| 1-on-1 / Mentorship | 90%+ | $$$$ | Максимальный результат |

## Self-check

```
АРХИТЕКТУРА (проверь ПЕРЕД дизайном контента):
□ Путь ученика описан как ЛИНЕЙНАЯ последовательность шагов (не категории/разделы)?
□ Каждый «раздел» = ЗАДАЧА (что делает), а не ТЕМА (что содержит)? [Merrill P1]
□ Навигация = путь ученика, а не информационная архитектура?
□ Прогресс встроен В путь, а не вынесен отдельно?
□ Нет якорения на текущий UI — задача переосмыслена с нуля?

КОНТЕНТ И МЕХАНИКИ:
□ Определена трансформация ученика (не «изучит тему», а «сможет делать X»)?
□ Backward design: assessment → опыт обучения → контент?
□ «Зачем» объяснено перед каждым блоком?
□ Quick win в первые 48 часов?
□ Spaced repetition встроен в структуру?
□ Когнитивная нагрузка управляется (один концепт, chunking, 10-min)?
□ Мотивация: автономия / компетентность / связность (SDT)?
□ Геймификация соответствует типу продукта (3-5 механик, не натянута)?
□ Социальный элемент присутствует (peer / group / accountability)?
□ Onboarding проработан отдельно (первые 48 часов)?
□ Early warning signals определены?
□ Есть план реактивации?
□ Обратная связь встроена в каждый модуль (не бонус — обязательный элемент)?
```
