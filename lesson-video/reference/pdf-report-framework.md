# PDF-отчёт воронки — фреймворк

## Что это

Персональный PDF-разбор натальной карты по теме трека (деньги / отношения / призвание).
Выдаётся в уроке как бонус. Генерируется LLM на основе данных карты из AstroGuru.

## Единственная цель

Конвертировать в бесплатную диагностику. Всё остальное — средство.

## Как работает конверсия

```
Кармическая задача в карте (центральная ось)
  │
  ├─ Барнум:        позиция карты → бытовая ситуация → "откуда знают?!"
  ├─ Потенциал:     йоги/сильные позиции → "у меня есть ЭТО?!"
  ├─ Внешний локус: задача в карте → "не я плохая, а задача не та"
  ├─ Враг:          усилия не туда → цикл ужесточается
  ├─ Снятие вины:   через факты карты, НЕ через фразы "не ваша вина"
  └─ Открытая петля: задача видна, КАК решить = только на разборе
```

## Барнум-формула

Каждый абзац с астро-позицией строится по формуле:

1. **Позиция**: планета + дом + знак
2. **Ситуация**: конкретная бытовая ситуация, которую узнает 80% людей с этой позицией
3. **Узнавание**: мягкий вопрос или утверждение

Пример (деньги):
> Сатурн в 11-м доме, ретроградный. Доход у вас растёт медленно — но стабильно.
> При этом крупные суммы приходят с задержкой: договорились, а деньги — через месяц, два.

Пример (отношения):
> Венера в Козероге, 7-й дом. Вы выбираете партнёров старше или серьёзнее себя.
> Лёгкие отношения «не цепляют» — нужна глубина. Но глубина оборачивается тяжестью.

## Снятие вины: механика

НЕ говорить «не ваша вина». Показать через факты:

1. Описать позицию карты (Раху в 12-м)
2. Описать цикл, который она создаёт (усилия → нет видимого результата → ещё усилия)
3. Показать: привычное направление усилий НЕ совпадает с задачей карты
4. Читатель сам делает вывод: «так вот почему...»

Это правило 9 из скилла: **показывай, не утверждай**.

## Визуалы из AstroGuru

Элементы платформы AstroGuru, которые усиливают PDF:

### Обложка (секция 1)
- **Натальная карта D1** — `chart.GetChartDataImage` → base64 → обложка
- Эффект: «это МОЯ карта» = мгновенная персонализация

### Скрытые ресурсы (секция 3)
- **Шадбала (сила планет)** — бары с числами
- Эффект: «У меня Луна 551?! А что это значит?» → усиление потенциала

### Кармическая задача (секция 4)
- **Кармические планеты (Раху/Кету)** — иконки + дом + знак
- Эффект: визуальный якорь к главной оси отчёта

### Текущий период (секция 6)
- **Маха-даша + антар-даша** — шкала с датами
- Эффект: «вот моё окно» → срочность

Правило: каждый визуал = усиление конкретной эмоции. Не набрасывать всё.

## Связка «жизнь → метафора → термин»

Из основного скилла. В PDF работает так же:

1. Ситуация из жизни (деньги утекают)
2. Метафора (сейф без замка)
3. Термин (Дхана-йога)

Термин — последним. Без подготовки он отпугивает.

## Мосты к диагностике

Формат: утверждение-факт. Не вопрос, не призыв.

Хорошо:
- «Какие именно шаги активируют этот ресурс — определяется при индивидуальном разборе.»
- «Конкретный маршрут коррекции зависит от полной конфигурации вашей карты.»
- «Это тема персонального разбора — общий рецепт здесь был бы безответственным.»

Плохо:
- «Хотите узнать больше? Запишитесь на диагностику!»
- «На нашей бесплатной консультации вы получите...»

Каждый мост — разный. Не повторять формулировку.

## Запреты

- Практические рекомендации, мантры, ритуалы, даты
- Банальные утешения и прямые «это не ваша вина»
- Общие фразы: «звёзды говорят», «вселенная», «энергия денег»
- Секция «направления коррекции» — даже общие советы обесценивают диагностику
- Вопросы в CTA: «Хотите?» = сомнение. Только факты
- Попсовый маркетинг: PAS, «знакомо, когда...» в первом предложении

## Техническая реализация

Файл: `platform/bot-admin-api/app/api/web/lessons.py`

- `PDF_SYSTEM_PROMPT` — системный промпт с алгоритмом сдвига
- `_build_pdf_user_prompt()` — данные карты + структура секций
- `_call_openrouter()` — вызов LLM (gemini-2.5-flash)
- `_parse_sections()` — парсинг `<section>` тегов
- Кэш: таблица `web_lesson_pdf_reports` (user_id + track = unique)

Стоимость: ~1.36₽ за отчёт (Gemini 2.5 Flash).
