# Видео-уроки для воронки

## Когда использовать
- Создание сценария урока для воронки
- Аудит / доработка существующего сценария
- Создание cue-файлов для синхронизации карты с видео

## Урок vs вебинар (ключевое отличие)

| | Вебинар | Урок |
|---|---------|------|
| Цель | Продать | Дать ценность → создать demand |
| Хронометраж | 60-90 мин | 12-17 мин |
| Proof | Каждые 5-7 мин | 1 кейс за урок |
| CTA | Агрессивный оффер | Мягкий: домашка + диагностика + open loop |
| Тон | Продающий наставник | Консультация один-на-один |

Применять вебинарный чеклист к уроку = главная ошибка. Proof каждые 5 мин, trial close, стэкинг — всё это ВРЕДИТ контентному формату.

## Правила

### Контент
1. **Hook = астро-факт** — первое предложение = конкретный элемент карты. Через боль/«знакомо?» — вебинарный шаблон, аудитория 2+ года его чувствует
2. **Связка «жизнь → метафора → термин»** — термин приходит последним как ярлык для понятой концепции. Без подготовки термин отпугивает. Без терминов — нет веса
3. **Персонализация каждые 2-3 мин** — карта = со-ведущий. Если 3 мин без обновления карты → блок слишком теоретический
4. **Netflix-мосты только между ключами** (3-4 за урок) — через интригу/open loop. Между каждым абзацем = вебинарный стиль
5. **Один кейс = иллюстрация метода** — ситуация → что в карте → рекомендация → результат. После кейса trial close не нужен
6. **Предутилизация всех веток** — у каждого зрителя разный результат. Каждый ключ с картой = ветки для всех исходов. «Плохой» результат = «диагноз, а не приговор» + что делать

### Формат и голос
7. **Голос Ирины** — короткие рубленые фразы, маркеры каждые 2-3 абзаца, ЕЁ метафоры (ГДЗ, экзамен Сатурна, навигатор). Подробнее → `reference/examples-and-voice.md`
8. **Формулировки через желаемое** — «карта показывает деньги через X» (X звучит привлекательно), а не «вам нужно делать X»
9. **Показывай, не утверждай** — факты карты → зритель/читатель делает вывод сам. Никаких прямых утешений («это не ваша вина»), банальных обещаний, директивных оценок. Снятие вины = через механику карты: показать задачу → человек сам понимает, что шёл не туда
10. **Готовых решений нет** — показать что решение существует и конкретное. Рецепт = нет смысла идти на диагностику. Фрейм: «дать рецепт для тысяч разных карт — безответственно»

### Техническое
11. **Документ для Ирины — без JSON** — один документ (сценарий + описание карты обычным языком), отдельный для разработки (cue JSON)
12. **Тайминг = слова/130 + паузы + 15%** — считать, а не фантазировать. ~15 мин = ~1700-1900 слов
13. **Сообщения Веды = только при персональных данных из карты** — нет данных = нет сообщения. 1 за раз, мин. 8 сек, + 1 резюме после видео
14. **Период = seed в уроке 1, reveal в уроке 3** — урок 1: метафора сезонов, без данных. Урок 3: полные данные периода

## Структура урока

```
[0:00-0:25]  HOOK — астро-факт
[0:25-0:55]  ОБЕЩАНИЕ — 3 пункта что узнает
[0:55-1:30]  ФРЕЙМ — почему карта (30 сек)
[1:30-4:00]  КЛЮЧ #1 + персонализация + предутилизация + Веда #1
             Netflix-мост →
[4:00-6:00]  КЛЮЧ #2 + персонализация + предутилизация + Веда #2
             Netflix-мост →
[6:00-8:30]  КЛЮЧ #3 (речь Ирины, seed для урока 3)
[8:30-10:00] КЕЙС — иллюстрация метода
[10:00-11:00] ТИЗЕР следующего урока (open loop)
[11:00-12:00] ДОМАШКА (2-3 вопроса, 30 сек) + ЗАКРЫТИЕ
```

Каждый КЛЮЧ: (1) текст Ирины, (2) что делает карта, (3) инсайт-панель, (4) паузы.

## Серия из 3 уроков

| Урок | Тема | Вера (Хормози) | Сдвиг (алгоритм) | Сообщения Веды |
|------|------|----------------|-------------------|----------------|
| #1 | 2-й + 11-й дом. Период = seed | В метод: «это про МЕНЯ» | Любопытство → Узнавание → Удивление | 2 (Ключ #1-2). Ключ #3 = речь |
| #2 | Дома потерь (6,8,12). Кармическая задача | В себя: «не я — задача» | Осознание → Тревога (через контент, не утешения) | 2-3 |
| #3 | Канал (10-й) + период reveal + коррекция | В условия: «навигатор» | Надежда → Фрустрация → Решимость | 2-3 |

Алгоритм эмоционального сдвига, враг/оружие, Барнум-формула → `reference/conversion-framework.md`

## Быстрый маршрут

```
1. КОНТЕКСТ: какой урок (1/2/3), тема, open loop из предыдущего
2. СКИЛЛЫ:   buyer-persona → сегмент, боли, язык
             astro-content → элементы карты
3. СЦЕНАРИЙ: заполнить структуру, применить правила 1-14
4. ПРОВЕРКА: self-check ниже
5. ВЫХОД:    Google Doc для Ирины + .md в репо + cue JSON отдельно
```

## Self-check

```
HOOK:
□ Первое предложение = астро-элемент (не боль, не «знакомо?»)?
□ Маркетинговые шаблоны (PAS, BAB) отсутствуют?

КОНТЕНТ:
□ Глубина через механику, не через санскрит?
□ Максимум 3-4 новых концепции за урок?
□ Контент для аудитории 2+ года в базе?

КАРТА:
□ Персонализация каждые 2-3 мин?
□ Первый контакт с картой до 1:30?
□ Предутилизация: ветки для всех исходов?
□ Сообщения Веды только где есть данные из карты?

ГОЛОС:
□ Короткие фразы, маркеры, ЕЁ метафоры?
□ Формулировки через желаемое (не «должен»)?
□ «Дорогие мои» (не «девочки/красотки»)?

ФОРМАТ:
□ Тайминг посчитан (слова/130 + паузы + 15%)?
□ 12-17 мин итого?
□ Документ для Ирины без JSON?
□ Это урок, не вебинар? (нет trial close, нет proof каждые 5 мин)
```

## PDF-отчёт (бонус воронки)

PDF — персональный разбор карты, который выдаётся в уроке как бонус. Цель = конвертировать в диагностику.

### Отличие от урока и вебинара

| | Урок | Вебинар | PDF-отчёт |
|---|------|---------|-----------|
| Канал | Видео + карта | Видео + слайды | Текст + визуал карты |
| Голос | Ирина | Ирина | LLM-текст (нет голоса) |
| Время контакта | 12-17 мин | 60-90 мин | 3-5 мин чтения |
| CTA | Мягкий seed | Агрессивный оффер | Мост через незавершённость |

PDF работает в самых жёстких условиях: нет голоса, нет интерактива, минимум времени. Каждое предложение бьёт точнее, чем в видео.

### Центральная ось: кармические задачи

Весь отчёт строится вокруг одной идеи: **в карте записаны кармические задачи**. Это единый стержень, который одновременно создаёт:

- **Барнум**: повторяющийся паттерн → каждый узнаёт себя
- **Внешний локус**: задача записана в карте → не вина человека
- **Враг**: решаете не ту задачу → цикл ужесточается
- **Потенциал**: решение задачи открывает ресурс
- **Снятие вины**: не вы плохая → вам не показали задачу
- **Открытая петля**: задача видна → КАК решить = на разборе

### Алгоритм эмоционального сдвига

Порядок секций создаёт цепочку внутренних переживаний:

```
Любопытство → Узнавание → Удивление → Осознание → Тревога → Надежда → Фрустрация → Решимость
"что в карте?"  "откуда     "у меня     "вот почему"  "я это     "но можно!" "но КАК?"    "иду на
                 знают?!"    есть ЭТО?!"               не исп."                             разбор"
```

### Барнум-формула (в каждом абзаце)

```
Позиция карты → типичная бытовая ситуация → узнавание
```

Плохо: «Венера в 6-м доме создаёт напряжение в финансовой сфере.»
Хорошо: «Венера в 6-м — деньги у вас связаны с ежедневным трудом. Знакомо: работаете больше других, а на счёте — как у тех, кто особо не напрягается?»

### Снятие вины: через контент, не через фразы

**ЗАПРЕЩЕНО**: «Это не ваша вина», «вы не виноваты», любые прямые утешения.
**ПРАВИЛЬНО**: показать через карту механику задачи → читатель **сам** приходит к выводу.

Пример: описать Раху в 12-м → цикл «усилия → нет результата → вина → ещё усилия» →
читатель узнаёт свою ситуацию → понимает: дело в задаче, а не в нём.

### Визуальные элементы из AstroGuru

PDF усиливается визуалами из платформы AstroGuru (chart.GetChartDataImage и др.):

| Элемент | Где в PDF | Зачем |
|---------|-----------|-------|
| Натальная карта (D1) | Обложка | «Это МОЯ карта» — персонализация |
| Кармические планеты (Раху/Кету) | Секция «Кармическая задача» | Визуальный якорь к главной оси |
| Шадбала (сила планет) | Секция «Скрытые ресурсы» | «У меня СИЛЬНЫЙ Юпитер?!» |
| Текущие периоды (даша) | Секция «Текущий период» | Срочность — визуально видно окно |

Не набрасывать всё подряд. Каждый визуал = усиление конкретной эмоции из алгоритма сдвига.

### Структура 12 секций

| # | Секция | Эмоция | Что делает |
|---|--------|--------|-----------|
| 1 | Обложка + карта | Любопытство | Визуал карты, имя, тема |
| 2 | Как тема устроена в карте | Узнавание | Барнум: позиции → ситуации |
| 3 | Скрытые ресурсы | Удивление | Йоги, накшатры → нереализованный потенциал |
| 4 | Кармическая задача | Осознание | Раху-Кету → цикл повторений |
| 5 | Паттерны поведения | Осознание | Усилия не туда → механизм через карту |
| 6 | Текущий период | Тревога + надежда | Даша + транзиты = окно/давление |
| 7 | Окно возможностей | Срочность | Ближайшие месяцы — конкретика |
| 8 | Что видно — и что скрыто | Фрустрация | Незавершённость: решение есть, но... |
| 9 | Что получите на разборе | Решимость | Конкретный список выгод |
| 10 | Почему именно сейчас | Срочность | Привязка к периоду |
| 11 | Как записаться | Действие | Шаги + ссылка |
| 12 | Поделитесь с близкими | Виральность | Реферальный механизм |

### Запреты

- Никаких практических рекомендаций, мантр, ритуалов, дат
- Никаких банальных утешений («это не ваша вина», «всё будет хорошо»)
- Никаких общих фраз («звёзды говорят», «вселенная посылает»)
- Секция «направления коррекции» = обесценивание диагностики. Не давать даже общих советов
- Мосты к диагностике = утверждения-факты, не вопросы («Хотите узнать?» — нет)
