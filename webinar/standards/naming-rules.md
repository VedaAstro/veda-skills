# Naming Rules — правила именования для любого запуска Vedantica

## Принцип (one-liner)

> **Название работает на витрине (в title, в лендинге, в email) — через БОЛИ клиентки в её языке. Астрология — механизм внутри, не витрина. Проверка через Wordstat mandatory.**

Источник правила: `context/market-language-shift.md` (Wordstat 2024-2026).

---

## 1. TITLE ЭФИРА (вебинар / прямой эфир)

### ✅ Правильные паттерны

- **«Где твои Х»** — «Где твои деньги», «Где твои ресурсы», «Где твоя дверь дохода»
- **«Почему Y»** — «Почему нет денег», «Почему повторяются отношения», «Почему нет сил»
- **«Как вернуть / изменить / начать Z»** — «Как вернуть энергию», «Как изменить судьбу», «Как начать зарабатывать»
- **«[Тема боли]: как [решение]»** — «Карма рода: как переписать», «Предназначение: где моё место»
- **Личные/притяжательные местоимения** («твои», «твой», «моё») — включают идентификацию

### ❌ Запрещённые паттерны в title

- ❌ «Ведический астролог за N дней / месяцев» (термин −48%)
- ❌ «Натальная карта с нуля» (термин −38%)
- ❌ «Джйотиш для начинающих» (термин)
- ❌ «Нейросети в астрологии» / «Нейроастролог за 60 дней» (в 10× хуже «Деньги/Карма» по Vedantica analytics)
- ❌ «Шад-бала / даши / астакаварга / накшатры» (профессиональные термины)
- ❌ «15 лет практики» (запрещено Алексом)
- ❌ «Прогноз на знаки зодиака» (газетная астрология)
- ❌ «Заработай миллион на астрологии» (обещание-ловушка)

### 📋 Checklist перед финализацией title

- [ ] Содержит ли боль клиентки? (что она прямо испытывает)
- [ ] Использует ли её собственные слова? (проверено через Wordstat top-requests)
- [ ] Отсутствуют ли астро-термины в самом title? (допустимы в подзаголовке)
- [ ] Есть ли персональное местоимение? («твои», «моё», «твой»)
- [ ] Есть ли решение / обещание, не лже-обещание?

**Если хоть один пункт «нет» — правь.**

### Примеры для контраста (из Vedantica аналитики)

| Title с финзвуком | Ср.выручка | Title «Нейросети» | Ср.выручка |
|---|---|---|---|
| «Деньги / Карма» | 3.9 M | «Нейросети в астрологии» | 384 K |
| Фактор ×10 в пользу болевых title |  |  |  |

## 2. НАЗВАНИЕ КУРСА / ПРОГРАММЫ

### Принцип

Название курса работает на 2 аудитории:
1. Холодный трафик — должен понять за что платит («профессия» / «доход» / «путь»)
2. Тёплая база — должна увидеть наследование концепции (Vedantica, академия)

### ✅ Правильные примеры

- **«Профессия НейроАстролог»** — есть «профессия» (мотив) + продуктовый бренд (НейроАстролог)
- «Путь астролога» — если акцент на передаче
- «Академия [тема]» — если акцент на школе
- «Наставничество [тема]» — если акцент на поддержке

### ❌ Нежелательные

- ❌ «Курс ведической астрологии» — термин без боли
- ❌ «Практикум по натальной карте» — термин без результата
- ❌ «Школа джйотиш» — специалист-говорящий (boring холодной базе)

### 📋 Checklist

- [ ] Есть ли в названии «что я получу» (профессия / доход / поддержка / путь)?
- [ ] Отсутствуют ли чисто астрологические термины без мотива?
- [ ] Удобно ли произнести в разговоре с подругой / мужем?

## 3. НАЗВАНИЯ МОДУЛЕЙ КУРСА

### Принцип

Модуль = этап пути клиентки. Название модуля описывает **что она получит пройдя его**, не «о чём лекции».

### ✅ Правильные примеры (переформулируем)

| Астро-термин (сейчас) | Рекомендация — через результат |
|---|---|
| «Натальная карта. Основы» | «Карта как навигатор: где ваши ресурсы» |
| «Накшатры» | «27 психотипов: понять себя и клиентку» |
| «Дробные карты» | «Второй слой: почему одно идёт, другое блокируется» |
| «Даши и периоды» | «Ритмы судьбы: когда действовать, когда ждать» |
| «Транзиты» | «Что приходит в вашу жизнь сейчас» |
| «Йоги богатства» | «Коды благосостояния в карте» |
| «Поток клиентов» | «Как первые клиентки находят вас (без рекламы)» |
| «Консультирование» | «Как проводить разбор и не бояться» |

**Не обязательно полностью заменять — можно давать через двоеточие: «Накшатры: 27 психотипов».**

### 📋 Checklist на модуль

- [ ] Отвечает ли название на вопрос «что я получу пройдя модуль»?
- [ ] Использует ли потребительский язык или только профессиональный?
- [ ] Узнаваемо ли название для тёплой базы (преемственность с прошлыми курсами)?

## 4. НАЗВАНИЯ КОНСУЛЬТАЦИЙ ПРОДУКТА (в нашем случае — НейроАстролог)

### Принцип

Каждая консультация = ответ на **конкретный жизненный вопрос** клиентки. Название = её вопрос в её языке.

### Проверка через Wordstat

Для каждой консультации:
1. Вытащить JTBD из SQL-промпта (поле `description`, `prompt`)
2. Найти 3-5 потребительских запросов, которые попадают в этот JTBD
3. Проверить объём этих запросов в Wordstat
4. Название консультации = **близкое к самому частому запросу**

### ✅ Хорошие названия (близки к запросу)

| Название | Близкий запрос | Объём |
|---|---|---|
| **Где мои деньги** | «как заработать денег» | 220 081/мес |
| **Совместимость с ребёнком** | «как понять ребенка» | 155 023/мес |
| **Где встретить партнёра** | «как встретить мужа / как найти любовь» | 6 125/мес + |
| **Здоровье и энергия** | «нет сил и энергии» | 11 178/мес |
| **Годовой прогноз** | «прогноз на год» | 236 800/мес (сезонный) |

### ⚠️ Спорные (требуют решения)

| Название | Близкий запрос | Проблема |
|---|---|---|
| «Распаковка личности» | «как понять свой характер» | Термин «распаковка» маркетинговый, сленг |
| «Книга силы» | «как раскрыть свой потенциал» | Хорошо — бренд, но холодный трафик не поймёт без контекста |
| «Прогнозирование 2.2 ст.» | — | Технический термин, не понятен клиентке |
| «Деньги в лёгкости» | — | Маркетинговая абстракция без прямого запроса |

### ❌ Плохие (переименовать)

- «Джаимини-карамша разбор» → переименовать в потребительский язык
- «Астакаварга 8 домов» → «Сильные и слабые зоны жизни»

### 📋 Checklist на название консультации

- [ ] Есть ли близкий потребительский запрос в Wordstat с объёмом ≥1000/мес?
- [ ] Понимает ли клиентка без словаря что это?
- [ ] Не содержит ли только профессиональных терминов?

## 5. НАЗВАНИЯ БОНУСОВ

### Принцип

Бонус = **конкретный артефакт или действие**, которое получает клиентка. Название — через артефакт + результат.

### ✅ Правильно

- «Лакшми Пуджа во Вриндаване» — артефакт (Пуджа) + место (сильный якорь)
- «Транзит 25 дней PDF» — артефакт (PDF) + срок (конкретно)
- «Банк 50 постов для соцсетей» — артефакт (посты) + число (конкретно)
- «Аудио-медитация "Финансовый код"» — артефакт + результат

### ❌ Плохо

- ❌ «Эксклюзивный бонус для досидевших» — нет артефакта
- ❌ «Подарок к покупке» — пустое
- ❌ «Дополнительный модуль» — о чём?

### 📋 Checklist на бонус

- [ ] Назван ли конкретный артефакт (PDF / видео / аудио / доступ / участие)?
- [ ] Указан ли результат или срок?
- [ ] Стоит ли бонус в value-stack конкретно в рублях с рыночным обоснованием?

## 6. MANDATORY WORKFLOW ПЕРЕД ФИНАЛИЗАЦИЕЙ ЛЮБОГО НАЗВАНИЯ

```
1. Взять JTBD / боль / результат из продукта
        ↓
2. Сформулировать 3-5 вариантов названия в потребительском языке
        ↓
3. ПРОВЕРИТЬ через Wordstat каждый вариант — объём + динамика 2024→2026
        ↓
4. Убрать варианты с объёмом <500/мес и варианты с падением >50%
        ↓
5. Проверить через чек-лист этого документа
        ↓
6. Утвердить → в материалы → фиксировать в post-launch learnings
```

**Без шага 3 — не финализировать. Каждое название = проверено через Wordstat.**

## Summary

| Объект | Главное правило |
|---|---|
| **Title эфира** | Через боль клиентки в её языке. Wordstat проверено. Без астро-терминов в title |
| **Название курса** | Что получит клиентка (профессия / доход / путь), не о чём курс |
| **Название модуля** | Результат прохождения, не название астро-техники |
| **Название консультации** | Близкое к запросу клиентки в Wordstat (объём ≥1000/мес) |
| **Название бонуса** | Конкретный артефакт + результат, не «эксклюзив» |

**Без проверки через Wordstat финализация названия = fail.**
