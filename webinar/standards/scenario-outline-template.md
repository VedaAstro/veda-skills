# Шаблон: SCENARIO-outline

## Зачем этот промежуточный слой

Пишущий веб не может держать в контексте 100+ слайдов TSV одновременно. Смотрящий веб не может читать 100 слайдов дословной речи чтобы поймать структурные провалы. Outline решает и то, и другое:

- **Для пишущего:** карта всей арки на одном экране → не теряется ни один щелчок / защита / хук / drip
- **Для смотрящего:** читается за 15 минут → ловит провисы ритма, дубли, пропуски щелчков, перекосы блоков
- **Для правок:** одна строка outline = одно решение. Одна строка TSV = целый слайд переписывать

Порядок работы: **бриф → outline → ревью → TSV → Google Slides**.

## Обязательные разделы документа

### 1. Header — мета
Короткая таблица из брифа: тип веба / база / CR-цель / повод / главный сегмент / главная защита / продукт.

### 2. Route-map
6 блоков архитектуры × тайминг × главная задача блока (одной строкой). Чтобы сразу видеть пропорции эфира.

### 3. Карта 3 щелчков идентичности
Где именно срабатывает каждый щелчок (№ слайдов + механика). Из `core/mission.md`.

### 4. Карта защит из 9K
Где преутилизируется каждая из 7 защит (№ слайдов + как декодируется). Из `buyer/defenses.md`.

### 5. Карта рабочего листа зрителя
Этапы того что зритель физически рисует по ходу эфира — единая растущая диаграмма (8 этапов).

### 6. Легенда тегов
```
OPEN    — приветствие / техсверка
HOOK    — open loop / Netflix-крючок
Н0-1..4 — слой Н0
EXEC    — упражнение / действие на листе
ASTRO   — астро-механика (диагноз)
CASE    — кейс выпускницы
ID-1..3 — щелчок идентичности
DEF-X   — декодирование защиты
MISS    — миссионерский переход
BIO     — био Ирины
OFFER   — ценностный блок / тариф / цена
GUAR    — гарантия
URG     — urgency через факт
BRIDGE  — переход между блоками
CLOSE   — закрытие
```

### 7. Таблица слайдов (основное тело)
Столбцы:
- **№** — номер слайда
- **Мин** — точка на таймлайне
- **Тег** — роль слайда (из легенды)
- **Смысл** — что зритель выносит после слайда (не что говорится, а что он понимает или чувствует)
- **Образ/механика** — какой рабочий образ или астро-связка (из арсенала `core/narratives.md`)
- **Интерактив** — действие зрителя (чат / лист / пауза)
- **Связка** — как подхватывает предыдущий (одной фразой)

### 8. Финальные проверки
Чеклист перед передачей в TSV:
- [ ] Формула покупки сходится к слайду ~95 мин
- [ ] 3 веры посажены (место каждой)
- [ ] 3 щелчка идентичности прошли (маркеры на листе)
- [ ] 8 инвариантов из WEB-ARCHITECTURE соблюдены
- [ ] Образный нарратив в каждом астро-слайде
- [ ] Ритм: нет 3 продающих подряд, нет 15 астро без drip

### 9. Changelog
Список правок по ходу ревью (чтобы видеть эволюцию решений).

## Что в документе НЕТ

- Полных notes (это TSV)
- Финальных title (только черновые смыслы)
- Img_tag (подбирается после того как смыслы стабилизированы)
- Дословной речи Ирины
- Финальных эмоджи в body

Задача outline — **смысловая карта**, не форма. Форма = следующий шаг (TSV).

## Правила работы с outline

1. **Сначала route-map и карты щелчков/защит/листа** — крупный каркас
2. **Потом таблица слайдов** — каждый слайд получает тег, смысл, образ
3. **Проверка чеклистом** — ловим структурные дыры
4. **Правки по ревью** — одна строка за раз
5. **Только после OK** — перенос в TSV с добавлением дословной речи, title, body, img_tag
