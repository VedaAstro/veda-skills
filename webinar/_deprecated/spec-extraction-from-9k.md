## Шаг 3. Задача № 2 — Карта защит (декодирование)

Для каждой повторяющейся защитной формулировки из `main_problems` построить таблицу:

### Базовые защиты для поиска (из data-bias.md)

1. **«Найти себя / предназначение / саморазвитие»** — маркеры слоя 🟢
2. **«Помогать людям / быть полезной»** — соц-желательность
3. **«Для себя, если получится — буду консультировать»** — защита от провала
4. **«Не было страхов / страха нет»** — вытеснение
5. **«Интересно / любопытно / интересна астрология»** — обесценивание
6. **«Разобраться в себе / познать себя»** — мягкая форма кризиса
7. **«Улучшить финансы / финансовое положение»** — социально-приемлемая форма «нет денег»

### Что делать с каждой

Для каждой защиты:
1. Посчитать частоту
2. Найти **контекст** (что ещё в той же анкете? какой возраст, доход, профессия?)
3. Определить **скрытую боль** — через кросс-референс с `fears` и жизненными маркерами ТОЙ ЖЕ анкеты
4. Сформулировать **реконструкцию слоя 🔵** (как звучит эта боль в языке ЖИЗНИ, не в языке цели)

## Шаг 4. Задача № 3 — Надёжный слой 🟡 ОФФЕР из `fears` и `overcome_fears`

Поле `fears` наименее искажено — прямой вопрос «чего боялась при покупке». Здесь ищем **реальные барьеры**.

### Методология

1. Собрать все ответы `fears` (кроме пустых, «нет», «не было»)
2. Частотный анализ + кластеризация по 5 классическим барьерам:
   - Дорого (27%)
   - Не справлюсь (24%)
   - Нет времени (16%)
   - Не найду клиентов (7%)
   - Муж против (6%)
3. Найти **новые барьеры**, не покрытые классической пятёркой
4. Топ-20 дословных формулировок по каждому барьеру

### Обработать «не было страхов» правильно

387 анкет пишут «страха не было». Это **НЕ ответ** — это защита / вытеснение.

**Правильная обработка:**
- Не удалять эти анкеты, а **пересекать с полями `main_problems` и `purchase_triggers`** — что там?
- Скорее всего в других полях той же анкеты найдутся скрытые страхи («боялась не справиться» → «для себя, вдруг не выйдет»)
- Если нет — это указывает на ВИП-сегмент или повторных покупателей

## Шаг 5. Задача № 4 — Реконструкция слоя 🔵 ЖИЗНЬ

Это **не цитирование анкет**, а **реконструкция**:

### Алгоритм

1. Для каждого сегмента (A1/A2/B1/B2/C) взять все анкеты этого сегмента
2. Пересечь **защитные формулировки** с **жизненными маркерами той же анкеты**
3. Сформулировать **типовой внутренний монолог** — как этот человек думает ДО покупки
4. Проверить через `segments.md` (там уже есть базовые монологи — расширить, не заменить)

### Пример реконструкции

**Анкета A1, 42 года, 40К доход, бухгалтер, город N**
- `main_problems`: «хочу найти себя и своё предназначение, понять почему у меня всё как у всех»  → **🟢 рационализация**
- `fears`: «боялась что не справлюсь, что дорого» → **🟡 реальный страх**
- `application_plans`: «для себя, если получится — буду консультировать подруг» → **🟢 защита**
- `occupation`: «бухгалтер» — **факт**
- `age`: 42, `income`: 40К — **факты**

**Реконструированный слой 🔵 для этой анкеты:**
> «Мне 42, я бухгалтер 15 лет, 40 тысяч на руки. Встаю в 6:30, дети, работа, вечером усталая домой. Смотрю на подруг которые что-то пробуют, что-то делают — а я как белка в колесе. И внутри: а я зачем? Всю жизнь для кого-то — для родителей, для детей, для начальника. А я сама кто? И страшно даже подумать что-то сменить — куда я пойду в 42 года? Кому я нужна?»

Это **не цитата** из анкеты. Это **реконструкция** через слой 🔵 на основе **фактов** анкеты + знания buyer playbook + язык ЦА.

### Результат

Файл `webinar/buyer/life-layer-reconstruction.md` — новый файл с 5 полными монологами (по одному на сегмент), каждый 200-400 слов, с конкретными жизненными маркерами из анкет, **без защитных формулировок**.
