> Источники: Stephen Few ("Information Dashboard Design"), Edward Tufte ("Visual Display of Quantitative Information"),
> Cole Nussbaumer Knaflic ("Storytelling with Data"), Geckoboard, Tableau Blueprint, GoodData, Sigma Computing,
> Onetribe Advisory, Stacey Barr, Power BI Consulting. 62 правила из 12+ профессиональных источников.

## A. КОГНИТИВНЫЕ ОГРАНИЧЕНИЯ (почему дашборды проваливаются)

**A1. Закон Миллера (7 +/- 2).** Max 5-9 визуализаций на экране. Рабочая память = ~7 элементов.

**A2. Правило 5 секунд.** Руководитель понимает здоровье бизнеса за 5 секунд. Не получается = дашборд провалился.

**A3. Правило 1 экрана.** Дашборд НЕ скроллится. Если нужен скролл - это отчёт, не дашборд.

**A4. Каждый элемент = когнитивная цена.** "Every visual adds cognitive cost" - убирай всё, что не сокращает время принятия решения.

## B. ВИЗУАЛЬНАЯ ИЕРАРХИЯ

**B1. Перевернутая пирамида:**
```
УРОВЕНЬ 1 (верх экрана, без скролла):
  3-5 KPI-карточек = здоровье бизнеса за 5 секунд

УРОВЕНЬ 2 (середина):
  1-2 графика = тренды и сравнения, ОБЪЯСНЯЮЩИЕ уровень 1

УРОВЕНЬ 3 (низ):
  Таблица = детализация для drill-down
```

**B2. F-паттерн.** Глаз сканирует: горизонтально вверху, потом вертикально вниз по левому краю. Самый важный KPI = верхний левый угол.

**B3. Размер = важность.** KPI-числа крупнее всего (display-sm). Подписи - мелкие (text-sm). Если всё одного размера - ничего не важно.

**B4. Max 3-4 уровня визуального приоритета:**
1. Primary KPI (display-sm font-semibold) - то, на что смотрят первым
2. Вторичные метрики (text-md) - поясняют primary
3. Детали (text-sm) - подписи, легенды
4. Фон (text-xs text-tertiary) - timestamps, определения

**B5. Max 3 типа визуализаций на дашборде.** Scorecard + BarChart + Table = достаточно. Добавление LineChart, PieChart, HalfCircle, Area, Radar = хаос.

## C. ЛОГИКА ДАННЫХ: "ТАК ЧТО?" ТЕСТ

### C1. Каждое число отвечает на "это хорошо или плохо?"

"$10M revenue" бессмысленно без контекста. Каждая метрика сопровождается **минимум одним** из:

| Контекст | Что показывает | Когда использовать |
|----------|---------------|-------------------|
| **Таргет/план** | Факт vs план + % выполнения | Есть утвержденный бюджет/план |
| **Прошлый период** | +18% vs прошлый месяц | Нет плана, но есть история |
| **Бенчмарк** | Наш CR 3.2% vs рынок 2.5% | Есть отраслевые данные |
| **Тренд** | Sparkline за 6-12 периодов | Нужно направление движения |

**Без контекста метрика запрещена на дашборде.**

### C2. Фильтр "Если изменится - что сделаем?"

Перед добавлением метрики: "Если эта метрика существенно изменится, что мы сделаем по-другому?" Ответ "ничего" = vanity metric, убрать.

### C3. Заголовок = вывод, не описание

- ЗАПРЕЩЕНО: "Данные продаж Q1"
- ПРАВИЛЬНО: "CR продаж упал на 22pp от таргета"
- ПРАВИЛЬНО: "Западный регион обеспечил 68% роста Q4"

## D. АБСОЛЮТНЫЕ ЧИСЛА vs ПРОЦЕНТЫ vs RATES

### D1. Правило: никогда одно без другого

Процент при малой базе = обман ("рост 300%" при базе 3 штуки).
Абсолют при меняющейся базе = обман (рост смертей при росте населения).
**Показывай оба**: "347 лидов (+12% vs прошлый месяц)"

### D2. Таблица выбора формата

| Тип метрики | Формат | Контекст | Пример |
|---|---|---|---|
| Деньги (выручка, бюджет) | Абсолют + % изменения | План, прошлый период, YoY | 1.2M (+18% MoM, 83% плана) |
| Конверсия (CR) | % с 1 десятичным | Прошлый период, бенчмарк | 3.2% (было 2.8%, рынок 2.5%) |
| Объём (лиды, заказы) | Абсолют | План, прошлый период | 347 лидов (план 400, -13%) |
| Скорость (время отклика) | Абсолют в единицах времени | SLA/таргет, тренд | 2.3 дня (SLA 3 дня) |
| Удовлетворённость (NPS) | Балл + изменение в п.п. | Прошлый период, бенчмарк | NPS 42 (+5 п.п., рынок 35) |

### D3. Процент для сравнения, абсолют для действия

Руководитель сравнивает каналы по CR (проценты), но принимает решение о бюджете в абсолютных деньгах. Конверсия 5% при 100 визитах = 5 продаж. Конверсия 2% при 10,000 = 200 продаж.

### D4. Проценты при малой базе (<1K) = шум

Рост с 3 до 9 = "+200%". Это не инсайт, это статистический шум. При базе <1K показывай абсолюты.

## E. ПЛАН/ФАКТ/GAP

### E1. Формула

```
Абсолютный GAP = Факт - План
Процентное выполнение = Факт / План * 100%
Процентное отклонение = (Факт - План) / План * 100%
```

### E2. Что показывать

| Тип метрики | GAP формат | Почему |
|---|---|---|
| Выручка, деньги | ОБА: "gap -300K (-20%)" | Деньги = язык руководства, % = масштаб |
| Расходы | Абсолют первичен | "Перерасход 50K" > "+8.3%" |
| CR, конверсия | Процентные пункты (п.п.) | "gap -0.8 п.п." |
| Кросс-отдел | % отклонения | Нормализует разный масштаб |

### E3. Пороги status colors (RAG)

```
>= 90% плана  ->  зелёный (success)  ->  на траектории
70-90% плана  ->  жёлтый (warning)   ->  мониторинг
< 70% плана   ->  красный (error)    ->  требует действия
Нет плана     ->  нейтральный        ->  ЗАПРЕЩЕНО красить
```

**Красный = только для реальных проблем.** Не для декора. Когда всё красное - ничего не важно.

### E4. Декомпозиция GAP на драйверы

Не показывай только итоговый gap. Разложи:
- **Volume**: продали больше/меньше штук
- **Price**: средний чек выше/ниже
- **Mix**: изменился состав продуктов
- **Timing**: оплаты пришли раньше/позже

## F. RUN RATE (прогноз внутри периода)

### F1. Формула

```
Monthly Run Rate = Факт MTD * (дней в месяце / прошедших дней)
% прогноза выполнения = Run Rate / План * 100%
```

### F2. Карточка KPI с run rate (золотой стандарт)

```
Выручка (апрель)
Факт MTD: 420K
Run Rate: 1.2M
План месяца: 1.5M
Прогноз выполнения: 80%
[===████████░░░░] 7/30 дней
```

### F3. Три линии на графике внутри месяца

1. **Кумулятивный факт** (сплошная) - что заработали
2. **Линейный план** (пунктир) - равномерный темп для выполнения плана
3. **Run rate проекция** (полупрозрачная) - куда идём при текущем темпе

### F4. Когда run rate обманывает

- Сезонный бизнес (декабрьский темп != мартовский)
- Разовые продажи в периоде (исключать перед расчётом)
- Малый период (run rate по 3 дням = шум)

## G. ВОРОНКА КОНВЕРСИИ

### G1. Step-to-step CR, НЕ только overall CR

```
Визиты -> Регистрация: 12%    (норма)
Регистрация -> Заявка: 45%    (норма)
Заявка -> Оплата: 5.5%        (КРАСНЫЙ, норма 15-25%) <- тут ломается
─────────
Overall CR: 3%
```

### G2. Revenue Impact > Percentage Drop

Не чини этап с самым большим % падения. Считай потерянные деньги:
```
Revenue at Risk = Трафик на этапе * Drop-off * Средний чек
```

### G3. Сегментируй воронку

ВСЕГДА минимум по:
1. Устройство (мобайл конвертит ниже десктопа)
2. Источник трафика (органика vs платный)
3. Когорта (новые vs возвращающиеся)

### G4. Диагностика красного этапа

1. **Awareness gap**: не видит CTA
2. **Motivation gap**: видит, не убеждён
3. **Friction**: хочет, но сложно (длинная форма)
4. **Technical**: сломана страница

## H. ВРЕМЕННЫЕ СРАВНЕНИЯ

### H1. Что с чем сравнивать

| Метрика | Сравнение | Почему |
|---|---|---|
| Выручка | YoY | Убирает сезонность |
| CR | WoW | Быстрый сигнал |
| Трафик | MoM + YoY | Тренд + сезонность |
| Новый канал | WoW первые 8 недель | Нет YoY истории |

### H2. Минимум точек для тренда

- **Sparkline**: 6-12 периодов (восприятие направления)
- **Trend line**: 20+ точек (статистическая значимость)
- **Moving Average**: окно = период цикла (7 дней для дневных данных)

### H3. Частота обновления = частота решений

| Частота | Что показывать | Пример |
|---|---|---|
| Ежедневно | Пейсинг, операционные метрики | Расход рекламы, заявки |
| Еженедельно | CR, pipeline, активность | Воронка продаж |
| Ежемесячно | Revenue, ROI, CAC, LTV | P&L |
| Ежеквартально | Brand health, атрибуция | Стратегический обзор |

## I. TUFTE: ГРАФИЧЕСКАЯ ЧЕСТНОСТЬ

### I1. Data-Ink Ratio

```
Data-Ink Ratio = Data-Ink / Total Ink
Цель: стремись к 1.0
```

Каждый пиксель несёт данные. Всё остальное - удалять: 3D, тени, градиенты, фоновые паттерны, декоративные рамки.

### I2. Chartjunk - удалять немедленно

- 3D-эффекты и тени на графиках
- Декоративные рамки вокруг графиков
- Фоновые изображения и текстуры
- Тяжёлые gridlines (осветлить до border-secondary, толщина 1px)
- Pie charts и donut charts (Few: "generally poor choices")
- Gauges и speedometers (Few изобрёл bullet chart как замену)

### I3. Gridlines

- Горизонтальные > вертикальных
- Через 2-3 метки оси, не каждую
- Цвет: stroke-border-secondary (не чёрные)
- Цель: 60%+ пикселей графика = данные

## J. ANTI-PATTERNS CHECKLIST

| # | Антипаттерн | Что делать |
|---|---|---|
| 1 | Начинать с данных, а не с вопросов | Сначала: "что руководитель должен решить?" |
| 2 | >9 визуализаций на экране | 5-8 KPI max, остальное в drill-down |
| 3 | Pie charts / donut charts | Bar chart или stacked bar |
| 4 | Gauges / speedometers | Bullet chart или ProgressBar |
| 5 | Метрика без контекста | План/тренд/сравнение обязательно |
| 6 | Скролл на дашборде | Один экран или разбить на несколько |
| 7 | Декоративные элементы | Data-ink ratio -> удалять шум |
| 8 | Заголовок-описание | Заголовок = вывод ("CR упал на 22pp") |
| 9 | Абсолют без процента | Показывать оба |
| 10 | Процент при базе <1K | Показывать абсолют |
| 11 | Всё красное | Красный только >10% от плана |
| 12 | Нет timestamp | "Last Updated" обязателен |
| 13 | Нет определений метрик | Tooltip с определением |
| 14 | Смешивание аудиторий | Executive и operational - разные дашборды |
| 15 | Тренд из 1-2 точек | Min 4 точки, лучше 6-12 |
| 16 | Номинальные деньги в timeseries | Корректировать на инфляцию/сезон |

## K. GOLDEN TEMPLATE: KPI-КАРТОЧКА

```
┌─────────────────────────────────────────┐
│ Выручка MTD                    [зелёный]│  <- название + traffic light
│                                         │
│ 1 247 000 R                             │  <- текущее значение (КРУПНО)
│ ▲ +18% vs прошлый месяц                │  <- сравнение с прошлым периодом
│ План: 1 500 000 | Выполнение: 83%       │  <- таргет + % выполнения
│ ▁▂▃▃▅▅▆▇▇█ (sparkline 12 мес)          │  <- тренд
└─────────────────────────────────────────┘

5 элементов:
1. Название + status color (зелёный/жёлтый/красный)
2. Текущее значение (самый крупный шрифт)
3. Дельта к прошлому периоду (стрелка + %)
4. Таргет + % выполнения (или run rate если середина периода)
5. Sparkline (6-12 периодов)
```

## L. 4 ВОПРОСА РУКОВОДИТЕЛЯ

Каждая визуализация мгновенно отвечает на:
1. **Что не так?** (проблема)
2. **Где?** (локализация - отдел, канал, продукт)
3. **Насколько плохо?** (масштаб в рублях)
4. **Кто отвечает?** (ownership)

"If visuals don't answer these questions quickly, they get ignored."
